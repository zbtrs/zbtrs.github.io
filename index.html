算法竞赛进阶指南第七章

发表于 2018-06-29 | 分类于 算法竞赛进阶指南 |

前言

  终于在暑假到来前干完了这本书.

1.dijkstra算法的独特用法

  1.图论模型的最值问题，因为dijkstra每次取出权值最小的点来进行转移，所以到达每个状态的花费一定是最小的，因此可以用dijkstra求解最优值问题. (dp转移是min,max类型的).
  2.没有负边权的最短路. 不要用spfa!
  3.最短路和次短路的计数问题……

2.差分约束系统技巧

  1.转成≤号连边，求最短路，可以求得最大值. 转成≥号连边，求最长路，可以求得最小值.
  2.注意找一些非常显而易见的不等式!
  3.如果有等式，则转化为两个不等式.
  4.如果出现了超过2个未知数的不等式，则枚举其中的n-2个未知数.

3.双连通分量

  求点双连通分量要注意当前点是否为根节点.
  求边双连通分量要注意当前边是不是上一条边的反向边.
  边双连通分量利用并查集缩点，点双连通分量利用vector连边缩点，最后都会形成一棵树!

4.必经边/点问题

  有向无环图的必经边/点利用方案数来求.
  无向图的必经边/点利用tarjan缩点来求.
  普通有向图的必经边/点貌似要用支配树来做？太难了我不会……

Description
煤矿工地可以看成是由隧道连接挖煤点组成的无向图。为安全起见，希望在工地发生事故时所有挖煤点的工人都能有一条出路逃到救援出口处。于是矿主决定在某些挖煤点设立救援出口，使得无论哪一个挖煤点坍塌之后，其他挖煤点的工人都有一条道路通向救援出口。请写一个程序，用来计算至少需要设置几个救援出口，以及不同最少救援出口的设置方案总数。

poj1275 Cashier Employment

发表于 2018-06-29 | 分类于 算法竞赛进阶指南 |

题目大意

在一家超市里，每个时刻都需要有营业员看管，R(i) (0 <= i < 24)表示从i时刻开始到i+1时刻结束需要的营业员的数目，现在有N(N <= 1000)个申请人申请这项工作，并且每个申请者都有一个起始工作时间 ti，如果第i个申请者被录用，那么他会连续工作8小时。现在要求选择一些申请者进行录用，使得任何一个时刻i，营业员数目都能大于等于R(i)。

分析

  这道题主要难在建模.
  a[i] 表示在第i时刻开始工作的人数，是个未知量
b[i] 表示在第i时刻能够开始工作人数的上限， 是个已知量
R[i] 表示在第i时刻必须值班的人数，也是已知量.
令 s[i] = a[0] + … + a[i]，特殊地，s[-1] = 0
上面三个式子用s[i]来表示，如下：
s[i] - s[i-8] >= R[i] (i >= 7) (1)
s[i] + s[23] - s[i+16] >= R[i] (0 <= i < 7) (2)
0 <= s[i] - s[i-1] <= b[i] (0 <= i < 24) (3)
  因为差分约束系统要求每一个不等式都只能有2个未知数，观察第二个不等式，有3个未知数. 而$s_{23}$在所有不等式中都出现过，所以可以采用枚举的方法消除未知数!
  差分约束系统中引入等式，就相当于引入两个不等式. 对于当前枚举的值x,从0向23连边x,23向0连边-x即可.

小结

  未知数个数比较多就枚举确定其中的几个未知数.

中文题面

Description
小Q最近学习了一些图论知识。根据课本，有如下定义。树：无回路且连通的无向图，每条边都有正整数的权值来表示其长度。如果一棵树有N个节点，可以证明其有且仅有N-1 条边。 路径：一棵树上，任意两个节点之间最多有一条简单路径。我们用 dis(a,b)
表示点a和点b的路径上各边长度之和。称dis(a,b)为a、b两个节点间的距离。
直径：一棵树上，最长的路径为树的直径。树的直径可能不是唯一的。
现在小Q想知道，对于给定的一棵树，其直径的长度是多少，以及有多少条边满足所有的直径都经过该边。

Input
第一行包含一个整数N，表示节点数。
接下来N-1行，每行三个整数a, b, c ，表示点 a和点b之间有一条长度为c
的无向边。

Output

共两行。第一行一个整数，表示直径的长度。第二行一个整数，表示被所有
直径经过的边的数量。

Sample Input

6

3 1 1000

1 4 10

4 2 100

4 5 50

4 6 100

Sample Output
1110

2

【样例说明】

直径共有两条，3 到2的路径和3到6的路径。这两条直径都经过边(3, 1)和边(1, 4)。
HINT
对于100%的测试数据：2≤N≤200000，所有点的编号都在1..N的范围内，

边的权值≤10^9。

代码

分析

Codeforces 701 F. Break Up

发表于 2018-06-29 | 分类于 计划 |

题目大意

  n个点的无向图，每条边有一个费用，要求割掉不超过2条边，使得s不能到达t,求最少费用,输出方案.

代码

分析

  一开始想着求双连通分量然后缩点,枚举割桥还是割一个双连通分量内的两条边. 桥比较好处理，但是双连通分量内部的边就不好处理了.
  换一个思路. 既然要s与t不连通，那么肯定是破坏掉s到t的所有路径. 先随便找一条从s到t的路径. 随便删掉该路径上的一条边来破坏这条路径. 这时可能还会有其他的路径,那么就要找是否存在必经边，也就是桥. s和t必须分布在桥的两边. tarjan即可.
  需要注意几种特殊情况的判断. 如果删掉该边后从s不能到t,那么只需要删一条边即可，如果可以到，并且不存在桥，则无解.

Codeforces 701 E. Connecting Universities

发表于 2018-06-28 | 分类于 计划 |

题目大意

  n个点的树上有2k个特殊点，要把这些点分成不相交的k对，使得每一对的点在树上的路径长度和最大.

分析

  这道题有两种做法，都非常神奇.
  1.对于每条边$(u,v)$,统计以v为根的子树内有多少个特殊点,以v为根的子树外有多少个特殊点，这条边的贡献就是$min(size_v,n - size_v)$. 可以用意识流证明：每个点连出去的路径尽量往外延伸. 或者利用树的性质证明: 受到点对数的限制，边的贡献不会超过$min(size_v,n - size_v)$,若边的贡献小于$min(size_v,n - size_v)$,则说明有一对点的路径没有经过边$(u,v)$，那么在v的子树内就会有一条从a到b的路径，v的子树以外的部分有一条从c到d的路径，这两条路径不经过边$(u,v)$. 如果让a连到c,b连到d,答案就会增加. 那么只需要一次dfs即可完成统计.
  2.写出答案的表达式：$ΣΣdep_u + dep_v - 2dep_{lca(u,v)}$. 前面一部分是固定的，那么只需要后面一部分最小即可. so,树形dp?点数太多，没有办法定义状态，于是想到贪心: LCA自然是越往上越好. 从上往下，找到当前点u拥有特殊点最多的子节点v,如果$size_v ≤ size_u - size_v$,则u子树内的所有特殊点都能以u为LCA. 否则就匹配掉v以内的$size_u - size_v$个点，v内剩下的点递归处理.这个做法写起来非常麻烦，需要讨论u是不是特殊点.点数最多的子节点有多少个.

小结

  求整体答案对应的两种方法：1.计算贡献. 2.求答案的表达式.

分析

  一眼线段树题.
  这道题我一开始想着维护区间$[l,r]$内满足要求的数的最左下标，以及$a_i - sum_{i - 1}$,如果一个位置上的$a_i - sum{i - 1} = 0$,则$pos_i = i$,否则$pos_i = n + 1$. pos很好合并，前面这个东西不好搞，不能直接累加合并. 于是我的一个想法就是只操作叶子区间. 向上传递pos即可. 这样做有一个问题：标记无法及时下传. 查询时需要将所有区间都访问一遍，才能修改所有叶子区间的值，并更新pos的值.

CH17 舞动的夜晚

发表于 2018-06-18 | 分类于 算法竞赛进阶指南 |

题目大意

  求二分图最大匹配的不可行边.

代码

分析

  1w个点，10w条边，如果直接上匈牙利算法就会爆掉. 正确的做法是网络流. 先做一次dinic，在残余网络上做文章.
  (x,y)为不可行边的条件是：(x,y)的流量为0，并且在残余网络上属于不同的强连通分量. 如果流量为1,说明(x,y)可以是最大匹配边. 如果x,y在同一个强连通分量里，那么将该强连通分量内的所有边的状态取反，可以得到另一组最大匹配. 可以画几个图来观察性质得到这个结论.

小结

  利用tarjan算法和网络流来求最大匹配的可行边，必须边与不可行边. 注意是找到强连通分量内边的特征，反转所有边后可以找到一条增广路，从而得到另一组最大匹配.
