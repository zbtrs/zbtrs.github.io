CH17 舞动的夜晚

发表于 2018-06-18 | 分类于 算法竞赛进阶指南 |

题目大意

  求二分图最大匹配的不可行边.

代码

分析

  1w个点，10w条边，如果直接上匈牙利算法就会爆掉. 正确的做法是网络流. 先做一次dinic，在残余网络上做文章.
  (x,y)为不可行边的条件是：(x,y)的流量为0，并且在残余网络上属于不同的强连通分量. 如果流量为1,说明(x,y)可以是最大匹配边. 如果x,y在同一个强连通分量里，那么将该强连通分量内的所有边的状态取反，可以得到另一组最大匹配. 可以画几个图来观察性质得到这个结论.

小结

  利用tarjan算法和网络流来求最大匹配的可行边，必须边与不可行边. 注意是找到强连通分量内边的特征，反转所有边后可以找到一条增广路，从而得到另一组最大匹配.

二分图

发表于 2018-06-18 | 分类于 算法竞赛进阶指南 |

前言

  感觉以前学了假二分图.

定义

  1.无向图.
  2.所有点被分成两个非空点集，每个点集内部没有边.

性质及判定

  没有奇环的无向图是二分图. 逆定理也成立. 可以利用这个定理判断一个图是不是二分图，也可以判断图上有没有奇环.
  如何判定？染色法! 每次找一个没有被染色的点开始黑白染色. 如果访问到一个点和其相邻的点颜色相同，则一定存在奇环.
  如果扩展域并查集只扩展了一个域，那么可以转化为二分图来求解. 处理方法是二分+二分图判定. 例题：noip2010关押罪犯.

二分图的最大匹配

  两种算法：1.最大流. 2.匈牙利算法.
  最大流算法更快，时间复杂度$O(sqrt(m)n)$,匈牙利算法的时间复杂度：$O(nm)$.
  介绍一下匈牙利算法：
  本质是不断地找增广路，利用增广路的性质使得匹配数不断+1. 从一个非匹配点出发，到另一个非匹配点，这两个点之间的路径上非匹配边和匹配点交替出现，则称这条路径为一条增广路.
  因为是从左到右到左……最后到右，所以路径长度为奇数. 而且路径上第1,3,5,……,len条边是非匹配边，2,4,……,len-1条边是匹配边，所以非匹配边比匹配边多1，将路径上所有边的状态反转，就能使得匹配数+1.
  匈牙利算法的主要过程是找增广路. 枚举左部的点x和边(x,y). 如果(x,y)本身就是一条非匹配边，则构成一条长度为1的增广路. 若y已经被匹配了，则找到y的匹配点x’,从x’出发找到能与x’匹配的非匹配点y’,那么x—y—x’—y’就是一条增广路.
  因为是依次考虑左边的非匹配点(一个一个考虑)，并且不会将已经匹配的点变成非匹配的点，所以这样找出来的匹配数一定是最大的.
模型特点：0-1准则:
  0.该问题可将状态抽象成点，并且恰好分成两个非空点集.
  1.每个点最多只能连一条边.
比较经典的模型是棋盘模型，行向列连边.
  注意：每个点只能属于一个点集！在棋盘上可以通过黑白染色来分配点.

二分图的多重匹配

  一个点最多能连$a_i$条边，那么将该点拆成$a_i$个点即可.
  或者利用网络流的边的容量限制.
  二分图的最大匹配数表示的意义非常狭小，如果要求具体的答案，不便于用最大匹配数来表示，则采用枚举+二分图匹配判定的方法.详见bzoj导弹防御塔.

二分图的覆盖集与独立集

  选出最少的点，使得每条边的端点至少有一个被选出.
  一个结论：二分图的覆盖集大小=二分图的最大匹配数.
  另一个结论：二分图的独立集大小=n-二分图的最大匹配数.
  模型：一条边对应的两个点至少有一个被选. 即n个任务，每个任务有两种处理方式，必须至少选一种处理方式. 和2-SAT问题不同的是，两种处理方式可以都选. 例如poj2226.

有向无环图的最小路径点覆盖

  分两类：一类是路径不能相交，一类是可以相交.
  对于第一类问题，如果有边(x,y),则从左边第x个点连向右边第y个点. 从左部到右部的边对应DAG上的一条边，那么左部的非匹配点就对应一条路径的终点(出度为0).所以答案为n-最大匹配数.
  对于第二类问题，利用floyd传递闭包即可. 原来是不允许两条路径：x—>c—>y和x’—>c—>y’,所以只能单一传递，现在因为允许路径相交了，故只考虑点与点之间的连通性即可.
  有向无环图的最长反链=最小点路径覆盖. 最长反链就是选出尽量多的点，使得这些点之间不能互相到达. 证明方法比较简单，先证明最小点路径覆盖一定≥最长反链，接着构造一种方案，使得最长反链=最小点路径覆盖数.
  求出最长反链数比较简单，但是求出最长反链的一种方案比较难.因为最长反链=最小点路径覆盖数，而这些路径是互不相交的，所以可以一开始令每条路径的终点为最长反链上的点，然后逐步做出调整. 假设选出来的点集为E. E中所有点能到达的点的集合为next(E). 要符合反链的要求，就要$E∩next(E) = ∅$ 先把next(E)处理出来，如果有点x在$E∩next(E)$中，那么就从x跳到匹配点，从匹配点跳到匹配点的匹配点……直到x符合要求. 用跳到的点x’替换x. 可以证明一定能找到这样的x’,因为如果找不到，说明从x到其路径起点的上的所有点都能被其它路径的终点到达. 那么将终点能走到该路径起点的那一条路径延伸，就能使得路径数-1，与最小路径点覆盖的最小性矛盾.

二分图最大匹配的边

  分为三种：必须边，可行边与不可行边. 利用网络流和tarjan算法缩环来判定.详见ch17 舞动的夜晚.

后缀自动机SAM

发表于 2018-06-15 | 分类于 计划 |

前言

  网上的各色教程都很奇怪. 知乎上说学SAM可以看一个外国网站，或者直接看clj的论文. 外国网站写的确实很好懂，但是只讲了”how”,没有讲”what”. clj的论文应该算是最规范的讲稿，只是概念繁多，有些地方说的不是很清楚，导致有点难以理解.我在看了clj的论文后似懂非懂，查阅了网上众多学习笔记，最后看了一下dkf的课件才算搞懂. 这篇博客就简要记录一下我对SAM的理解吧.

为什么要构建这么一个东西？

  最大的好处是快!其次是好写. $O(n)$的复杂度，代码只有20行不到，比起后缀数组动辄十余个for循环好太多了.

这玩意儿是啥？

  先说一下自动机是什么东西. 我的理解是自动机可以对应一个有向图. 图上的点代表状态(抽象理解),边则代表转移. 自动机是对于某种特定的东西定义的，比如字符串，整数等等.以字符串为例,若字符串$p$从状态$s$经过转移$x$可以到状态$t$.,则对应的有向图上从$s$到$t$有一条边上信息为$x$的边. (转移过后p可能会发生改变).将自动机上的一些节点打上标记，这些被打上标记的节点称为$end$节点. 当$p$从初始状态开始，能够经过若干次转移到达一个$end$状态时，则称$p$是可该自动机可接受的.
  后缀自动机是只能识别一个字符串所有后缀的自动机.后缀自动机上的边对应一个字符的转移，从起始状态开始，从前往后依次读入字符并转移，如果最终能转移到一个$end$状态，则当前字符串是可被接受的.

一些定义和性质

  转移之前已经定义了，就是如果满足某个条件，那么从一个状态可以转移到另一个状态. 那么状态的定义是什么呢？我们需要找到描述字符串特征的量来记录状态.
  定义：
  1.$ST(a)$表示从字符串a从起始节点能够转移到的状态,$ST(a) = null$表示$a$在自动机上不能进行转移.
  2.字符串$y$接在字符串$x$后面形成的字符串为$xy$.
  若$a$不是$s$的一个连续子串，那么$ST(a) = null$. 因为不论$a$后面接什么字符，都不可能到$end$状态，形成$s$的一个后缀.
  若$a$是$s$的一个连续子串，只有$x$是$s$的一个后缀时，$ST(ax)$才是$end$状态. $x$要接在$a$后面，那么要找到$s$中所有$a$出现的右端点的位置，这些位置记做集合$right(a)$. 可以发现，若$right(a) = right(b)$,那么$a$与$b$的后继转移是一样的，因为能拼接后缀都是一样的.
  要让SAM的时间复杂度最优，就必须要让状态最简. 因为当$right(a) = right(b)$时，$a,b$的后继转移一样，所以状态可以记录right集合一样的字符串构成的集合.
直接记录字符串集合也不方便. 容易发现：如果知道了这些字符串出现的右端点的位置集合还有这些字符串的长度，就能唯一确定这些字符串. 但是会有一个问题：每个字符串的长度可能不一样，需要记录所有字符串的长度吗？
  一个结论：每个状态所表示的字符串集合的字符串长度构成了一个连续的区间. 若状态$s$内最短的字符串长度为$min(s)$,最长的字符串长度为$max(s)$,那么$s$记录的所有字符串的长度都在$[min(s),max(s)]$内.
  这个很容易意识流证明. 如果长度小了，那么这个字符串出现的位置也就变多了，如果长度大了，在某些位置就可能出现不了.

SAM线性复杂度的证明

  要证明SAM是线性的复杂度，就必须证明SAM的状态数是线性的，转移数是线性的.
  先证明一个定理：两个不同的状态所表示的right集合要么不相交，要么一个是另一个的真子集.
  因为每个状态表示不同的字符串集合，所以两个不同状态所表示的字符串是不会有重叠的. 如果两个状态u,v表示的right集合有交集，假设位置r在交集中. 因为两个状态表示的字符串集合不重叠，所以$[min(u),max(u)]$与$[min(v),max(v)]$不重叠. 假设$min(v) > min(u)$,那么在位置r上. u表示的所有字符串都能在v表示，并且v表示的字符串更多，因此u表示的right集合是v表示的right集合的真子集. 两个相交的right集合，一个是另一个的真子集.
   有了上面这条定理，就能够把right集合和树联系起来. 如果集合u包含集合v,那么就令u是v的祖先. 考虑最坏情况：树的节点数最多. 因为每个节点的儿子之间都不会产生交集，它们是父亲节点的真子集. 要使得节点数最多，那么每个节点就要有两个儿子，构成了一棵二叉树，最坏情况下点数为$2n-2$.
  下面来证明SAM的转移数是线性的.
  先构造一棵SAM的树形图(有向图的生成树). 假设有$n$个状态，那么就有$n-1$条树边. 从SAM的起始状态出发，经过若干条边到达一个$end$状态. 因为有$m$个后缀，那么最多就有$m$个$end$状态. 经过的路径可以只由树边构成，也可以由树边+非树边构成. 考虑最坏情况，因为树边的数量是固定的，那么就想办法让非树边尽量多. 也就是考虑每一条路径都产生尽量多的之前没有出现过的非树边的情况.
  盲目地统计可能会产生重复计数. 利用基准点计数原理. 将每一个后缀对应到其路径上经过的第一条非树边，这样最多会产生$n$条非树边，加上$n-1$条树边，总共就有$2n-1$条边，转移也是线性的. 故整体的复杂度是线性的.

如何构造？

  先定义一下parent树：
  对于一个状态x,若存在另一个状态y，使得x表示的right集合是y表示的right集合的真子集，并且y表示的right集合的大小最小，那么就令y是x的父亲. 和上述证明SAM的线性复杂度构造的树差不多. parent树上x表示的字符串长度区间为$[min(x),max(x)]$,x的父亲表示的字符串长度区间为$[min(y),max(y)]$,则$max(y) + 1 = min(x)$. 利用right集合的真子集关系与区间的连续性可以证明.
SAM的构造是在线的，将字符一个一个地插入到SAM中.
从字符串S的SAM到字符串SX的SAM的构造过程：
  1.设S的长度为len,新建节点np,$right(np) = len + 1$,np表示的字符串是SX本身，所以right集合大小为1.
  2. 对于一个状态x,若其right集合中不包含len,则这个状态不能转移. 已知上一次插入的节点p，它的right集合中一定包含len. 而在parent树中，它的祖先节点的right集合一定都包含len，所以尝试从p以及其祖先节点向np转移.
  3.不断地从p往上跳. 如果当前点没有到np的转移为x的边，则连一条转移为x的边到np上. 如果最后跳到了0,则令np在parent树上的祖先为1.
  4.如果中途有一个点$ST(p,x) != null$,令$q = ST(p,x)$.这说明p可以通过x转移到q. 当前新加了一个字符x到S末尾，那么能不能直接将q的right集合并上len+1呢？ 可能会出现问题! 新加的这一个位置有可能不合法! 为什么会出现这种情况呢？因为p已经有到q的转移x了,也就是有其他的转移y使得$ST(p,y) = q$(这样就有连边了),如果$x != y$,直接将len+1插入到right集合中就有可能会出错(通过y转移过去的,x不一定可行). 而如果不存在$ST(p,y) = q$,并且$x != y$,那么$len(q) = len(p) + 1$,np在parent树上的父亲就是q. 否则新建一个节点nq. nq的right集合是q的right集合并上len + 1. nq的父亲是q的父亲，q和np的父亲变成了nq. 所有到q的边都连到nq上，q连出去的边nq也连出去.
  第四部可能比较难以理解，特别是为什么要建nq. 可以这么来理解：p可以通过x转移到q,而len+1这个位置不一定能转移. 只有满足$len(q) = len(p) + 1$时该位置才可以转移. 因为p能通过x转移到q,说明len位置后加入x合法. (不会出现容纳不下的情况).

题目大意

代码

分析

  如果子串可以是不连续的，那么理论上最优复杂度是$O(n^2)$. 子串要求连续的话，可以用选用后缀数据结构来处理，因为所有后缀的前缀构成的集合就是该字符串所有子串构成的集合.
  本题中，字符串过长，故只能用后缀自动机来处理.
  先对一个串建SAM,另一个串在上面匹配. 类似AC自动机,指向parent树父亲节点的边就相当于AC自动机的失配指针. 记录一个变量LCS,如果当前点能够通过当前字符s[i]转移，则LCS++,进行转移. 否则不停的跳到fa节点上，直到可以转移. 如果最后跳到了0,则令当前节点为1号点，$LCS = 0$. 否则$LCS = len[p] + 1$.
  注意len数组表示的含义. 它表示状态p表示的字符串中最长的长度是多少.从当前节点不能进行转移，但是祖先节点可以转移,因为祖先节点表示的字符串一定比当前节点表示的字符串短，所以祖先节点表示的最长的字符串一定被匹配完了. 因为可以直接令$LCS = len[p] + 1$,而不能令$LCS = len[ST(p,s[i])]$.

题目大意

代码

分析

  普通的SAM是对于一个字符串而言的，所以先对一个字符串建SAM.
  在每个状态上记录一个匹配长度. 每一个串都在上面匹配. 对于当前串，可能会多次匹配到节点p,取最长的长度. 因为公共子串是对于所有字符串而言的，所以每个节点的匹配长度要取min. 最后的答案就是所有节点的匹配长度的max值.
  特别注意：若所有状态都要记录一个答案，最后的答案是所有状态的答案的最值时，要考虑子节点向父亲节点传递答案.
  当一个节点p能够被匹配时(匹配长度不为0)，那么它的父节点的匹配长度一定是$len[fa_p]$. 为什么呢？考虑怎么样才能转移到p上来. 两种途径：要么是先跳到父节点，再转移，要么是从初始状态一步步转移到p上来. 对于第一种途径，父节点表示的字符串已经被匹配完了. 对于第二种状态，既然能够走到p来，那么匹配长度至少是min(p),根据parent树的性质,$min(p) = max(fa_p) + 1$，因此p的父节点的字符串已经被匹配完了.
  因为SAM建出来的图是一个有向无环图，所以可以拓扑排序来更新父节点的信息. 更方便的，直接基数排序也可以. 不要用sort!这样会使复杂度退化!

题目大意

n个点m条边的DAG,从S点走到T点，如果这其中的所有路径都要经过某条边，则称这条边为桥边. 现在可以用两次传送的机会，可以在任意位置上下车，但是每一次连续行驶的距离不超过q,求步行经过的桥的最短长度.

代码

分析

  超级好题，不做后悔!
  先把图中所有的桥边求出来，怎么求呢？一般的有向图求桥边需要用到一种叫做支配树的高级数据结构，不过在DAG上，因为没有环，有一种特别简单的方法可以求出所有的桥边：求出从s到每个点的路径数，记做$fs_i$,从每个点到t的路径数，记做$ft_i$,如果一条边(x,y),满足$fs_x * ft_y == fs_t$,那么(x,y)就是一条桥边. 但是路径数太大了存不下怎么办？参考noip2014 d2t3的处理方法. 只有大数的比较操作，可以将大数对一个大素数取模，如果时间允许，可以对多个素数取模.
  求出桥边后，要用两条长度不超过q的链去覆盖一条从s到t的路径，使得覆盖的桥边长度尽量长. 因为桥边是所有路径都要经过的边，利用贪心的思想，求出任意一条从s到t的最短路. 这样的话如果两条桥边中间被隔开了，就可以以最小的代价覆盖它们.
  接下来就是dp了. 两条链的端点肯定有一端是在点上. 我一开始的想法是以每个点为阶段进行dp. 但是这样不好处理，因为不知道当前点是作为左端点还是右端点.而且枚举决策点，复杂度是$O(n^2)$的. 因为只有两条链嘛，这两条链肯定是不相交的(可以共点)，为了更方便的处理左右端点的情况. 令$f_i$表示从s到最短路上的第i个点用一条链的答案，$g_i$则表示最短路上的第i个点到t的用一条链的答案. 最后的答案就是$sum - max\{f_i + g_i\}$

一些坑

  发现了一个我写DAG上dp的错误：我竟然用dfs来dp…… 正确方法应该是topo排序.

分析

  对于第二类数据，不是特别好搞. 整体上来说是一个基环树，基本处理方法，先把环求出来. 对于环上的每一个点，以它为根，求出它的子树内的每个点的倍增数组，还要求出以它为一端，另一端在其子树内的最长链. (这里说的子树不包括环上其它点). 如果要求的两个点在同一个子树内，那么可以直接利用倍增求得答案，否则路径就是从一个点走到环上，然后走到另一个点. 那么就要先预处理出环上的点，以及环上的每个点到环上第一个点的路径长度. 涉及到点到下标的映射，处理起来比较麻烦.
