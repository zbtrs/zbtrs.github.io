poj1945 Power Hungry Cows

发表于 2018-05-07 | 分类于 算法竞赛进阶指南 |

英文题面

Description

FJ’s cows would like to be able to compute integer powers P (1 <= P <= 20,000) of numbers very quickly, but need your help. Because they’re going to be computing powers of very large numbers, they can only keep around two work variables for intermediate results.

The first of those work variables is initialized to the number (denoted x) for which they are calculating the power; the other is initialized to 1. The cows can both multiply and divide any pair of the work variables and store the result in any work variable, but all results are stored as integers.

For example, if they want to compute x^31, one way to perform the calculation is:
WV1 WV2

                                  Start:   x    1

Multiply first by first, store in second: x x^2

              Multiply second by second:   x   x^4

              Multiply second by second:   x   x^8

              Multiply second by second:   x   x^16

              Multiply second by second:   x   x^32

                 Divide second by first:   x   x^31

Thus, x^31 can computed in six operations. Given the power to be computed and the the number of work variables, find the minimum number of operations to calculate the power.

Input

A single line with one integer: P.
Output

A single line with a single integer that is the minimum number of operations it requires to compute the power.
Sample Input

31
Sample Output

6
Source

USACO 2002 February

翻译

一开始有两个数0,1,每次可以将其中一个数加上/减去其中的一个数（可以是它自己），并替换掉其中的一个数，问最少多少次操作可以使其中一个数为P.

代码

分析

  先选定搜索方法. DFS肯定是不可以的，很容易走到死胡同. 很显然要用广搜.
  数字P比较大，决策也比较多，直接搜肯定会TLE. 考虑优化算法A*,维护一个优先队列，每次取出当前操作数+估值最小的状态扩展.
  如何设计估价函数？ 利用快速幂算法，想让一个数尽可能快地接近另一个数，肯定是不断地复制它自己. 所以对于一个状态(x,y),假设较大者是y,找到最小的整数k,使得$2^k * y > P$,k就是估价值.
  A*算法需要判重，因为x,y比较大，不能直接用一个二维vis数组存储，利用hash表来判重，设计hash函数$H(x,y) = (x + y + x*y) \% mod$. 模数选一个比较大的7位素数.
  这样就完了？交上去会MLE. 优先队列占用了太多的内存, 如果输出一下优先队列的大小，会发现最大数据中大小会达到80w,直接用STL中的优先队列是过不了的，要手写堆.(毒瘤！)
  然后会发现TLE. woc?为什么都这样了还会TLE? 可以加上一些剪枝：
  1.若$(x,y)$不整除$p$,说明当前状态不合法.
  2.根据第一条剪枝，若x,y中的任何一个等于0，则gcd会陷入死循环中，由此可以想到另一条剪枝：若$x = y = 0$,一定不可能最优
  3.一个不容易想到的剪枝.若$x = y$,当前状态一定不可能最优. 怎么说呢，与其让x,y一样，不如让更大者更大，更小者更小. 根据快速幂算法原理，这样是更优的.
  终于，卡时1000ms过了，真tm毒瘤！

Description

Flood-it is a fascinating puzzle game on Google+ platform. The game interface is like follows:

At the beginning of the game, system will randomly generate an N×N square board and each grid of the board is painted by one of the six colors. The player starts from the top left corner. At each step, he/she selects a color and changes all the grids connected with the top left corner to that specific color. The statement “two grids are connected” means that there is a path between the certain two grids under condition that each pair of adjacent grids on this path is in the same color and shares an edge. In this way the player can flood areas of the board from the starting grid (top left corner) until all of the grids are in same color. The following figure shows the earliest steps of a 4×4 game (colors are labeled in 0 to 5):

Given a colored board at very beginning, please find the minimal number of steps to win the game (to change all the grids into a same color).

The following N lines show an N×N matrix (ai,j)n×n representing the game board. ai,j is in the range of 0 to 5 representing the color of the corresponding grid.

The input ends with N = 0.
Output

For each test case, output a single integer representing the minimal number of steps to win the game.

2
0 0
0 0
3
0 1 2
1 1 2
2 2 1
0

翻译

每次从0~5中选一种颜色，给左上角的点染色，和左上角的点相连的点也会被染成同样的颜色(若有路径连接这两个点，并且路径上的所有点的颜色都是一样的). 问最少多少次操作能够将所有点变成同一种颜色.

代码

分析

  比较有技巧性的一道搜索题.
  选定搜索方法：首先肯定是不能广搜的，每一层扩展的局面非常多，广搜记录不下所有的状态. 搜索不是最好的选择，因为可能会一条道走到黑（64个格子，每种格子变一种颜色，状态可能有$6^{64}$之多）.那就用迭代加深搜索. 可以变成IDA*来优化.
  将和左上角的格子相连的所有格子看作一个连通块. 枚举颜色，将在连通块边界附近并且与枚举的颜色相同的点加入到连通块中. 然后继续搜下一层. 整个过程只需要维护每个点是否在连通块内以及是否是连通块的边界.
  估价函数可以设计为当前有多少种不同的颜色数-1. 满足估价函数的要求.每次更改颜色操作不需要具体更改每个点的颜色，只需要更改连通块的信息以及边界即可.
  最奇葩的一点：对边界染色一定要用dfs写，不要用bfs写，因为在每一层开一个队列太耗时间！

bzoj4559 [JLoi2016]成绩比较

发表于 2018-05-07 | 分类于 计划 |

中文题面

Description
G系共有n位同学，M门必修课。这N位同学的编号为0到N-1的整数，其中B神的编号为0号。这M门必修课编号为0到M-
1的整数。一位同学在必修课上可以获得的分数是1到Ui中的一个整数。如果在每门课上A获得的成绩均小于等于B获
得的成绩，则称A被B碾压。在B神的说法中，G系共有K位同学被他碾压（不包括他自己），而其他N-K-1位同学则没
有被他碾压。D神查到了B神每门必修课的排名。这里的排名是指：如果B神某门课的排名为R，则表示有且仅有R-1
位同学这门课的分数大于B神的分数，有且仅有N-R位同学这门课的分数小于等于B神（不包括他自己）。我们需要
求出全系所有同学每门必修课得分的情况数，使其既能满足B神的说法，也能符合D神查到的排名。这里两种情况不
同当且仅当有任意一位同学在任意一门课上获得的分数不同。你不需要像D神那么厉害，你只需要计算出情况数模1
0^9+7的余数就可以了。
Input
第一行包含三个正整数N,M,K，分别表示G系的同学数量（包括B神），必修课的数量和被B神碾压的同学数量。第二
行包含M个正整数，依次表示每门课的最高分Ui。第三行包含M个正整数，依次表示B神在每门课上的排名Ri。保证1
≤Ri≤N。数据保证至少有1种情况使得B神说的话成立。N<=100,M<=100,Ui<=10^9
Output
仅一行一个正整数，表示满足条件的情况数模10^9+7的余数。

Sample Input
3 2 1

2 2

1 2
Sample Output
10

提示

  选用哪种方式计数？

代码

分析

  非常牛逼的一道题!
  题目想让我们干什么？计数！有什么方法？暴力数学dp法！
  先尝试数学法. 计数的步骤是先计算合法的相对排名的方案数，再计算在该排名下分数的方案数.排名可以用式子表示出来，但是无法计算在特定排名下分数的方案数.
  那么尝试使用dp. 先定义状态$f_{i,j}$表示前i个科目中,有j个人被碾压的方案数. 考虑如何从$f_{i,j}$转移到$f_{i+1,k}$,会发现转移不过来. why？因为被碾压的条件太苛刻了，只考虑当前的科目是不能确定这$k-j$是否被碾压. 换一个状态表示：令$f_{i,j}$表示前i个科目中，有j个人不被碾压的方案数. 那么从$f_{i,j}$转移到$f_{i+1,k}$就可以只考虑第i个科目对人的影响. 令$t_1 = k - j$,表示从被碾压的人中选k个人变成新的不被碾压的人(第i个科目的分数比B高). 而第i个科目有$R_i - 1$个人不被B碾压，剩下的$t_2 = R_i - 1 - t_1$个人就要从之前不被B碾压的人中选. (不能从被碾压的人中选，否则会使得k变大,矛盾). 方案数就是$C(n - j - 1,t_1) * C(j,t_2)$. 如果令科目i的所有人得分的方案数为$s_i$,那么$f_{i + 1,k} = f_{i,j} * C(n - j - 1,t_1) * C(j,t_2) * s_i$.
  下面来说一下$s_i$要怎么计算. 因为那$R_i - 1$个人的分数只要高于B的就好了，具体是多少分没有限制，剩下的人的分数要低于B的分数，只需要确定B的分数，就能得到方案数. 枚举B的分数x,则方案数为$Σ(U_i - x)^{R_i - 1} * x^{n - R_i} (1 ≤ x ≤ U_i)$, 因为$U_i$太大不好算. 事实上这是一个n-1次的多项式求和，答案一定可以表示为不超过n次的一个多项式. n+1个点即可确定一个n次多项式的唯一表示，利用拉格朗日插值法.
  先算出$y_1$到$y_{n+1}$的所有值，若中途碰到了上界$U_i$,返回，因为x是连续的，预处理出lsum和rsum. $lsum_i = lsum_{i - 1} * (U - i)$,rsum类似. 然后预处理出逆元的阶乘. 若$(n + 1 - i) \%2 == 1$,则计算出来的答案是负数，需要取相反数.（至于为什么，看代码实现）.

总结

  理一下基本思路、为什么要这么做、怎么样更好地实现？
  1.为什么dp?
  因为数学式子无法在计算排名的情况下计算分数的方案数.
  2.为什么状态的第二维定义为不能被超越的人数？
  因为被超越的人的定义太苛刻了，不能只通过一个科目的分数来决定.
  关于拉格朗日插值法：要求一个多项式的值，多项式的次数很小，但是要求的那一项很大，就可以利用拉格朗日插值法. 需要熟悉求和多项式可以用单个多项式来表示，n+1个点确定一个n次多项式，都涉及到次数的变化.
  对于x连续的n次多项式，可以利用预处理前缀+后缀信息来快速插值.

分析

  很显然是一道搜索题. BFS?状态记录不下. DFS?边数太多，容易进入不是最优解的搜索树中,而搜索树节点的扩展速度超级快. 那么将这两种做法综合起来，就是迭代加深搜索.
  迭代加深搜索可以限制一个搜索的深度，使得DFS函数的返回类型是bool型，这样一旦搜索失败就可以不用继续搜了. 而普通的DFS不行，不知道具体要搜多少层，求最优解，只能将所有的可能都给搜遍，最坏的结果是将所有边都给枚举一遍.
  为了避免多次搜到同一局面，规定一个搜索顺序：每次只搜当前网格图中边长最小的正方形. 枚举这个正方形上的每一条边，删掉这条边后继续递归下去. 如果边长最小的正方形都不行，那么其它的正方形肯定不行. 可以用“决策包容性”来证明. 这种策略所能到达的后继局面一定包含了搜其它正方形所能到达的后继局面. 破坏了当前局面的最小正方形，后继局面的最小正方形的边长一定不小于当前局面的最小正方形的边长,这样相当于把所有的正方形都给考虑到了，并且每种局面最多只会被考虑一次.
  想到了优先队列的BFS就要想到A*,想到了迭代加深搜索就要想到IDA* ,估价函数设计为：每次找到一个边长最小的正方形，砍掉它的所有边，计数器+1，直到找不到正方形为止. 这个估价函数一定满足估价函数的设计要求，因为对于一个正方形，我们只需要砍掉它的一条边就能破坏它，在估价函数中砍掉了它的所有边，将这个正方形可能影响到的所有正方形都考虑进去了. 所以$H()≤g()$

分析

  一个叫DLX的算法可以完美地解决此题. 尝试用搜索解决.
  对于9*9的数独，我们可以每次找能填的数最少的格子去填，但是对于一个16*16的数独，这样做会TLE. 数独的大小变大了，不合法的状态也就会变多对搜索进行剪枝，因为不能最优化剪枝，就进行可行性剪枝.
  考虑不合法的局面和只能填一个特定的数的格子. 先遍历所有的空格，若某个位置A~P都不能填，那么局面不合法，若某个位置只能填一个字母，填上去，不要递归.
  再遍历所有的行，若某个字母在这一行上没有出现，并且不能填在任何一个空位上，不合法. 若某个字母只能填在该行的一个空位上，填上去.
  对于每一列和十六宫格而言都是这样.
  然后判断一下数独是否已经填好了. 如果没有，就找可填字母最少的格子，枚举填哪一个字母. 递归下去. 如果成功了，返回true,否则就要还原以前的局面.为了方便还原，在递归之前将当前局面记录下来，如果失败了，覆盖即可.

总结

  可行性剪枝，考虑不合法的局面和必须的局面. 如果dfs不是用来求最优值，而是用来求是否可行（一种局面），那么成功了就不必还原以前的局面. 一个知识盲点.
