USACO 2005 January Gold ###翻译 一个星球上一天由n个小时构成，在第i个小时睡觉能够恢复$U_i$点体力，有一头牛要休息$B_i$个小时，可以不连续. 但是每连续的一段的第一个小时都不恢复体力，第N个小时和下一天的第1小时是连续的，要求每N个小时休息B小时，求如何安排使得每天恢复的体力最多？ ###分析   链上的dp和在环上的dp有什么不同？ ###代码

###分析   环形dp一个基本的处理方法是将环从某一处断开，形成一条链，再将这条链复制一份接到后面，在这条新链上求. 这种做法在这道题上会有问题：先要枚举这N天的起始点，然后枚举长度为n的序列，在这个序列上dp，时间复杂度是$O(n^3)$的.对于环形dp,还有另一种处理方法：先将环从任意一个位置断开，不考虑连接处的情况，然后再考虑连接处会有哪些情况，做两次dp,将答案合并就是原问题的答案. 在这道题中，如果将环从第N天处断开，那么在第1天休息就不能恢复体力. 连接处的情况就是：第N天和第1天休息，并且在第1天恢复体力. 那么在dp初始化和统计答案的时候强制让第1天和第n天休息即可.   $f_{i,j,0} = min{f_{i - 1,j,0},f_{i - 1.j,1}}$   $f_{i,j,1} = min{f_{i - 1,j - 1,0},f_{i - 1.j - 1,1} + u_i}$.

金字塔

发表于 2018-05-20 | 分类于 算法竞赛进阶指南

###中文题面 虽然探索金字塔是极其老套的剧情，但是这一队 探险家还是到了某金字塔脚下。经过多年的研究，科 学家对这座金字塔的内部结构已经有所了解。首先， 金字塔由若干房间组成，房间之间连有通道。如果把 房间看做节点，通道看做边的话，整个金字塔呈现一 个有根树结构，节点的子树之间有序，金字塔有唯一 的一个入口通向树根。并且，每个房间的墙壁都涂有 若干种颜色的一种。 探险队员打算进一步了解金字塔的结构，为此，他们使用了一种特殊设计的机器人。这 种机器人会从入口进入金字塔，之后对金字塔进行深度优先遍历。机器人每进入一个房间（无 论是第一次进入还是返回），都会记录这个房间的颜色。最后，机器人会从入口退出金字塔。 显然，机器人会访问每个房间至少一次，并且穿越每条通道恰好两次（两个方向各一次）， 然后，机器人会得到一个颜色序列。但是，探险队员发现这个颜色序列并不能唯一确定金字 塔的结构。现在他们想请你帮助他们计算，对于一个给定的颜色序列，有多少种可能的结构 会得到这个序列。由于结果可能会非常大，你只需要输出答案对 $10^9$ 取模之后的值。 输入格式 输入文件包含一行，含有一个字符串，表示机器人得到的颜色序列。 输出格式 输出一个整数表示答案。 样例输入 ABABABA 样例输出 5 ###提示   在树上dfs得到的一个序列，能想到什么？ ###代码

###分析   由在树上dfs形成的一个序列可以想到dfs序. 因为dfs是一个一个子树遍历的，所以一个子树在所形成的序列上是连续的. 由数据范围很容易想到区间dp,即令$f_{l,r}$表示子串S[l,r]对应着多少种可能的金字塔结构. 根据区间dp的一般转移形式，很容易想到$f_{l,r} = Σf_{i,k} * f_{k + 1,j}$.这种转移方式是错的.   考虑序列ABABABA,两种方案A|BAB|A|B|A和A|B|A|BAB|A是一样的，因为它们都对应一个A延伸出3个B. 为什么会错呢？因为k只是我们枚举的一个分割点, 每一层枚举一个分割点，到最后可能就会有多种决策产生同一种局面，而他们选择的分割点相同，只是选择的次序不同罢了,也就是说：我们考虑的是排列的情况，而题目要求组合的情况. 要去重就必须要在每一层中确定选择几个分割点，并且还要确定这几个分割点的位置. 这样就有点搜索的意味了，会TLE.   因为题目说子树是有序的，**那么我们只需要让每一步的不同决策可能产生的不同局面不同即可.**在第一棵子树上下手. 只考虑第一棵子树是由哪些点组成的，将k作为划分点. 如果k不相同，那么第一棵子树的大小也就不相同，整棵树也就不同了，就可以计数了. 在转移的时候，要先考虑只延伸一棵子树的情况. 因为遍历完第一棵子树后要回到根，所以在s[k] = s[r]时才能转移,在转移的时候一定要考虑状态的合法性！ ###总结   dp计数问题中，若多种决策会产生同一种最终局面，就必须改变决策，使每一层的决策所导出的局面都是不同的. 转移时要考虑状态的合法性(定义).

###分析   首先这道题很容易看出来要用区间dp做. 对于环，有两种处理方法：要么枚举删掉哪一条边，要么任意选取一条边删掉并复制一份接在后面变成一条长链. 比较好的是第二种方法，因为在这条新链上考虑一条长为n的子链，就相当于割掉了一条边，并且起点已知，不用再额外话$O(n)$的时间去枚举割掉哪条边.   状态转移方程也很显然，就是令$f_{i,j}$为区间[i,j]的最大值，枚举中间点k,$f_{i,j} = max{f_{i,k} op f_{k + 1,j}}$ . 最后枚举起点i为哪一个即可. 事实上这种做法是错误的.因为有负数的情况，该状态设计不具备最优子结构性.那么还要维护最小值，加一维$f_{i,j,0/1}$表示区间[i,j]的最大值/最小值. 分别维护即可. ###总结   环的两种处理方法，以及数的乘除运算求最值问题要考虑负数与0的情况.
