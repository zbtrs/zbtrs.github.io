poj1084 Square Destroyer

英文题面

Description

The left figure below shows a complete 33 grid made with 2(3*4) (=24) matchsticks. The lengths of all matchsticks are one. You can find many squares of different sizes in the grid. The size of a square is the length of its side. In the grid shown in the left figure, there are 9 squares of size one, 4 squares of size two, and 1 square of size three.

Each matchstick of the complete grid is identified with a unique number which is assigned from left to right and from top to bottom as shown in the left figure. If you take some matchsticks out from the complete grid, then some squares in the grid will be destroyed, which results in an incomplete 33 grid. The right figure illustrates an incomplete 33 grid after removing three matchsticks numbered with 12, 17 and 23. This removal destroys 5 squares of size one, 3 squares of size two, and 1 square of size three. Consequently, the incomplete grid does not have squares of size three, but still has 4 squares of size one and 1 square of size two.

As input, you are given a (complete or incomplete) nn grid made with no more than 2n(n+1) matchsticks for a natural number 5 <= n . Your task is to compute the minimum number of matchsticks taken
out to destroy all the squares existing in the input nn grid.
Input

The input consists of T test cases. The number of test cases (T ) is given in the first line of the input file.
Each test case consists of two lines: The first line contains a natural number n , not greater than 5, which implies you are given a (complete or incomplete) nn grid as input, and the second line begins with a nonnegative integer k , the number of matchsticks that are missing from the complete nn grid, followed by
k numbers specifying the matchsticks. Note that if k is equal to zero, then the input grid is a complete nn grid; otherwise, the input grid is an incomplete nn grid such that the specified k matchsticks are missing from the complete n*n grid.
Output

Print exactly one line for each test case. The line should contain the minimum number of matchsticks that have to be taken out to destroy all the squares in the input grid.

2
0
3
3 12 17 23
Sample Output

3

分析

  很显然是一道搜索题. BFS?状态记录不下. DFS?边数太多，容易进入不是最优解的搜索树中,而搜索树节点的扩展速度超级快. 那么将这两种做法综合起来，就是迭代加深搜索.
  迭代加深搜索可以限制一个搜索的深度，使得DFS函数的返回类型是bool型，这样一旦搜索失败就可以不用继续搜了. 而普通的DFS不行，不知道具体要搜多少层，求最优解，只能将所有的可能都给搜遍，最坏的结果是将所有边都给枚举一遍.
  为了避免多次搜到同一局面，规定一个搜索顺序：每次只搜当前网格图中边长最小的正方形. 枚举这个正方形上的每一条边，删掉这条边后继续递归下去. 如果边长最小的正方形都不行，那么其它的正方形肯定不行. 可以用“决策包容性”来证明. 这种策略所能到达的后继局面一定包含了搜其它正方形所能到达的后继局面. 破坏了当前局面的最小正方形，后继局面的最小正方形的边长一定不小于当前局面的最小正方形的边长,这样相当于把所有的正方形都给考虑到了，并且每种局面最多只会被考虑一次.
  想到了优先队列的BFS就要想到A*,想到了迭代加深搜索就要想到IDA* ,估价函数设计为：每次找到一个边长最小的正方形，砍掉它的所有边，计数器+1，直到找不到正方形为止. 这个估价函数一定满足估价函数的设计要求，因为对于一个正方形，我们只需要砍掉它的一条边就能破坏它，在估价函数中砍掉了它的所有边，将这个正方形可能影响到的所有正方形都考虑进去了. 所以$H()≤g()$

翻译

填一个16*16的字母数独.

代码

分析

  一个叫DLX的算法可以完美地解决此题. 尝试用搜索解决.
  对于9*9的数独，我们可以每次找能填的数最少的格子去填，但是对于一个16*16的数独，这样做会TLE. 数独的大小变大了，不合法的状态也就会变多对搜索进行剪枝，因为不能最优化剪枝，就进行可行性剪枝.
  考虑不合法的局面和只能填一个特定的数的格子. 先遍历所有的空格，若某个位置A~P都不能填，那么局面不合法，若某个位置只能填一个字母，填上去，不要递归.
  再遍历所有的行，若某个字母在这一行上没有出现，并且不能填在任何一个空位上，不合法. 若某个字母只能填在该行的一个空位上，填上去.
  对于每一列和十六宫格而言都是这样.
  然后判断一下数独是否已经填好了. 如果没有，就找可填字母最少的格子，枚举填哪一个字母. 递归下去. 如果成功了，返回true,否则就要还原以前的局面.为了方便还原，在递归之前将当前局面记录下来，如果失败了，覆盖即可.

总结

  可行性剪枝，考虑不合法的局面和必须的局面. 如果dfs不是用来求最优值，而是用来求是否可行（一种局面），那么成功了就不必还原以前的局面. 一个知识盲点.

算法竞赛进阶指南第二章

发表于 2018-05-04 | 分类于 算法竞赛进阶指南 |

前言

  花了5天时间把这一章搞完了，学习了一些数据结构的巧妙用法.

1.栈与队列

  普通的用法是个人都会. 但有一些比较高端的做法，就是用若干个栈or队列来维护信息.
  1.如果要维护整体信息，这些数据结构分别维护某一段的信息，若将它们“拼”起来，则能得到整体的信息比如说求中位数，开两个对顶堆，每个堆维护一半，正好对应了一整段序列;光标的左右移动，开两个栈，左右移则分别对应栈顶元素的变化.
  2.维护历史信息（或者说是需要维护的信息要保存历史信息）,用一个相同的数据结构来维护，两个数据结构同时操作比如说要维护一个栈，支持求最小值/最大值. 则另开栈维护最小值/最大值，插入一个元素x,即插入min(x,栈顶元素)或max(x,栈顶元素),弹出一个元素则对应栈也要弹出元素. 查询的时候直接输出栈顶元素即可.
  3.用若干个低级数据结构来替代高级数据结构，可以优化时间复杂度例如noip2016蚯蚓. 常见的就是队列替代优先队列.

2.链表

  有一类问题可以用平衡树或堆来做，链表也可以轻松解决. 这类问题通常涉及到多次排序(用平衡树或堆的原因)，多次询问，能够时光倒流,用链表处理的原理是我们可以只需要对所有操作结束后的局面排一次序，利用链表容易执行删除操作的特性来进行时光倒流.
  比如说：每次往一个序列中添数求中位数，我可以先把所有的数都添加进去，排好序，找到当前的中位数后删除，再来找下一个中位数，整个序列一直都是有序的.

3.Hash

  Hash分为两种,一种是字符串Hash,一种是Hash表. Hash表的Hash函数是由变量的特征决定的.你可以选择把特征值加起来mod一个数，也可以先乘后加再mod,由题目而定. 举个例子：一个雪花六个角，每个角都有一个数，要判断两个雪花能不能重合，设计的Hash函数就不能仅仅是将这六个数加起来，因为可能有两个雪花它们的六个数加起来相等，但实际不一样. 因此Hash函数应设计为：六个数的和+六个数的积%mod. mod一般取内存允许的范围内尽量大的质数，因为时间复杂度与n / 表头长度有关
  字符串Hash就是对一段序列，一个区间进行Hash. 我个人认为它有一种线性递推的感觉. 可以将字符串Hash看作是一个特殊的Hash函数，它几乎不可能重复! 一般形式是：$hash[i] = (hash[i - 1] * p + a[i]) \%mod$, 为了方便处理，将$hash$开成unsigned long long类型的，这样就不必mod了. P一般取131或13331.
  利用字符串hash的递推公式，我们可以算出任意一段子串的hash值. 方法是：先求出前缀子串的hash值$f[i] = f[i - 1] * P + (S[i] - “a” + 1)$,任意区间的hash值为$f[r] - f[l - 1] * P^{r - l + 1}$.

4.最小表示法

  一个循环同构字符串，可以用它的最小表示法唯一表示出来. 如果两个循环同构字符串的最小表示法一样，那么这两个字符串是相等的. 如何求最小表示法？
  1.破环成链，先将原字符串复制一份.
  2.初始化i=1,j = 2.
  通过向后扫描的方法，比较$B[i],B[j]$两个循环同构串，这里$B[i]$的意思是从第i个位置的n个字符表示的字符串.
  如果扫描了n个字符后仍然相等，说明S只由1种字符构成，任意$B[i]$都是他的最小表示.
  如果在i+k与j+k处发现不相等:
  若$S[i + k] > S[j + k]$,令$i = i + k + 1$,若此时$i = j$,则令$i = i + 1$.
  另一种情况也这么处理,变动的指针是j.
  若$i > n$,则$B[j]$为最小表示，否则$B[i]$为最小表示.

5.Trie

  当看到有一个数/字符串对多个数/字符串询问时，想到将这多个被询问的数/字符串放到Trie上.
  Trie的一个经典应用是xor,求任意两个数xor的最值、和之类的问题. 为什么强调两个？有的题目会让你选两个点，求这两个点所组成的区间的xor的值，因为xor具有相抵性，把“每个点表示当前点的数值“变成”每个点表示到当前点的前缀的数值“即可.

7.习题

  大多数习题都写了单独的题解，唯有一道题：括号画家.

大致题意

总结

  拉格朗日数乘法这玩意到底有什么用？
求解多元函数的最优值 (最大化或最小化)$f(x_1 , x_2 ,··· , x_n )$，
并满足约束条件 $g_i (x_1 , x_2 ,··· , x_n ) = 0,i = 1,2,··· ,m$
对于每一个约束条件，分配一个参数$λ$,注意啦，这里的约束条件的个数并不是取决于有多少个i,而是有多少个约束函数，即有多少个类似$g_i$的函数.
  定义函数$s(x_1,x_2,…,x_n) = f(x_1,x_2,…,x_n) + λ_1Σg_i(x_1,x_2,…,x_n) + λ_2Σh_i(x_1,x_2,…,x_n),这里的h也是约束函数$,想办法令s最小/最大. 即可让f最小/最大. 求最值，用导数来求. 因为有多个约束函数，所以求偏导，根据偏导=0列方程. 根据列出来的方程选用不同的方法求出$λ$,即可解出$x_1,x_2,…,x_n$. 注意，这里所求的偏导是对于$[1,n]$中的每一个$i$而言的，所以能够解出所有的$x_i$.
  利用拉格朗日乘数法，就能将取值对整体的影响变成取值对每一段的影响.例如本题中，$λ$一旦变小，所有的$v_i$都会变大，答案也会更优，但是要满足能量的限制，故可以二分.
  求偏导可以对λ求！，在这道题中为什么不用呢？因为$s$函数中有一项$λE$,不好解.

          aaaaa
          aacaa
          aaaaa
  其实也很好解决，对于每一行求最小循环节，再对于每一列求最小循环节，分别取最小公倍数嘛.
  又一次成功地WA了，在discuss中看到一组数据：
          aaabcaaa
          abababab
  如果求最小公倍数，最后的答案是16，事实上答案是12. 原因是行上的处理有问题，不一定是每一行选最小循环节最后得到的lcm最小，每一个字符串可能会有多个循环节. 问题就变成从n个位置中的每一个位置选1个可能的数，使得它们的lcm最小.
  似乎变成了一道不可解的爆搜题. 其实搞清楚lcm到底表示什么含义，就能想到正确解法了. 显然lcm要在每一行中都出现，那么开一个桶，将每一行可能的循环节在桶的相应位置+1，最后从小到大枚举，找到第一个值为n的数.
  如何找到串s的所有符合题意的长度？
  只需要对s进行一次kmp特征值计算，如果发现后面有多余的后缀，就把后缀再拿来做一次kmp，不断重复，直到没有后缀。这个过程就可以记录所有符合题意的串了。
   比如aaabcaaa，先求一遍kmp，得到5，再对后缀aaa求一遍，得到1(即6,7,8)。
  又比如aabaabaabaa，先求一遍kmp，得到3(即3,6,9)，再对后缀aa求一遍，得到1，(即10,11)。
  真正的算法不难实现，难的是边界的处理.
  一种更好的做法是hash+kmp.考虑放矩形的过程，肯定是一个接着一个拼上去的.中间肯定会出现分割线. 那么放的小矩形的两边一定是相同的.即左面和右面的右边一列肯定是相同的. 所以将每一列/行hash,求最小循环节即可.
