Description

法矿石炼制法杖的技术。那时人们就认识到，一个法杖的法力取决于使用的矿石。
一般地，矿石越多则法力越强，但物极必反：有时，人们为了获取更强的法力而

bzoj4004 [JLOI2015]装备购买

中文题面

Description
脸哥最近在玩一款神奇的游戏，这个游戏里有 n 件装备，每件装备有 m 个属性，用向量zi(aj ,…..,am) 表示
(1 <= i <= n; 1 <= j <= m)，每个装备需要花费 ci，现在脸哥想买一些装备，但是脸哥很穷，所以总是盘算着
怎样才能花尽量少的钱买尽量多的装备。对于脸哥来说，如果一件装备的属性能用购买的其他装备组合出（也就是
说脸哥可以利用手上的这些装备组合出这件装备的效果），那么这件装备就没有买的必要了。严格的定义是，如果
脸哥买了 zi1,…..zip这 p 件装备，那么对于任意待决定的 zh，不存在 b1,….,bp 使得 b1zi1 + … + bpzi
p = zh（b 是实数），那么脸哥就会买 zh，否则 zh 对脸哥就是无用的了，自然不必购买。举个例子,z1 =(1; 2;
3);z2 =(3; 4; 5);zh =(2; 3; 4)，b1 =1/2，b2 =1/2，就有 b1z1 + b2z2 = zh，那么如果脸哥买了 z1 和 z2
就不会再买 zh 了。脸哥想要在买下最多数量的装备的情况下花最少的钱，你能帮他算一下吗？
Input
第一行两个数 n;m。接下来 n 行，每行 m 个数，其中第 i 行描述装备 i 的各项属性值。接下来一行 n 个数，
其中 ci 表示购买第 i 件装备的花费。
Output
一行两个数，第一个数表示能够购买的最多装备数量，第二个数表示在购买最多数量的装备的情况下的最小花费

Sample Input
3 3

1 2 3

3 4 5

2 3 4

1 1 2
Sample Output
2 2
HINT
如题目中描述，选择装备 1 装备 2，装备 1 装备 3，装备 2 装备 3 均可，但选择装备 1 和装备 2 的花费最小，为 2。对于 100% 的数据, 1 <= n;m <= 500; 0 <= aj <= 1000。

新加数据三组—2016.5.13

代码

分析

  线性基的模板题
  将每个装备的属性看作行向量，进行高斯消元. 最后有多少不全为0的行，就说明基由多少个向量组成. 基的数量即是最多购买装备的数量.
  如何使得购买的费用最少？结合高斯消元的原理和贪心思想. 先将所有的装备按照代价从小到大排序. 一行行地考虑. 因为代价是从小到大排好序的，我们尽量让当前装备成为基中的一元，为了满足高斯消元的要求，令其最高位为t,将其它的t位不为0的行给消掉,实质上就是将当前行放到了第m - t + 1行(上三角矩阵)，但是为了满足贪心策略，我们并不改变矩阵. 这就是高斯消元的妙处. 知道了原理，可以写出许多种不同的高斯消元模板来.

总结

  线性基中，基的向量的数量是一定的. 若想使得第i行成为基中的一元，就要消光其它的在i的最高位上不为0的行,那么i因为其最高位不为0，在以后的过程中，就不会被消掉, 这样就让i占据了基中的一个位置，而基的向量的数量是一定的，故代价大的不会成为基中的一元.

XOR is a kind of bit operator, we define that as follow: for two binary base number A and B, let C=A XOR B, then for each bit of C, we can get its value by check the digit of corresponding position in A and B. And for each digit, 1 XOR 1 = 0, 1 XOR 0 = 1, 0 XOR 1 = 1, 0 XOR 0 = 0. And we simply write this operator as ^, like 3 ^ 1 = 2,4 ^ 3 = 7. XOR is an amazing operator and this is a question about XOR. 翻译

从n个数中选取若干个数，异或起来. 求所得的第k小的值.

代码

分析

  线性基套路题.
  首先找出这n个数中有用的数，即基底. 将每一个数看作m位二进制数，构造一个n行m列的矩阵. 对这个异或矩阵做高斯消元，最后得到的非0行即为矩阵的秩，也就是有用的那些数. 为什么呢？因为对矩阵做初等行变换矩阵的秩是不会发生改变的，当把矩阵变成相抵标准型后，没用的数都被消完了，只剩下基.
  一个定理：主元所在位为1的整数是唯一的,什么意思呢？在相抵标准型中，每一个有用的数都有一个位置(i,j),满足该位置系数非0，第j列的其它位置都是0，第i行的第1~j-1列都是0. 也就是说，在$b_1,b_2,……,b_t$中，有且仅有$b_i$的第$c_i$位为1，其它数的$c_i$位为0.并且j随i变大而递增,也就是说，基底越来越小. 利用这一点可以求得第k小的数.
  根据基的定义，这个数一定可以由基线性表示出来.j是单调递增的，所以选$b_i$一定比不选$b_i$得到的数要大. 将要查询的数k变成转换成二进制数. 若k-1的第i位为0，那么第t-i个基底就要选. (每个基底有选or不选两种可能). 两个二进制数一一对应. 若$k > 2^t$,则无解.
  有一个问题：题目不允许选一个数2次，而异或空间中是可能的. 从$a_i$中选出几个数来异或有可能得到0. 实际上在上述运算中，如果不可能异或出0，那么就直接对k - 1 + 1 = k求.(跳过掉基底异或的结果为0的情况).怎么判断$a_i$能否异或出0呢？看最后的矩阵中是否有一行为0即可. 因为做高斯消元实际上就是每次选不同的两行异或嘛.

总结

  1.xor问题一定要转化为二进制数再去思考.
  2.高斯消元后的主元下标单调性，基与行之间的关系.
  3.xor问题有0的情况.

Source

提示

  1.计数问题通用的方法是什么？
  2.n这么大一般用什么方法处理？

代码

分析

  吼题啊!
  计数题无非就是dp和数学. 这道题显然是dp. 令$f_{i,j}$表示序列的前i个数乘积对M取模的结果为j的方案数. 转移时枚举第i个数是哪一个. 时间复杂$O(nm^2)$.似乎一分都拿不到，在考场上实际能拿10分.
  观察第二档数据范围. n变成了$10^9$,而m变成了100,递推次数这么多，而递推方式不变，很容易想到矩阵快速幂优化递推，这样就有30分了.
  剩下的70分是给正解的.首先需要能想到：在取模意义下，利用原根，乘法可以等效转化为加法.模数是一个质数，有原根，而且这个模数不一般，可以用$2^k + 1$表示出来. 说明可以用NTT?
  将转移过程的乘法运算变成加法运算后确实可以用NTT. 将S中的元素看作一个多项式$f(x)$. 若大小为a的数出现了t次，则$f(x)$的a次项的系数为t. 对于dp数组也是一样,将数的大小看作x的幂次. 因为转化为了原根，所以a为$g^{a’} = a$中的a’.
  n很大，直接递推的话还是会T,根据矩阵快速幂的思想，每次乘的多项式都是同一个，我们可以对这个多项式快速幂.
  NTT和FFT很像，有三点不同：
  1.NTT中的乘法和加法操作要取模，FFT不用.
  2.NTT的旋转因子是原根的(mod - 1) / 步长次幂，或是其相反数.
  3.NTT算完一次后，要将原序列复制的那一部分的答案累加到原序列中.
  下面解释一下为什么有这些不同点：
  第一条是很显然的，这是用NTT算法的目的.
  第二条可以将原根的次数的取值范围类比复平面来看，在FFT中旋转因子也要除以步长，是因为我们把复平面分成了步长份. 而NTT中的原根是有取值范围的，即(0~mod - 1)，类似复平面一样除以步长。
  第三条是只有在模意义下才能这样. $g^t = g^{p + t} (mod  m)$,因为$g^p$与m互质，所以可以把$g^p$给拿走. 从而将序列后半段的答案移到前面去. 为了方便统计答案.

脑残错误

  1.把步长初始化成了1，上界写成了<.
  2.忘了取模.
  3.将序列后半段累加到前半段时，i的枚举顺序弄反了.

总结

  对不是1e9+7的模数要敏感！模意义下的乘法转化为加法操作！

提示

  列出每盏灯初始状态到终止状态的关系式.

代码

分析

  对于第i个开关，可以列出方程（以第1个为例）：
$a_{1,1}x_1  xor  a_{2,1}x_2  xor …… xor  a_{n,1}x_1 = s_1  xor  t_1$
  其中$a_{i,j} = 1$表示操作开关i对开关j有影响. $x_i$表示第i个开关有没有被操作. $s_i$表示第i盏灯的初始状态, $t_i$表示终止状态. 特殊地，我们令$a_{i,i} = 1$
  n元n个方程，挺好的. 题目没有说一定会存在唯一解，可能会出现无解或是多解的情况. 一般情况下，若有m个方程n个未知数，将方程看作系数向量+结果向量，若最大线性无关组的大小为k,则存在n-k个自由元. 若存在方程0 = x(x不为0)，则方程无解. 方程的解的个数和自由元的个数有关 在本题中，自由元可以任意取值0或1，若有cnt个自由元，则答案为$2^{cnt}$. 利用高斯消元求解.
  因为每个系数只有0,1两种取值，故可以用二进制数存储,当前扫到了第i行，若$sta_i = 0$,说明出现了自由元，答案为$2^{n - i + 1}$,若$sta_i = 1$,说明无解. 利用二进制数可以快速判断方程左边与右边的关系，并且能够快速按位异或，不用逐位操作. 高斯消元的具体步骤是找一个二进制表示下最大的数，找到它的最高位k,将其他的第k位不为0的数异或上这个最大的数. 类似普通方程的高斯消元，只不过加减法变成了异或.

总结

  1.遇到的新题型—xor方程组. 利用每一个变量的初始状态，最终状态，以及对它有影响的变量列方程. 方程的左边枚举j,对i是否有影响取决于是否操作j以及如果操作j是否会对i有影响. 方程的右边就是初始状态xor最终状态.
  2.二进制表示状态的方法很值得学习.
  3.高斯消元的一般步骤：找最高位不为0的，让其它最高位变成0，判断无解和自由元的情况.

分析

  很显然是矩阵快速幂优化转移. 但是时间不同，转移矩阵就不唯一，不太好处理. 幸运的是每一个转移操作的长度最大为6，也就是说最多过60s,所有点的转移方式都会和之前某一秒的相同.
  遗憾的是我总是想着如何构造一个转移矩阵. 一开始想的是每一秒的转移矩阵该怎么构造，发现转移并不总是不变的，于是又想怎么构造一个矩阵表示60s的转移，还是不好构造.死活就想不到正解上去.
  正解是构造60个矩阵……分别对应第0~59s的转移. 状态矩阵的行数为1，列数为n*m+1,转移矩阵的行数和列数均为n*m+1, 相当于把n*m个元素放到一个序列上编号，状态矩阵增加了第0列，转移矩阵增加了第0行第0列，为了方便给某个位置增加石头. 令状态矩阵为ans,转移矩阵为t. 一开始$ans_{0,0} = 1$，其它元素为0. $t_{0,0} = 1$. $id(i,j)$表示位置为$(i,j)$的点的序号具体的构造如下：
  1.若在第T秒，第i行第j列的操作是一个数x,那么$t_{id(i,j),id(i,j)} = 1$,并且$t_{0,id(i,j)} = x$. 因为$ans_{0,0} = 1$,所以0这个点为给$(i,j)$x个石头.
  2.若操作是一个方向,以N为例，则$t_{id(i,j),id(i-1,j)} = 1$,其它的情况类似.
  那还有一种操作D怎么处理呢？其实不需要处理.因为下一秒$(i,j)$的点的石头数取决于转移到它的石头数. 如果$t_{id(i,j),id(i,j)} = 0$，那么它的石头是不会被保留的. 而一个点的3种类型的决策，只有决策为数字的点的石头才可能被保留. 所以不处理就相当于处理了.

总结

  矩阵快速幂的思维不能仅仅只局限在构造单一转移矩阵上. 可能会有多个转移矩阵. 一般的应用情形是转移次数多，但是转移方式有一个周期，周期很小.
  构造转移矩阵可以通过加行、列，实现对某一个位置的单独修改!

提示

  写出式子，化简得到一个简单的同余式.

代码

分析

  先写出式子： $L | 8(10^x - 1) / 9$,将9乘过去: $9L | 8(10^x - 1)$.
  接下来怎么化简？不知道. 现在这个式子也无法求解，因为x的取值范围太大. 我们需要做的是将x的取值范围缩小. 需要先知道一个结论：
  若$a^x \% p == 1$,则x的最小取值一定是$φ(p)$的约数.证明如下：
  $φ(p) = kx + r$,$a^{kx + r} \% p == 1$则$a^{kx} * a^r \%p == 1$,而$a^{kx} \% p == 1$,所以$a^r \% p == 1$.若$r > 0$,$r$完全可以变成0来使得$x$更小，所以$r = 0$,成立.
  原式子中有$10^x - 1$，若将$-1$移到同余式的右边，则刚好可以用上这个结论.关键是如何消掉式子中的8.
  还有一个结论：互质的两个数不影响整除性，可以忽视其中一个数,可$9L$与$8$不一定互质，怎么办？ 左右两边同时除以$(9L,8)$即可，令$d = (9L,8)$,左边的$9L / d$与右边的$8 / d$互质，可以将$8 / d$拿走. 原式子变成了$(9L / d) | (10^x - 1)$. 可以利用上面的结论了.
  满心欢喜敲完代码交上去，发现WA了. Why? 因为对$9L / d$取模，模数可能大于int存储范围, 而快速幂可能就是两个大于int存储范围的数相乘，会爆long long,正确的方法是将快速幂中的乘法改成加法.

总结

  1.如何在整除式中越掉一个乘积？提公因数！
  2.一个扩展结论:模m意义下，与m互质的数对于整除性没有影响，也就是说：若$ac = bc (mod  m),(c,m) == 1—->a = c(mod  m)$
  3.模一个数等于1的次数最小值的取值范围.

算法竞赛进阶指南第三章

前言

  历时5天终于干掉了这一章. 码量之巨，技巧之多，无不使人见之而走.

1.bitset的应用

  一般用于可达性统计. 想要统计一张图上某一个点对于其它点的到达情况，仅仅只关心它能否到达，可以用bitset统计. 如果自己用二进制状压在n很大的情况下是写不了的.

2.剪枝方法

  1.每次找分支最少的状态去搜，也就是找“最难符合要求”的状态去搜.
  2.若搜索的多种决策会导向同一种状态，应确定决策的先后顺序. 若一种决策的后继状态包含了另一种决策的后继状态，可以只选择前一种决策.
  3.可行性剪枝.
  4.最优化剪枝.
  5.记忆化，若能记忆化最好写成递推的形式(dp).

3.边权为0,1的BFS

  通过双端队列广搜来使得队列中的状态满足步数单调性. 若决策转移的代价为0，则将扩展状态放到队首，否则放到队尾.

4.两种DFS类型

  1.可行性dfs,返回类型为bool型.
  2.最优性dfs,要求出一个最优解.
  这两种dfs的区别是剪枝和回溯的不同. 对于可行性dfs,没有最优性剪枝. 若失败了才回溯（还原历史状态）. 对于最优性dfs,两种剪枝都可以有，并且不论成功与否，都要还原历史状态.

5.迭代加深搜索与DFS的关系

  1.迭代加深搜索实质上就是可行性dfs.
  2.与迭代加深搜索功能相对应的就是最优性dfs.

6.搜索方法的选择

  1.优先考虑最优化搜索算法(A*,IDA*).
  2.若期望步数比较小，状态能存下.优先选择BFS类搜索.
  3.若期望步数较大，决策单一，优先选择DFS类搜索.
  4.若搜索树有的节点深度非常大（搜不到底，或者要搜特别多次），选择迭代加深类搜索.
  5.若能优化(BFS优化成优先队列BFS,优先队列BFS优化成A*,迭代加深搜索优化成IDA*),选择优化算法.
