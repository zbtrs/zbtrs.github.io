poj1390 Blocks

发表于 2018-06-02 | 分类于 算法竞赛进阶指南 |

英文题面

Description

Some of you may have played a game called ‘Blocks’. There are n blocks in a row, each box has a color. Here is an example: Gold, Silver, Silver, Silver, Silver, Bronze, Bronze, Bronze, Gold.
The corresponding picture will be as shown below:

Figure 1

If some adjacent boxes are all of the same color, and both the box to its left(if it exists) and its right(if it exists) are of some other color, we call it a ‘box segment’. There are 4 box segments. That is: gold, silver, bronze, gold. There are 1, 4, 3, 1 box(es) in the segments respectively.

Every time, you can click a box, then the whole segment containing that box DISAPPEARS. If that segment is composed of k boxes, you will get kk points. for example, if you click on a silver box, the silver segment disappears, you got 44=16 points.

Now let’s look at the picture below:

Figure 2

The first one is OPTIMAL.

Find the highest score you can get, given an initial state of this game.
Input

The first line contains the number of tests t(1<=t<=15). Each case contains two lines. The first line contains an integer n(1<=n<=200), the number of boxes. The second line contains n integers, representing the colors of each box. The integers are in the range 1~n.
Output

For each test case, print the case number and the highest possible score.
Sample Input

2
9
1 2 2 2 2 3 3 3 1
1
1
Sample Output

Case 1: 29
Case 2: 1
Source

Liu Rujia@POJ

翻译

  n个方块，每种方块都有一种颜色，每次可以选择消掉一个方块，与方块相连的所有同色方块都会被消掉，代价为消掉的方块数的平方，求最大代价.

提示

  消数，求最值，区间dp.

代码

分析

  这道题非常容易想到区间dp. 按照区间dp的常规套路来，令$f_{i,j}$表示区间[i,j]的最大代价. 有些情况是转移不了的. 比如多个不相邻相同颜色的色块合并，转移只能枚举k,最多只能处理3个相同颜色的色块的合并.
  区间外会影响区间内，端点色块可能会和区间外的相同色块合并，二维状态只能维护区间内部的情况，如果要处理区间外对区间内的影响，就必须再多加一维. 令$f_{i,j,k}$表示区间[i,j],右端点j与其右边的k个相同色块合并的最大代价. 转移的时候考虑右端点色块，要么只和区间外的色块合并，要么和区间内的色块合并.

总结

  区间dp的常规状态表示只能处理区间内部的情况，对于区间外部的情况需要多加一维. 只需要考虑左端点or右端点向外扩张的情况即可.

poj1187 陨石的秘密

发表于 2018-06-01 | 分类于 算法竞赛进阶指南 |

中文题面

???

提示

  基准点计数.

代码

分析

  吼题啊！学到了许多新姿势.
  首先很容易想到状态$f_{i,j,k,l}$表示选了i个大括号，j个中括号，k个小括号，深度为l的方案数. 表达式可以看作是许多独立的部分组成的，深度即为这些部分的深度的最大值.
  每次考虑当前括号，是放在前面的一个部分中还是新创一个部分？一个比较麻烦的问题是深度不好维护. 因为不知道最大值是哪一个部分. 难道要记录每一部分的深度？这样就成了状压dp了，复杂度是不能接受的,而且还不知道具体有多少部分.
  换一种计数方式,不在对括号计数，而是对部分计数. 每次选取一个整体计数，采取基准点计数法. 以第i对括号所在的部分为基准点. 这样还是没有解决维护深度的问题,因为还要枚举前面其它部分的深度是多少，复杂度是指数级的. 那么改变一下状态:令l表示深度≤l. 这样枚举当前部分的深度d,因为第i个括号已经被确定了，所以当前部分的方案数就是$f_{a,b,c,d - 1}$, a,b,c需要枚举. 其它部分的方案数就是$f_{l1 - a,l2 - b,l3 - c,d}$,前3维可能要-1，取决于第i个括号是哪种类型的括号.
  最后的答案就是$f_{l1,l2,l3,d} - f_{l1,l2,l3,d - 1}$，相当于将具体值变成了前缀.

小结

  计数对象的选择：对整体，可以用基准点计数法，对个体，看这个个体的决策.
  计数=类型的方案数可以变成计数≤类型的方案数. 范围相减得到具体答案.

一开始都是0的n个数，有q种操作，每种操作都是将一个区间内的数全部加上x,对于1到n的每个数x,能否从操作集合中选出一些操作，使得操作后整个序列的最大值为x?

代码

分析

  一步非常重要的转化：若一个数能够拼出，那么就可以选出操作集合中的子集使得拼出这个数为最大值, 因为我们可以只选包含这个数的区间，这个数是选定的区间的交集，一定是最大的.
  那么我们只需要求出有哪些数能被拼出即可. 一个做法是利用bitset,先进行所有操作，向上合并的时候利用bitset更新那些数出现过.

poj1722 SUBTRACT

发表于 2018-05-31 | 分类于 算法竞赛进阶指南 |

英文题面

Description

We are given a sequence of N positive integers a = [a1, a2, …, aN] on which we can perform contraction operations.
One contraction operation consists of replacing adjacent elements ai and ai+1 by their difference ai-ai+1. For a sequence of N integers, we can perform exactly N-1 different contraction operations, each of which results in a new (N-1) element sequence.

Precisely, let con(a,i) denote the (N-1) element sequence obtained from [a1, a2, …, aN] by replacing the elements ai and ai+1 by a single integer ai-ai+1 :

con(a,i) = [a1, …, ai-1, ai-ai+1, ai+2, …, aN]
Applying N-1 contractions to any given sequence of N integers obviously yields a single integer.
For example, applying contractions 2, 3, 2 and 1 in that order to the sequence [12,10,4,3,5] yields 4, since :
con([12,10,4,3,5],2) = [12,6,3,5]

con([12,6,3,5] ,3) = [12,6,-2]

con([12,6,-2] ,2) = [12,8]

con([12,8] ,1) = [4]

Given a sequence a1, a2, …, aN and a target number T, the problem is to find a sequence of N-1 contractions that applied to the original sequence yields T.
Input

The first line of the input contains two integers separated by blank character : the integer N, 1 <= N <= 100, the number of integers in the original sequence, and the target integer T, -10000 <= T <= 10000.
The following N lines contain the starting sequence : for each i, 1 <= i <= N, the (i+1)st line of the input file contains integer ai, 1 <= ai <= 100.
Output

Output should contain N-1 lines, describing a sequence of contractions that transforms the original sequence into a single element sequence containing only number T. The ith line of the output file should contain a single integer denoting the ith contraction to be applied.
You can assume that at least one such sequence of contractions will exist for a given input.
Sample Input

5 4
12
10
4
3
5
Sample Output

2
3
2
1
Source

CEOI 1998

提示

  不要被将数合并给迷惑了，找到操作的本质.

代码

分析

  我一开始就被合并操作给迷惑了，加上n的范围是100，让我想到了区间dp. 区间dp其实也可以，但是数不好表示,除非再开一维.
  这道题是很容易想到整体的算法框架的—先求可行性dp,然后通过dp的结果逆推得到答案.关键就是如何记录状态使得状态能够“吻合”题目中的合并操作以及能推出解来？找到操作的本质：实际上所有数都是不变的，只是在一些位置加上括号和减号. 对样例进行操作，不把数合并，而是拆分成减法的形式，可以发现最后所有数的前面要么是加号要么是减号. 也就是说：我们要给n-1个数的前面添加符号，使得它们运算的结果等于T. 求出答案后再转化为操作.
  那么可行性dp就出来了：令$f_{i,j}$表示前i个数是否能表示出j. 两种转移：$f_{i - 1,j - a_i}$或$f_{i - 1,j + a_i}$,可以根据前继状态的可行性推出每个数前的系数.先从前往后处理前面符号是加号的数，然后不断操作1即可. 因为前面是加号的数必须通过偶数次减法才行，从前往后处理符号是加号的数就把这些数合并成了若干连续的段，最后不断地操作1就能使得它们前面的符号是加号了.

坑点

  第二个数前面的符号一定是减号，要特殊处理边界！

题目大意

  m个点，每个点的权值都在[0,2^n - 1]内，两个点x,y有边当且仅当x and y = 0,求图中有多少个连通块.

提示

  联想一下在普通的无向图中找连通块的方法.

代码

分析

  读错题意就像吃了翔一样难受. 一开始看成了有多少条边，只能想到$O(m^2)$的算法.
  对于一个数，如何找到所有与它and起来等于0的数？首先确定查找的方向：找比它小的与它and起来等于0的最大的数. 每次从这个数中消掉一个0,若出现了给定的数，则找比这个数小且and起来等于0的最大的数，并且继续消1. 加上记忆化，就能在$O(2^n)$的复杂度内搜完.

总结

  1.有限集合的双向关系(由A可以推导B,由B也可以推导A),可以只考虑一个方向，例如本题中只考虑比x小的数.
  2.先找最大元，然后消1枚举子集法.

poj1934 Trip

发表于 2018-05-30 | 分类于 算法竞赛进阶指南 |

英文题面

Description

Alice and Bob want to go on holiday. Each of them has planned a route, which is a list of cities to be visited in a given order. A route may contain a city more than once.
As they want to travel together, they have to agree on a common route. None wants to change the order of the cities on his or her route or add other cities. Therefore they have no choice but to remove some cities from the route. Of course the common route should be as long as possible.
There are exactly 26 cities in the region. Therefore they are encoded on the lists as lower case letters from ‘a’ to ‘z’.
Input

The input consists of two lines; the first line is Alice’s list, the second line is Bob’s list.
Each list consists of 1 to 80 lower case letters with no spaces inbetween.
Output

The output should contain all routes that meet the conditions described above, but no route should be listed more than once. Each route should be printed on a separate line. There is at least one such non-empty route, but never more than 1000 different ones. Output them in ascending order.
Sample Input

abcabcaa
acbacba
Sample Output

ababa
abaca
abcba
acaba
acaca
acbaa
acbca
Source

CEOI 2003

翻译

  按字典序从小到大打印所有的LCS.

提示

  打印所有的，只能用什么方法？

代码

分析

  先用dp求出LCS是多少，再来考虑如何输出答案.
  因为题目要输出所有解，不是输出个数，因此不能依靠dp记录路径. 只能通过搜索的方法. 从后往前搜. 令当前在第一个字符串内扫的位置为pos1,在第二个字符串内扫的位置为pos2,若s1[pos1] = s2[pos2],则在(pos1 - 1,pos2 - 1)位置继续搜，并记录s1[pos]这个字符. 否则选择就有两种了，要么是(pos1 - 1,pos2)或(pos1,pos2 - 1). 如果构造两个全是a的字符串，这种算法就超时了，需要对第二种决策进行优化.
  容易发现，累计答案当且仅当s1[pos1] = s2[pos2]时，若当前两个字符不相同，则跳到前面相同的字母即可,同时还要保证是从最优的位置转移过来的. 用两个数组l1[i][j]记录s1[i]及其以前的字符中最晚出现字符j的位置,l2类似，每次枚举字符i,往前面最优的决策中跳.
  为了方便处理，利用string记录路径，set排序去重. 注意：最后字符串一定要翻转！！！

小结一下

  题目要输出所有解时，只能用搜索了，没别的方法.
  考虑清楚什么情况下才会对答案有影响. 若指针会进行多次无意义的跳动，则预处理出从每个位置能跳到的下一个有效位置是哪一个. 当然，也可以用倍增法来优化.

Codeforces 909 F. AND-permutations

发表于 2018-05-30 | 分类于 计划 |

题目大意

构造两个长度均为n的排列，第一个排列要求$p_i != i$并且$p_i and i = 0$,第二个排列要求$p_i != i$并且$p_i and i != 0$.

提示

  找基准点

代码

分析

  有一个结论：对于每一个正整数x,总能找到一个比他小的并且每一位都与之相反的一个数. 如果把数位取反这种操作看作一种映射，那么变换前和变换后的数是一一映射的. 若两个数每一位都相反，那么这两个数and起来就等于0.
  怎么才能找到比x小并且每一位都与之相反的数呢？ 利用基准点—x最高位的1. 假设x最高位的1是第k位，若$x = 2^k + i$,那么对应的数就是$2^k - i - 1$. 其实就相当于将$2^k$当作对称轴.那么对于第一问，若n是偶数，动态更新对称轴，并更新答案. 注意：不要去枚举k然后去枚举i,正确的做法是枚举x,找最高位k并更新对应的数，因为原则：优先处理对答案有直接影响的量. 若n是奇数,因为不能两两配对，所以构造不出来.
  对于第二问，只要求一位相同即可. 还是找基准点—最高位k. 将所有最高位为k的分成一组(一定是连续的)，将这些数同时右移一位(类似循环排列)即可. 需要特殊处理的是1,最高位为第1位的数只有1，而1右移一位还是1，不合法，可以手动推出：在n < 6时，构造不出合法的解，当n = 6时，可以构造，n = 7时，前7个数正好可以用完(放在前7个位置)，那么剩下的位置就可以用之前的构造算法了. 需要注意的是若n是2的整数次幂是构造不出来的，因为最后一定会存在k,使得最高位为k的数只有1个，这个数没法挪动.

总结

  二进制数问题以最高位为基准点.
  两个数每一位都不相同的结论.
  优先处理对答案有直接影响的量.
