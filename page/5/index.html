poj1737 Connected Graph

发表于 2018-05-27 | 分类于 算法竞赛进阶指南

题目大意

求n个节点的无向连通图有多少个，节点有标号.

提示

  考虑以什么为基准进行划分来计数.

代码

分析

  连通图不容易计数，考虑计算不连通图的数量，再用总的图的数量减去不连通图的数量即为答案.
  以标号为1的节点所在的连通块为基准进行划分. 令$f_i$表示i个节点的无向连通图的个数，枚举选j个点构成第一个连通子图，那么方案数就是$f_j * C(i - 1,j - 1) * 2^{(i - j)(i - j - 1) / 2}$,即从剩下的i-1个点中选出j-1个点构成标号为1的节点所在的连通块. 剩下的点与第一个连通块都是不连通的，它们之间的连通情况任意. 总的方案数是$2^{i*(i - 1) / 2}$. $f_i = 2^{i*(i - 1) / 2} - Σf_j * C(i - 1,j - 1) * 2^{(i - j)(i - j - 1) / 2}$.
  比较毒瘤的是没有取模操作，需要手写高精度.

poj1037 A decorative fence

发表于 2018-05-27 | 分类于 算法竞赛进阶指南

题目大意

求字典序为第C小的波动排列.

提示

  想一想二分法求第k小的数的原理.

代码

分析

  诸如求第k小的具体是哪一个的问题，都采用步步逼近的方法. 不断调整，缩小范围，直到当前答案就是第k小的. 如果答案具有单调性，可以像二分一样从两边向中间缩小范围，否则就要dp求出各种情况的方案数，从单方面缩小范围，就像倍增一样.
  现在的问题就是如何dp处理出各种情况的方案数？这实质上是一个排列计数问题. 因为排列有一个非常好的性质：1到n中所有数仅出现一次，那么我们可以考虑第i个数放在哪个位置而不是第i个位置放哪个数. 后者一般要用状压dp记录当前已经用了哪些数.
   如果考虑第i个数放在哪个位置，是不好处理的，因为如果要这样考虑的话，就要考虑把第i个数插入到某个位置会对答案整体造成什么影响，显然这是不能直接计算的，应该采用递推的形式，即第二种考虑方式.
  第二种方法采用递推的形式，从左往右,依次确定第i个位置放哪一个数. 比较难解决的是数会重复这一问题. 若用状压的话，这一题的N比较大，又是多组数据，有TLE的危险. 正确的方法是改变记录的状态.令$f_{i,j,k}$表示当前已经处理到了第i个位置，第i个位置放的木板是从小到大第j个没被占用的木板，并且状态为k(0代表低位，1代表高位)的方案数. 这样考虑的就是数的相对大小，而且考虑的还是没有占用的数，就不会出现有两位出现相同的数这种情况了. 转移时要特别注意j的取值范围，取决于k和j表示的意义！ $f_{i,j,0} = Σf_{i - 1,p,1}(j≤p≤i-1)$,$f_{i,j,1} = Σf_{i-1,p,0}(1≤p≤j - 1)$.这里要保证k的合法性(比周围小or大)，而j表示的是未被占用的第j小的木板，举几个例子即可得出范围.

总结

  1.求第k小的具体值用逐步逼近法.
  2.排列dp计数问题的2种考虑方式，3种实现方法. 记住什么时候求方案数(一位一位递推)，什么时候求对整体的影响(插到某一个位置).

Codeforces 559 C. Gerald and Giant Chess

发表于 2018-05-27 | 分类于 算法竞赛进阶指南

题目大意

有一个h*w的棋盘，棋盘上有n个格子是黑色的，其它都是白色的. 求从左上角走到右下角不经过黑色格子的路径数，每次只能往下或往右走. n较小.

提示

  n比较小，以什么为基准进行计数？

代码

分析

  考虑到w,h巨大，而n很小. 所以对黑点进行计数. 只需要求出从起点到终点的所有路径减去经过黑点的路径即可.
  将终点当作第n+1个黑点. 令$f_i$表示不经过其它黑点到达第i个黑点的方案数. 那么$f_i = C(x_i - 1 + y_i - 1,x_i - 1) - Σf_j * C(x_i - x_j + y_i - y_j,x_i - x_j)$. 为什么满足这个递推式呢？ 递归地考虑状态表示的意义！f表示的是不经过其它黑点并到达当前点的方案数. 要统计不经过其它黑点并到达第i个黑点的方案数就要用总方案数减掉经过其它可能经过的黑点的方案数. 这时$f_j$的含义是从起点走到j,j是第一个经过的黑点的方案数. 这其实就相当于一个基准点. 经过了j则说明该路径一定经过黑点，至于后面怎么走随意. 因为经过的第一个黑点不同，所以对应的路径不同,而我们将所有可能经过的第一个黑点的情况都考虑了，所以路径数被不重不漏地统计了. 很奇妙，和金字塔那道题类似. 都是多种同类型决策，会造成重复计数时，考虑所有可能的“第一个”.

Uva1602 Lattice Animals

发表于 2018-05-27 | 分类于 计划

题目大意

在一个n*m的网格图上放有k个点的连通块，求方案数. 若一个连通块通过任意旋转、翻折、平移可以与另一个连通块重合，则这两种方案算一种.

提示

  是先考虑构造出这些连通块还是先在网格图上放再来考虑是否合法？
  对于一个连通块，用什么保存？
  怎么构造出连通块？

代码

分析

  先考虑怎么存连通块. 用结构体来存，找到连通块的基准点，将其挪到原点，然后存下所有点的坐标即可. 基准点是横坐标和纵坐标最小的点. 本质上就是存所有点到基准点的相对位置.
  接下来就要考虑求解顺序问题了. 是在网格图上一个一个构造连通块还是构造出所有合法的连通块后再往网格图上放呢？ 显然是先构造合法的连通块. 我们可以预处理出有k个点的不重复的连通块. 然后在这些连通块旁加一个点，就能变成k+1个点的连通块.
  最主要的问题是判重. 对于平移操作，因为以基准点作为参考，所以不需要考虑. 对于旋转操作，因为转四次就又回来了，所以处理出当前连通块顺时针依次转4次的情况，翻折后再转4次，即可处理所有的情况.
  现在的问题是如何判断两个连通块是否是一样的(固定了的). 也就是比较若干个二元组是否一样. 常规做法是将两个连通块内的二元组按照横坐标从小到大，纵坐标从小到大的顺序排序. 然后一一比较. 因为涉及到多次排序，可以用set来维护. 对于连通块内部的点可以用set维护，对于每个连通块构成的集合也可以用set来维护.

教训

  判断两个集合是否一样可以用set使集合内的元素有序化，再来比较.
  对一个对象本身进行操作一定要检查传入的是否是引用，不然改了白改了…

Codeforces 311 B. Cats Transport

发表于 2018-05-26 | 分类于 算法竞赛进阶指南

题目大意

   p个人,m只猫,n座山，告诉每座山距离前一座山的距离，和每只猫在哪座山上，等待时间. 所有人一开始都在一号山，每单位时间挪动距离为1，求如何安排这些人的出发时间使得所有的猫的等待时间总和最少？

代码

分析

  首先要知道猫在什么条件下才能被接到. 对于每只猫，设$A_i = T_i - ΣD_j(1≤j≤H_i)$.只有当人的出发时间在$A_i$之后时，这只猫才能被接到，等待时间为$t - A_i$,t是出发时间.
  将所有猫按照A从小到大排好序后，就能够分配人去接猫了. 每个人接的猫对应的A一定是连续的一段. 因为要考虑有多少个人，接了多少只猫，因此状态定义为：$f_{i,j}表示前i个人，带走第k+1~j$的费用，枚举k,即可得到状态转移方程：$f_{i,j} = min\{f_{i - 1,k} + A_j *(j - k) - (S_j - S_k)\}$,其中S是A排好序后的前缀和. 斜率优化就能过.

坑点

  一定要注意初始化！

小G是一个出色的诗人，经常作诗自娱自乐。但是，他一直被一件事情所困扰，那就是诗的排版问题。

一首诗包含了若干个句子，对于一些连续的短句，可以将它们用空格隔开并放在一行中，注意一行中可以放的句子数目是没有限制的。小G给每首诗定义了一个行标准长度（行的长度为一行中符号的总个数），他希望排版后每行的长度都和行标准长度相差不远。显然排版时，不应改变原有的句子顺序，并且小G不允许把一个句子分在两行或者更多的行内。在满足上面两个条件的情况下，小G对于排版中的每行定义了一个不协调度, 为这行的实际长度与行标准长度差值绝对值的P次方，而一个排版的不协调度为所有行不协调度的总和。

小G最近又作了几首诗，现在请你对这首诗进行排版，使得排版后的诗尽量协调（即不协调度尽量小），并把排版的结果告诉他。

提示

  实现上怎么判断答案是否会大于1e18?

代码

分析

  这道题的本质就是序列划分问题.考虑代价函数$val(x,y) = |sum_i - sum_j + i - j - 1 - L|^p$,因为p很大，所以肯定是不能展开的. 如果要优化dp，就必须要在状态上或代价函数上下功夫. 状态已经最简了，只有一维。那么考虑这个代价函数有什么特点. 很容易想到它满足四边形不等式(因为其它的优化方法都已经无效了). 分情况讨论p的奇偶性，x的大小，去掉绝对值符号，利用导数很容易证明. 那么套用四边形不等式优化模板即可。

bzoj2726 [SDOI2012]任务安排(斜率优化模板的选择)

发表于 2018-05-26 | 分类于 算法竞赛进阶指南

中文题面

Description
机器上有N个需要处理的任务，它们构成了一个序列。这些任务被标号为1到N，因此序列的排列为1,2,3…N。这N个任务被分成若干批，每批包含相邻的若干任务。从时刻0开始，这些任务被分批加工，第i个任务单独完成所需的时间是Ti。在每批任务开始前，机器需要启动时间S，而完成这批任务所需的时间是各个任务需要时间的总和。注意，同一批任务将在同一时刻完成。每个任务的费用是它的完成时刻乘以一个费用系数Fi。请确定一个分组方案，使得总费用最小。
Input
第一行两个整数，N,S。
接下来N行每行两个整数，Ti,Fi。
Output
一个整数，为所求的答案。
Sample Input
5 1

1 3

3 2

4 3

2 3

1 4

Sample Output
153

代码

分析

  很显然这是一道dp题. 要知道当前划分到了哪一位，已经划分了几段，状态中必须记录两个信息：$f_{i,j}$表示当前划分到了第i位，已经划分了j段的总费用.
这个dp显然是过不了的，光是状态数就已经爆了，还没有算上转移的复杂度.
  其实划分问题都只用保留第一维. 但是要计算S的影响，必须要知道当前划分了多少段，只知道划分到了哪里不能推出来. 于是就要用上一种叫做费用提前计算的方法. 若对于当前枚举到的状态，不知道有多少前继状态影响到它，但知道它影响哪些后继状态，可以将它对后继状态的影响记入状态中. 那么$f_i = min\{f_j + sumT_i * (sumC_i - sumC_j) + S * (sumC_N - sumC_j)\}$,最后一部分即为状态i对后继状态的影响.
  对dp转移进行优化. 转移方程内的多项式包含i和j的乘积项，因此不能用单调队列优化，”只能”用斜率优化. 关于斜率优化我有两种不同的模板，一种是博客园上的，一种是书上介绍的，下面说一说在什么情况下用哪种模板比较好：
  1.如果是普通情况，当然是博客园上的模板更好，因为斜率可能会出现不存在的情况. 使用第一种模板时要记住：斜率递增，符号相反，斜率递减，符号相同.
  2.如果斜率不一定递增，但是横坐标是递增的，那么用第二种模板. 保存整个凸包，不进行队首的操作，每次二分查找合适的斜率位置.
  3.其它情况都用第二种模板 + cdq分治. 先递归处理左边的部分，然后维护水平序凸包，对右边的斜率也进行排序. 两个单调指针扫描转移即可.
  对于本题而言，属于第二种情况.

Codeforces 876 E. National Property

题目大意

规定大写字母的字典序小于小写字母. 现在将所有的字母都表示成数字,可以将若干个字母变成大写字母，替换所有单词中的该字母. 问能不能将这些单词的字典序变成单调上升的? 如果能，输出变哪些字母.

提示

  一个字母对应两种状态，能想到什么？

代码

分析

  打比赛时非常遗憾的一道题. 一直想的是贪心or dp,就是没往2-sat上面想. 其实2-sat也很容易想到. 因为每个字母只有两种状态. 考虑有哪些限制.
  1.若第i行与第i+1行相邻的两个字母x,y, x < y，那么这两个字母是合法的，x不变y就不能变，y变了x就必须变，所以连边x —> y, y’ —> x’.
  2.若x > y,那么x必须变，y不能变. 连边x —> x’,y’ —> y.
  剩下的就是2-sat模型了. 最后要输出方案，拓扑排序一下即可.

教训

  1.字母不要一列一列地存，要一行一行地存，不然可能会出现两个字母在行上不连续出现的情况.
  2.要检验一种不合法的特殊情况：第i个和第j个单词的前len位一样(len是两个单词长度的较小值)，但是它们的长度不一样. 这样无论怎么变都不可能使单词的字典序有序.

从序列a中选取区间，要求这个区间内的所有数的或值>这个区间内所有的数. 求有多少个区间可以满足答案？

提示

  统计区间个数题一般有哪几种方法？

代码

分析

  统计区间个数题的做法非常套路. 要么是枚举其中一个端点，再来快速跳另一个端点,或者是以每一个点为基准点，向两边扩展合法的区间.
  先尝试第一种方法. 枚举左端点，右端点怎么样才能快速移动呢？可以利用or操作的单调性？虽然or的结果是单调递增的，但是最大值也是单调递增的，两者的大小不好确定. 那么只好采用第二种方法。

poj3017 Cut the Sequence

英文题面

Description

Given an integer sequence { an } of length N, you are to cut the sequence into several parts every one of which is a consecutive subsequence of the original sequence. Every part must satisfy that the sum of the integers in the part is not greater than a given integer M. You are to find a cutting that minimizes the sum of the maximum integer of each part.

The first line of input contains two integer N (0 < N ≤ 100 000), M. The following line contains N integers describes the integer sequence. Every integer in the sequence is between 0 and 1 000 000 inclusively.

Output

Output one integer which is the minimum sum of the maximum integer of each part. If no such cuttings exist, output −1.

Sample Input

8 17

12
Hint

Use 64-bit integer type to hold M.

Source

POJ Monthly—2006.09.29, zhucheng

翻译

  给定一个长度为N的序列A,要求把该序列分成若干段，在满足每段的和不超过M的前提下，让每段中所有数的最大值之和最小。计算这个值.

提示

  连续序列划分问题，用dp解，先写出状态转移方程，再来想用什么方法去优化。

代码

分析

  这是一道非常特殊的用单调队列优化的dp题. 先写出状态转移方程：
$f_i = min\{f_j + max\{a_k\}(j + 1≤k≤i)\}(Σa_k ≤ M(j+1≤k≤i))$
  枚举i和j的时间复杂度是$O(n^2)$的，需要优化掉其中一维的枚举. 一般是优化j的枚举. 观察状态转移方程，可以发现随着i的变大，j的取值范围的上界和下界都是单调变化的. 那么可以利用单调队列来优化.
  可以用单调队列优化的dp状态转移方程后面一般接一个多项式，这个多项式的每一项只与i或j有关.然而这个状态转移方程后面根本就算不上多项式. 那么需要特殊的方法去处理：
  1.找到最优解出现满足的情况,弄清楚这些情况是and关系还是or关系.
  2.单调队列维护满足上面条件的位置.
  3.用某种数据结构得到最优解.
  也就是说，用单调队列优化这类dp问题的转移和优化“后面接多项式”dp的转移是不一样的. 前者是维护限制条件，也就是可能是最优决策的位置，后者是直接维护最优决策.
  第一步是关键. 一般的方法是贪心地考虑有哪些情况可能成为最优解. 对于本题而言,有两种需要考虑的决策：
  1.$A_j$是[j,i]中的最大值，那么从$f_j$转移而来.
  2.从j+1开始，到i的和正好≤M的最小的j.
  为什么考虑上面两种转移呢？对于第一种转移可以这样贪心考虑：将序列分成两段s1和s2. 从$f_j$转移而来，说明将$A_j$放在了s1,$A_{j + 1}$到$A_i$都放在了s2. 若s1段的最大值不是$A_j$那么相当于s1的答案不变，而s2的答案变小了.若s1段的最大值是$A_j$,这种情况只会发生在边界. 因为按照这种转移方式，s1一定能被分成更小的段，使得有$A_k≥A_j$. 因为边界处的特殊性，所以要考虑第二种决策. 两种决策关系是or，也就是说只要满足其中的一条，就有可能成为最优决策.
  当YY出两种决策后，就要利用决策选择性来证明. 利用反证法，假设决策j两个条件都不满足, 因为$f_j$的单调性和max的单调性，$f_{j - 1} + max\{A_k\} (j≤k≤i) ≤ f_j + max\{A_k\}(j + 1≤k≤i)$. 那么我们完全可以选$f_{j - 1}$而不是$f_j$. 这个不等式是建立在两个条件都不满足的基础之上的，所以这两种条件至少要满足一个.
  然后将所有可能的情况看作图上的点，对于and关系的情况，在它们之间连上一条边，那么图中就会有若干个连通分量. 分别对这些连通分量转移.
  比如这两个限制条件的关系是or,那么就有两个连通分量，分开来考虑. 对于第二个决策，用一个单调指针维护从哪里转移即可. 对于第一个决策. 维护一个j递增，$A_j$递减的队列，用单调队列正好可以维护. 而$A_j$是j到i中最大的一个数，那么单调队列中的第i项的f值和第i+1项的a值即为一种可能的转移. 注意下标运算.
  因为用单调队列维护的是可能成为最优决策的位置，要用一个数据结构维护最优值. 这个数据结构要支持删除最优值、添加一个值、取出最优值这些操作. multiset是首选，当然两个优先队列也是可以的.
