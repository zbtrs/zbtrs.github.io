poj1160 Post Office

发表于 2018-06-06 | 分类于 算法竞赛进阶指南 |

英文题面

Description

There is a straight highway with villages alongside the highway. The highway is represented as an integer axis, and the position of each village is identified with a single integer coordinate. There are no two villages in the same position. The distance between two positions is the absolute value of the difference of their integer coordinates.

Post offices will be built in some, but not necessarily all of the villages. A village and the post office in it have the same position. For building the post offices, their positions should be chosen so that the total sum of all distances between each village and its nearest post office is minimum.

You are to write a program which, given the positions of the villages and the number of post offices, computes the least possible sum of all distances between each village and its nearest post office.
Input

Your program is to read from standard input. The first line contains two integers: the first is the number of villages V, 1 <= V <= 300, and the second is the number of post offices P, 1 <= P <= 30, P <= V. The second line contains V integers in increasing order. These V integers are the positions of the villages. For each position X it holds that 1 <= X <= 10000.
Output

The first line contains one integer S, which is the sum of all distances between each village and its nearest post office.
Sample Input

10 5
1 2 3 6 7 9 11 22 44 50
Sample Output

9
Source

IOI 2000

翻译

  n个村庄，要求选k个村庄设置邮局，令每一个村庄的距离值为这个村庄到离它最近的邮局的距离，求最小距离值的和.

代码

分析

  我的dp方法和网上的大部分方法都不一样. 令$f_{i,j}$表示已经设置了i个邮局，考虑了前j个村庄，并且第j个村庄必须设邮局的答案. 因为最后一个村庄在实际答案中可能不设邮局，所以加一个虚拟村庄n + 1,位置为inf. 最后的答案为$f_{k + 1,n + 1}$.可以先$O(n^3)$预处理出转移代价$w_{i,j}$.
  这个dp方程不能进行优化，虽然可以通过原题，但是比较慢. 换一种dp方法：不再强制令第j个村庄设置邮局，而是在第k个村庄和第j个村庄中间设置邮局，利用中位数原理. 其中k是转移到j的决策位置. 如果在比赛中，状态不能简化，但是需要优化dp时，考虑优化状态转移方程. 优化状态转移方程考虑5种方法，要么是只增不减求最值的决策集合，利用一个变量维护即可，要么可以用数据结构优化，要么转移多项式只与一项有关，可以用单调队列优化.要么与两项有关，斜率优化. 都不行的话，就考虑四边形不等式了. 四边形不等式的证明比较难，大胆猜想，对拍验证即可.
$f_{i,j}$转移的区间范围在$f_{i - 1,j}$到$f_{i,j + 1}$的区间范围内，倒序枚举j即可.

Codeforces 963 D. Frequency of String

发表于 2018-06-05 | 分类于 计划 |

题目大意

  给定一个字符串s,n次询问，给定k和m,求出一个字符串，使得这个字符串是s的子串，并且m作为它的子串至少出现k次.

代码

分析

  如果只有一次询问要怎么做？KMP即可，找到m在s中所有出现的位置，第i+k个位置的坐标-第i个位置的坐标+字符串的长度的最小值即为答案.
  多次询问要怎么处理呢？n次KMP？换个角度，用s去匹配m. 因为是多对一，所以建AC自动机. 对于每一个询问，维护一个vector. 保存出现m的所有下标. 这样会有一个问题：一个字符串出现的位置可能很多，空间可能会爆. 注意到题目的限制：所有字符串的长度加起来不超过10w,并且所有字符串互不相同. 根据第一条可以得出最多有$sqrt(100000)$种不同长度的字符串(1 + 2 + 3 + …… + sqrt(100000) > 100000). 因为字符串互不相同，所以长度相同的字符串，一旦一个位置被其中一个字符串占据了，其它字符串就不会出现在这个位置上. 所以所有字符串在s中出现的总次数是$O(nsqrt(n))$的.

总结

  注意题目中的关键字：总和、互不相同…… 由此可以得到更低的复杂度上界.

Codeforces 983 E. NN country

发表于 2018-06-04 | 分类于 计划 |

题目大意

  给定一棵树和m条链，q次询问，每次询问最少用多少条链可以覆盖指定的一条链.

提示

  链实际上是树上的一条路径，一般处理方法是将路径(u,v)拆成(u,LCA)和(LCA,v).

代码

分析

  非常神的一道题.
  将路径(u,v)拆成(u,lca)和(lca,v). 先考虑一条链上的情况怎么处理. 利用贪心的思想，每个点尽可能往上跳. 预处理出点i经过$2^j$条链能到达的最高点. 这个预处理技巧性很强. 对于每条链先更新端点，然后一次dfs从下往上更新. 利用倍增算法，即可求出链上的答案.
  接下来的任务是将两条路径合并. 将两条路径的答案相加，但是这样答案可能会变多，因为有一种情况：一条链穿过了LCA. 若u经过$2^{p1}$条链可以到LCA.v经过$2^{p2}$条链可以到LCA,令x为u经过$2^{p1 - 1}$条链到达的点，y为v经过$2^{p2 - 1}$条链经过的点，那么要求的就是是否存在路径，这个路径的一端在x的子树内，另一端在y的子树内.
  我们考虑离线，先处理出所有需要判断的点对，如果存在点对x,y，就向x的询问数组中加入一个pair(y,id)（id表示询问下标）。我们建立一个数组chgi，以及一个树状数组sumi，按原树的边dfs一遍，访问到点x时，我们先访问所有x的询问数组，对其中的所有点对(y,id)，让chgid等于y的子树中的sum和（树状数组查询），接着再将以x为端点的每个车次的另一个端点y的值在树状数组的相应位置上加1，访问x的全部子树后，再访问所有x的询问数组，对其中的所有点对(y,id)，如果这时查询的y的子树中的sum和，如果结果不等于chgid，就说明被修改过了，即y的子树中至少有一个点和x的子树中的一个点属于同一趟车次，即x与y能在同一趟车次中互相到达，所以ansid需要减1.利用递归的特性，非常神奇的做法.

总结

  树上往上跳想到倍增.倍增数组的多种定义
  路径问题想到拆成两端，需要考虑两条路径合并对答案的影响.
  树上的自底向上更新，可以先更新端点，然后dfs从下往上更新.

题目大意

  有一个最多只能容纳4个人的电梯，现在给定n个人的出发楼层和目标楼层，必须按照顺序上电梯，即第i个人必须在第i-1个人之后上电梯，电梯移动一层花费1，上一个人or下一个人花费1.求最小花费. 电梯一开始在一楼.

分析

  阶段性.

代码

分析

  题目具有非常明显的阶段性:所有人上电梯的顺序是确定好的. 那么只需要考虑电梯上的人以及第i个人即可. 利用dp解决.
  比较容易想到的状态:$f_{i,j,k,l,p,q}$表示考虑前i个人，当前电梯在第j层，电梯中4个人的目标楼层分别为k,l,p,q的答案，转移非常直观，只可惜空间会爆. 事实上第i个人一定会被考虑到，只需要记录电梯上3个人的目标位置即可. 如果电梯上不满3人，就将第i个人放到这3个人的位置上，否则必须转移完第i个人，即接i再送走一个人，因为状态只能记录3个人的信息，如果不处理完一个人那么多出来的那一个人是没办法记录的，所以当人满时，要先将其中一个人送到目的地，再去接第i个人，或者直接去接第i个人然后送到目的地去.

总结

  找到问题的阶段性要想到dp. 尽可能简化dp记录的状态. 省掉的那一位状态的转移一定要考虑完全!

Description
上午的训练结束了，THU ACM小组集体去吃午餐，他们一行N人来到了著名的十食堂。这里有两个打饭的窗口，每个窗口同一时刻只能给一个人打饭。由于每个人的口味（以及胃口）不同，所以他们要吃的菜各有不同，打饭所要花费的时间是因人而异的。另外每个人吃饭的速度也不尽相同，所以吃饭花费的时间也是可能有所不同的。 THU ACM小组的吃饭计划是这样的：先把所有的人分成两队，并安排好每队中各人的排列顺序，然后一号队伍到一号窗口去排队打饭，二号队伍到二号窗口去排队打饭。每个人打完饭后立刻开始吃，所有人都吃完饭后立刻集合去六教地下室进行下午的训练。 现在给定了每个人的打饭时间和吃饭时间，要求安排一种最佳的分队和排队方案使得所有人都吃完饭的时间尽量早。 假设THU ACM小组在时刻0到达十食堂，而且食堂里面没有其他吃饭的同学（只有打饭的师傅）。每个人必须而且只能被分在一个队伍里。两个窗口是并行操作互不影响的，而且每个人打饭的时间是和窗口无关的，打完饭之后立刻就开始吃饭，中间没有延迟。 现在给定N个人各自的打饭时间和吃饭时间，要求输出最佳方案下所有人吃完饭的时刻。

bzoj3174 [Tjoi2013]拯救小矮人

发表于 2018-06-04 | 分类于 计划 |

中文题面

Description
一群小矮人掉进了一个很深的陷阱里，由于太矮爬不上来，于是他们决定搭一个人梯。即：一个小矮人站在另一小矮人的 肩膀上，知道最顶端的小矮人伸直胳膊可以碰到陷阱口。对于每一个小矮人，我们知道他从脚到肩膀的高度Ai，并且他的胳膊长度为Bi。陷阱深度为H。如果我 们利用矮人1，矮人2，矮人3,。。。矮人k搭一个梯子，满足A1+A2+A3+….+Ak+Bk>=H,那么矮人k就可以离开陷阱逃跑了，一 旦一个矮人逃跑了，他就不能再搭人梯了。
我们希望尽可能多的小矮人逃跑， 问最多可以使多少个小矮人逃跑。

Input
第一行一个整数N， 表示矮人的个数，接下来N行每一行两个整数Ai和Bi，最后一行是H。（Ai，Bi，H<=10^5）
Output
一个整数表示对多可以逃跑多少小矮人
Sample Input
样例1

2

20 10

5 5

30

样例2

2

20 10

5 5

35
Sample Output
样例1

2

样例2

1
HINT
数据范围

30%的数据 N<=200

100%的数据 N<=2000

提示

  实质上要求什么？

代码

分析

  要求的实质上是一个逃跑的小矮人的顺序序列. 这个序列虽然不能直接求得，但是可以知道哪个人在哪一个前面出来结果更优. 利用贪心中交换相邻项的思想：
  若i在j上面，也就是i先逃跑，然后j逃跑，那么$a_j + a_i + b_i ≥ H$,$a_j + b_j ≥ H$,假设这种情况是最优的，若交换i,j，即j先逃跑,那么可能会有两种情况：1.$a_i + a_j + b_j < H$. 2.$a_i + a_j + b_j ≥ H$ 并且 $a_i + b_i < H$,不管是哪种情况，都能导出一个关系式：$a_i + b_i < a_j + b_j$时，i在j前面逃跑.
  注意这里求出来的并不是最终的逃跑顺序，而是一种限制关系. 出现了决策，利用dp求解. 从前往后枚举第i个人，看他跑不跑. 令$f_i$表示有i个人逃跑身体能达到的最大高度. 若$f_j + b_i ≥ H$,则第i个人可以逃跑. 状态和转移都挺奇怪的，因为决策点的顺序确定了，状态里记录的主要就是其他信息了. 一个直观的状态表示是:$f_{i,j}$表示前i个人中，身体总长度为j最多能跑多少个矮人，因为长度太大存不下，所以将状态和状态表示交换. 需要用别的方法统计答案.

总结

  弄清楚要求的是什么. 如果要求一个表示顺序的序列，可以利用贪心法，用邻项交换法得到贪心策略.

bzoj1237 [SCOI2008]配对

发表于 2018-06-04 | 分类于 计划 |

中文题面

Description
你有n 个整数Ai和n 个整数Bi。你需要把它们配对，即每个Ai恰好对应一 个Bp[i]。要求所有配对的整数差的绝对值之和尽量小，但不允许两个相同的数配 对。例如A={5,6,8}，B={5,7,8}，则最优配对方案是5配8, 6配5, 8配7，配对整数 的差的绝对值分别为2, 2, 1，和为5。注意，5配5，6配7，8配8是不允许的，因 为相同的数不许配对。

Input
第一行为一个正整数n，接下来是n 行，每行两个整数Ai和Bi，保证所有 Ai各不相同，Bi也各不相同。

Output
输出一个整数，即配对整数的差的绝对值之和的最小值。如果无法配对，输 出-1。

Sample Input
3

3 65

45 10

60 25
Sample Output
32
HINT
1 <= n <= 10^5，Ai和Bi均为1到10^6之间的整数。

提示

  若没有限制该怎么做？为什么要这么做？这种做法满足什么原则？

代码

分析

  如果没有限制，将a,b两个序列的数分别从小到大排序后一一配对即可. 这体现了一个最优性原则：每个数都与离它尽可能近的数配对.
  如果有限制，那么就会有数与离他不是最近的数配对. 比如a,b两个数与对应位置上的数冲突了，那么a可以匹配b对应位置上的数，b可以匹配a对应位置上的数,此时每个数与配对的数的最长距离为1.如果a,b,c三个数都与对应位置上的数冲突了,此时不管怎么交叉着匹配，最长距离的最小值一定为2. 如果a,b,c,d四个数都冲突了，那么可以拆分成两个a,b冲突的情况，最长距离为1. 由此可以得到一个结论：最优解中，每个数与匹配位置的距离≤2.
  由此dp. 令$f_{i}$表示前i个数的答案，分类讨论4种转移，从$f_{i - 2}$或$f_{i - 3}$转移过来.
