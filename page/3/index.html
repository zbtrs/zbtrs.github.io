Codeforces 917 D. Stranger Trees

发表于 2018-06-09 | 分类于 计划 |

题目大意

给定一棵n个点的树，求与该树恰好有k条边重合的n个点的树有多少种.

提示

  对”恰好”进行转化. 想一想无向完全图怎么计算生成树个数.

代码

分析

  非常神的一道题，简直是计数问题的巅峰！
  找问题的突破口. 恰好k个非常不好处理，要确保恰好就必须知道选哪k条边，这样才能保证生成的图是一棵树. 结合noi2011陨石的秘密一题，可以想到把“等号”变成“不等号”.即求出某个范围内的答案，然后直接相减或者利用容斥原理得到答案.
  假设已经选出了k条边，那么就剩下n-k个连通块，剩下的边随便连. 连通块内部是不能连边的，否则就形成了环，不是树. 那么可以将每个连通块都看作一个点. 计算n-k个点的完全图的生成树个数，可以用prufer序列来做. 利用prufer序列计数的原理是通过一种变换将n个点的生成树与长为n-2的序列一一映射. 统计生成树的个数只需要统计序列个数即可. 因为将所有连通块都看成了点，所以序列长度变成了m-2,每个位置有n中选择，故方案数为$n^{m - 2}$.
  这样计数有一个问题.这种计数方式对于每一个连通块，只计算了它连向儿子节点的方案，没有计算它连向父亲节点的方案. 对于两个连通块连边的方案，应该是两个连通块大小的乘积，不仅要考虑一个连通块选择接受节点的方案数，还要考虑一个连通块选择传出节点的方案数. 那么还要乘上$πsize_i$,即所有连通块的大小的乘积,要加上每一种方案的$πsize_i$.
  这就又是一个难点了，怎么样才能知道将一棵树分成m个连通块的所有方案的$πsize_i$的和呢？限制很多，利用dp求解. 考虑dp需要知道哪些信息：
  1.当前是哪一个节点.
  2.已经构成了多少个连通块.
  3.当前点所在的连通块大小为多少.
  用$f_{i,j,k}$来表示.对于每一个子节点，分选/不选两种决策来转移.需要注意一点：状态的最后一维k是在变动的，只有确定了i不与父节点形成一个连通块时，k才能被计入答案中. 转移以及最后求答案都要注意到这一点.
  令$ans_i$表示有k条边与原树重合的方案数,下面着重讲一讲容斥原理.
  一开始求出来的$ans_i$表示的是至少有k条边与原树重合的方案数. 为了保留这个至少，采取从小到大更新的方式. 根据容斥原理的基本形式一减一加即可. 需要注意的是加减操作需要乘上组合数. 因为上述过程计数得到的结果不同当且仅当选的边的顺序不同，举个例子：本来有(1,2),(2,3)两条边，先选(1,2)再选(2,3)和先选(2,3)再选(1,2)是相同的方案，而上述方法则会认为这是不同的方案. 手推一下就能发现容斥原理时乘上组合数才是正确的计算方法.

总结

  1.从等于号到不等号的转变.
  2.特殊考虑，其它随意的思想，由此可以得到一般情况的计数公式.
  3.容斥原理一定要考虑清楚容斥的顺序!!!有的容斥原理并不只是简单的加加减减，还要考虑计数时认为两种方案不同的标准是什么.

算法竞赛进阶指南第六章

发表于 2018-06-07 | 分类于 算法竞赛进阶指南 |

前言

  花了三个星期才干完这一章，最主要的原因是我的dp水平不过关，加上最近复课了,花在oi上的时间也变少了.

1.一些思考成果

  线性dp总结： http://www.zbtrs.top/2018/05/17/%E7%BA%BF%E6%80%A7dp%E6%80%BB%E7%BB%93/
  为什么使用dp?
http://www.zbtrs.top/2018/05/23/why%20uses%20dp/
  一类基准点划分计数dp:
http://www.zbtrs.top/2018/05/27/%E4%B8%80%E7%B1%BB%E5%9F%BA%E5%87%86%E7%82%B9%E5%88%92%E5%88%86%E8%AE%A1%E6%95%B0dp/

2.不满足最优子结构性导致dp出错的例子

  做这种题不是很多，遇到过两种比较常见的例子.
  1.位运算的最值问题. 一般不能对整体进行dp,而言按位贪心，对每一位进行dp.
  2.求乘积最值，但有负数. 两个绝对值大的负数相乘可能会得到一个更大的正数，而如果仅记录最大值，是无法进行这种转移的，因此还需要记录最小值.

3.背包问题

  0-1背包就不说了，想说一下多重背包的可行性问题. 原则：迫不得已时再用. 对于每一个物品，如果当前体积拼不出来，但是用上这个物品后当前体积能拼出来，并且物品数不会超，就用这个物品，物品数—. 每一轮只考虑当前物品对体积的贡献.
  多重背包的二进制拆分法和单调队列优化. 注意：二进制拆分法只能适用于物品数为正整数的情况.
  还有一类神奇的分治背包问题. 见https://www.cnblogs.com/zbtrs/p/8708099.html (分治，控制边界，不让第i个物品插入背包，插入背包用单调队列优化，还是基于单调队列优化的多重背包问题).
  多重背包问题一定要先枚举余数再来优化. 有时候也可以用斜率优化，见http://www.zbtrs.top/2018/04/13/bzoj4073%20%E6%96%9C%E7%8E%87%E4%BC%98%E5%8C%96+%E8%B4%AA%E5%BF%83/

4.区间dp的两种类型

  1.计算操作的影响时，可以只考虑区间内部或与区间端点相邻的点的值.
  2.需要考虑区间外的影响.
  如何看出用区间dp? $O(n^3)$的复杂度，并且数字会被消掉(挪动).
  对于第一类问题，一般的状态就是$f_{l,r}$,枚举中间点k来转移. 可能还要考虑端点的转移，因此转移可能会分多种情况讨论. 如果需要考虑的信息比较多，可能要多加一维. 见http://www.cnblogs.com/zbtrs/p/8454935.html
  对于第二类问题，通常需要多加一维表示向右/左扩展多少位. 见poj1390

5.换根与二次扫描法

  先从下往上dfs一次，再从上往下更新. 一般要用到两个数组.注意递归与更新子节点的先后顺序.

6.环形dp与环形状态转移方程

  环形dp的两种处理方法：
  1.强行经过分界点，处理分界点的情况.
  2.复制一份，在链上处理.
  一般是选用第二种方法，不仅状态转移方程唯一，还能知道环是从哪里断开的. 如果用第二种方法不能使得状态转移方程唯一(或者是有特殊情况，必须枚举起点，使得复杂度特别高),就必须使用第一种方法了. 通过改变边界条件进行两次dp实现目的.
  后效性的处理方法：列出状态转移方程，高斯消元求解. 注意特殊的矩阵.

7.状压dp

  需要知道一些元素的信息. 不仅可以用二进制状压，还可以用三进制状压…….十进制状压就和普通的记录信息没什么区别了.
  可能会有大量的无用状态，处理无用状态的方法是预处理出有用状态(对于每一阶段而言约束条件都一样)或是利用刷表法，只扩展有用状态.(配合dfs转移).

8.倍增优化dp转移

  状态转移具有单一重复性. 你可以手动模拟状态转移，发现转移的方向是确定的，在一维上转移量比较大，转移可以叠加，那么就可以用倍增来优化dp转移.一般都是先预处理出倍增数组，然后从高位到低位拼凑.
  常见模型：1.拼数. 2.树上往祖先节点跳.

9.数据结构优化dp转移

  这类优化针对的是对决策的权值有要求、决策点的位置有要求的状态转移方程. 如果仅仅只是对决策点的位置有要求，可以用单调队列，单调栈之类的单调数据结构进行维护. 这类优化还是很好看出来的.

10.单调队列的神级应用

  1.最简单的一类应用：决策点的范围随着i的增大而上下界同时增大. 可以认为决策点和i是连在一起的.
  2.比较容易写错的一类应用：决策点和i不连在一起，每次加入队列的不是第i个元素，而是第i-k个元素.
  3.放入单调队列的不仅仅是$f_j$,而是关于j的整个多项式.
  4.利用贪心思想，数学证明找到最优决策点的特征，发现正好可以用单调队列维护. 此时，单调队列维护的仅仅是可能的最优决策点，找最优决策点还需要在队列中枚举判断，可以用set加速.或者利用单调队列维护的元素的性质，可以直接从队尾/队首找到最优决策点.
  5.若决策点位置满足的多项式与转移代价多项式不同，将哪一个放入单调队列中取决于用单调队列维护什么. 若维护可能的最优决策点，则将决策点位置满足的多项式放入单调队列中，否则放入转移代价多项式.
详见poj3071,bzoj1233.

11.费用提前计算思想

  不知道它之前的具体代价，但知道它对以后的代价的影响. 比如物品加工的等待时间，不知道前面有多少组物品加工对当前物品的等待时间有影响，但是知道当前物品加工对后面物品的等待时间一定有影响，将影响加入到转移的代价中，就消除了后效性.

12.斜率优化与四边形不等式

  原则：优先选择斜率优化，不行再用四边形不等式优化.
  斜率优化的两种形式，一种是$f_i = f_j + ……$一种是$f_j = f_i + ……$,第一种是直接计算，第二种是将$f_i$变成截距，反解得到. 一般采用第一种形式，如果第一种形式的斜率不单调，则采用第二种形式，保留整个凸包(不进行队首的操作)，二分找斜率分界点.
  四边形不等式优化的两种类型：
  1.一维状态dp. $w_{i,j + 1} + w_{i + 1,j} ≥ w_{i,j} + w_{i + 1,j + 1}$.
  2.二维状态dp. 满足上述式子的同时，还要满足$w_{i,j + 1} ≥ w_{i + 1,j}$.这样二维状态dp也满足四边形不等式，决策位置$pos_{i,j - 1}≤pos_{i,j}≤pos_{i + 1,j}$.注意枚举顺序.

13.数位dp

  前导0的处理. dfs的所有参数都丢到状态中记录.

中文题面

Description
　　有n个木块排成一行，从左到右依次编号为1~n。你有k种颜色的油漆，其中第i种颜色的油漆足够涂ci个木块。
所有油漆刚好足够涂满所有木块，即c1+c2+…+ck=n。相邻两个木块涂相同色显得很难看，所以你希望统计任意两
个相邻木块颜色不同的着色方案。

Input
　　第一行为一个正整数k，第二行包含k个整数c1, c2, … , ck。

Output
　　输出一个整数，即方案总数模1,000,000,007的结果。

Sample Input
3

1 2 3
Sample Output
10
HINT
100%的数据满足：1 <= k <= 15, 1 <= ci <= 5

提示

  减少状态数，数字之间有差异吗？

代码

分析

  非常巧妙的一道题.
  一开始看题面，这么短肯定是数学方法吧？去看数据范围，woc怎么只有这么点. 感觉不对劲,利用容斥原理等各种数学方法都做不出来. 那么考虑dp. 因为是排列计数问题，有两种思考方式：要么考虑插入一个数到某个位置，要么按照顺序填数. 如果是插数，因为可以放数的位置数不确定，所以这种方法不行. 那么只有第二种方法了.
  每个集合内的数都是有限的，那么在填第i个数之前，需要知道每个集合剩下的数的个数. 这样状态数就是$5^{15}$,显然是过不了的. 如果反过来，$15^5$就可以过了. 事实上所有数字都是没有差异的，那么只需要知道大小为i的集合有多少个以及上一次选的数所在的集合的大小是多少即可.

总结

  计数题小数据范围一般对应dp
  排列计数的两种dp顺序.
  若对于所有对象而言，决策都是一样(类似)的，那么可以认为这些对象没有区别. 状态中只需要记录公共特征值即可(是多少—>有多少个)

代码

分析

  我的dp方法和网上的大部分方法都不一样. 令$f_{i,j}$表示已经设置了i个邮局，考虑了前j个村庄，并且第j个村庄必须设邮局的答案. 因为最后一个村庄在实际答案中可能不设邮局，所以加一个虚拟村庄n + 1,位置为inf. 最后的答案为$f_{k + 1,n + 1}$.可以先$O(n^3)$预处理出转移代价$w_{i,j}$.
  这个dp方程不能进行优化，虽然可以通过原题，但是比较慢. 换一种dp方法：不再强制令第j个村庄设置邮局，而是在第k个村庄和第j个村庄中间设置邮局，利用中位数原理. 其中k是转移到j的决策位置. 如果在比赛中，状态不能简化，但是需要优化dp时，考虑优化状态转移方程. 优化状态转移方程考虑5种方法，要么是只增不减求最值的决策集合，利用一个变量维护即可，要么可以用数据结构优化，要么转移多项式只与一项有关，可以用单调队列优化.要么与两项有关，斜率优化. 都不行的话，就考虑四边形不等式了. 四边形不等式的证明比较难，大胆猜想，对拍验证即可.
$f_{i,j}$转移的区间范围在$f_{i - 1,j}$到$f_{i,j + 1}$的区间范围内，倒序枚举j即可.

Codeforces 963 D. Frequency of String

题目大意

  给定一个字符串s,n次询问，给定k和m,求出一个字符串，使得这个字符串是s的子串，并且m作为它的子串至少出现k次.

代码

分析

  如果只有一次询问要怎么做？KMP即可，找到m在s中所有出现的位置，第i+k个位置的坐标-第i个位置的坐标+字符串的长度的最小值即为答案.
  多次询问要怎么处理呢？n次KMP？换个角度，用s去匹配m. 因为是多对一，所以建AC自动机. 对于每一个询问，维护一个vector. 保存出现m的所有下标. 这样会有一个问题：一个字符串出现的位置可能很多，空间可能会爆. 注意到题目的限制：所有字符串的长度加起来不超过10w,并且所有字符串互不相同. 根据第一条可以得出最多有$sqrt(100000)$种不同长度的字符串(1 + 2 + 3 + …… + sqrt(100000) > 100000). 因为字符串互不相同，所以长度相同的字符串，一旦一个位置被其中一个字符串占据了，其它字符串就不会出现在这个位置上. 所以所有字符串在s中出现的总次数是$O(nsqrt(n))$的.

总结

  注意题目中的关键字：总和、互不相同…… 由此可以得到更低的复杂度上界.

Codeforces 983 E. NN country

发表于 2018-06-04 | 分类于 计划 |

题目大意

  给定一棵树和m条链，q次询问，每次询问最少用多少条链可以覆盖指定的一条链.

提示

  链实际上是树上的一条路径，一般处理方法是将路径(u,v)拆成(u,LCA)和(LCA,v).

代码

分析

  非常神的一道题.
  将路径(u,v)拆成(u,lca)和(lca,v). 先考虑一条链上的情况怎么处理. 利用贪心的思想，每个点尽可能往上跳. 预处理出点i经过$2^j$条链能到达的最高点. 这个预处理技巧性很强. 对于每条链先更新端点，然后一次dfs从下往上更新. 利用倍增算法，即可求出链上的答案.
  接下来的任务是将两条路径合并. 将两条路径的答案相加，但是这样答案可能会变多，因为有一种情况：一条链穿过了LCA. 若u经过$2^{p1}$条链可以到LCA.v经过$2^{p2}$条链可以到LCA,令x为u经过$2^{p1 - 1}$条链到达的点，y为v经过$2^{p2 - 1}$条链经过的点，那么要求的就是是否存在路径，这个路径的一端在x的子树内，另一端在y的子树内.
  我们考虑离线，先处理出所有需要判断的点对，如果存在点对x,y，就向x的询问数组中加入一个pair(y,id)（id表示询问下标）。我们建立一个数组chgi，以及一个树状数组sumi，按原树的边dfs一遍，访问到点x时，我们先访问所有x的询问数组，对其中的所有点对(y,id)，让chgid等于y的子树中的sum和（树状数组查询），接着再将以x为端点的每个车次的另一个端点y的值在树状数组的相应位置上加1，访问x的全部子树后，再访问所有x的询问数组，对其中的所有点对(y,id)，如果这时查询的y的子树中的sum和，如果结果不等于chgid，就说明被修改过了，即y的子树中至少有一个点和x的子树中的一个点属于同一趟车次，即x与y能在同一趟车次中互相到达，所以ansid需要减1.利用递归的特性，非常神奇的做法.

总结

  树上往上跳想到倍增.倍增数组的多种定义
  路径问题想到拆成两端，需要考虑两条路径合并对答案的影响.
  树上的自底向上更新，可以先更新端点，然后dfs从下往上更新.
