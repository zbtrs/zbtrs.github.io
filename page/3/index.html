bzoj1237 [SCOI2008]配对

发表于 2018-06-04 | 分类于 计划 |

中文题面

Description
你有n 个整数Ai和n 个整数Bi。你需要把它们配对，即每个Ai恰好对应一 个Bp[i]。要求所有配对的整数差的绝对值之和尽量小，但不允许两个相同的数配 对。例如A={5,6,8}，B={5,7,8}，则最优配对方案是5配8, 6配5, 8配7，配对整数 的差的绝对值分别为2, 2, 1，和为5。注意，5配5，6配7，8配8是不允许的，因 为相同的数不许配对。

Input
第一行为一个正整数n，接下来是n 行，每行两个整数Ai和Bi，保证所有 Ai各不相同，Bi也各不相同。

Output
输出一个整数，即配对整数的差的绝对值之和的最小值。如果无法配对，输 出-1。

Sample Input
3

3 65

45 10

60 25
Sample Output
32
HINT
1 <= n <= 10^5，Ai和Bi均为1到10^6之间的整数。

提示

  若没有限制该怎么做？为什么要这么做？这种做法满足什么原则？

代码

分析

  如果没有限制，将a,b两个序列的数分别从小到大排序后一一配对即可. 这体现了一个最优性原则：每个数都与离它尽可能近的数配对.
  如果有限制，那么就会有数与离他不是最近的数配对. 比如a,b两个数与对应位置上的数冲突了，那么a可以匹配b对应位置上的数，b可以匹配a对应位置上的数,此时每个数与配对的数的最长距离为1.如果a,b,c三个数都与对应位置上的数冲突了,此时不管怎么交叉着匹配，最长距离的最小值一定为2. 如果a,b,c,d四个数都冲突了，那么可以拆分成两个a,b冲突的情况，最长距离为1. 由此可以得到一个结论：最优解中，每个数与匹配位置的距离≤2.
  由此dp. 令$f_{i}$表示前i个数的答案，分类讨论4种转移，从$f_{i - 2}$或$f_{i - 3}$转移过来.

Codeforces 981 G. Magic multisets

发表于 2018-06-04 | 分类于 计划 |

题目大意

  有n个可重集合，两种操作，一种操作是将第l到第r个集合都加入一个数x,若x在集合中已存在，则不加入，将该集合中的所有数复制一份，否则直接加入. 第二种操作是询问第l到第r个集合的大小之和.

提示

  一般的做法是要记录每个集合中放了哪些数. 这道题中修改操作只有两种情况，要么复制，要么插入. 类似区间开方，欧拉定理求幂次之类的. 区间有固定的操作次数，暴力修改.

代码

分析

  不同于网上利用set维护区间端点的做法，我从AwD神犇那里学到了一种非常神奇的线段树做法.
  有一个结论：对于要操作的数v,每一个区间都只会被v操作一次. ，也就是说，可以将所有区间变成若干个一样的区间，每一个区间是针对不同的v而言的. 因为不同的v对区间的影响是独立的，但是同样都会累加进答案. 那么可以像主席树一样，对于每一个要操作的数v建一棵线段树，记录区间[l,r]上有多少个集合被插入了v. 若当前递归到的区间被修改完了，就不用继续递归下去，否则暴力修改所有没有被插入v的集合.
  这样做可能会有一个问题：空间开不下. 常用的解决方法是动态开点，前提是线段树中只需要用到与操作有关的部分,与操作无关的部分无需存储. 这道题涉及到pushdown操作，所以在pushdown操作中动态开点.(如果在递归到的每一层动态开点，在pushdown的时候可能就不存在左右儿子).
  维护一棵主线段树和n棵附属线段树. 第i棵线段树对应要操作的数i. 同时对两棵线段树进行暴力修改，若当前区间覆盖完了，则可以直接将主线段树上的对应区间和*2,同时pushdown,pushup. 询问只需要在主线段树上进行.

总结

  线段树的神奇用法. 主要思想是将线段树按照权值分类. 有多少种可能的权值就建多少个线段树，这样做的好处是每一种权值的操作都是独立的,将在一棵线段树上的复杂操作变成了多棵线段树上的简单操作. 这种对权值分类的方法的前提是权值不大and不多. 另一个例子：要在一棵Trie树上所有长度为k(1 ≤ k ≤ n)的倍数的字符串上操作,不好处理，那么就建n棵Trie树，第i棵存储为i的倍数的字符串. 主要就是一种分类简化的思想，前提是空间够用.
  动态开点解决线段树内存不够的问题. 注意有没有pushdown函数. 如果动态开点了，一定不要再用常规的线段树的左右儿子的写法，而是lc[o],rc[o].

bzoj1233 [Usaco2009Open]干草堆tower

发表于 2018-06-03 | 分类于 算法竞赛进阶指南 |

中文题面

Description
奶牛们讨厌黑暗。 为了调整牛棚顶的电灯的亮度，Bessie必须建一座干草堆使得她能够爬上去够到灯泡 。一共有N大包的干草（1<=N<=100000）(从1到N编号)依靠传送带连续的传输进牛棚来。第i包干草有一个 宽度W_i(1<=w_i<=10000)。所有的干草包的厚度和高度都为1. Bessie必须利用所有N包干草来建立起干草堆，并且按照他们进牛棚的顺序摆放。她可以相放多少包就放 多少包来建立起tower的地基（当然是紧紧的放在一行中）。接下来他可以放置下一个草包放在之前一级 的上方来建立新的一级。注意：每一级不能比下面的一级宽。她持续的这么放置，直到所有的草包都被安 置完成。她必须按顺序堆放，按照草包进入牛棚的顺序。说得更清楚一些：一旦她将一个草包放在第二级 ，她不能将接下来的草包放在地基上。 Bessie的目标是建立起最高的草包堆。

Input
第1行：一个单一的整数N。 第2~N+1行：一个单一的整数:W_i。

Output
第一行：一个单一的整数，表示Bessie可以建立的草包堆的最高高度。

Sample Input
3

1

2

3

Sample Output
2

输出说明：

前两个（宽度为1和2的）放在底层，总宽度为3，在第二层放置宽度为3的。

       +----------+

       |    3     |

       +---+------+

       | 1 |   2  |

       +---+------+

代码

分析

  很容易想到$O(n^3)$的dp.因为转移必须要知道当前考虑到了第几个干草包，上一层的宽度多少.所以可以令$f_{i,j}$表示前i个干草包中，上一层宽度为j的最大高度值，因为每一层所用的宽草包都是连续的，所以可以将j优化成上一层是从哪个干草包开始的.
  这样dp的复杂度太高了，没办法对状态和转移进行化简，记录的信息没办法更简单了. 走投无路之下可以利用贪心将决策减少为一种，从而减少状态维数. 考虑如何使高度尽量高，肯定是每一层的宽度能少就尽量少，那么转移只需要从可能的宽度最小的状态转移过来即可. 那么令$f_i$表示前i个干草包中，当前层的最小宽度，再开一个数组$g_i$记录高度. $f_i$从$f_j$转移而来，状态转移方程后面接的多项式只与i,j中的一项有关，所以可以利用单调队列优化到$O(n)$. 这样做其实有一个问题：j的取值范围不符合单调队列的要求，并且不满足单调性. 回忆一下单调队列优化的原理：考虑k < j的情况，若最优决策为j时，k一定是最优决策，而k是最优决策时j不一定是最有决策，那么就可以将k弹出. 解决这种问题的方法是换个顺序推，取值范围正好就变了回来，并且满足单调性.
  令$f_i$表示后i个干草包的min值，$f_i = min\{sum_{j - 1} - sum_{i - 1}\}(f_j ≤ sum_{j - 1} - sum_{i - 1})$,将$sum_{j - 1} - f_j$看作一个整体，从可行性推最优决策是谁. $k < j,sum_{k - 1} - f_k ≤ sum_{j - 1} - f_j$时，k就可以弹出了，因为决策j能使得i的取值范围更大.
  注意队首元素的处理. 这里队首弹出元素不光要考虑是否满足要求，还要考虑最优性. 判断队首元素的下一个元素是否可行，如果可行则弹出队首元素，因为两个决策都可行的情况下，肯定是选j更小的决策.

总结

  dp复杂度与要求复杂度差距悬殊，而状态和转移都无法化简时，考虑贪心，优化为一种转移方式，状态也可以随之改变.
  单调队列优化的原理.不仅仅是最优性可以用单调队列优化，可行性也可以.
  队尾排除非最优决策，队首排除不满足限制的决策，在满足可行性的前提下求最优解. 通常将出现在限制条件中的式子看作一个整体放到单调队列中，若和转移式中出现的式子不一样，可能要进行特殊处理. 比如在单调队列中枚举解，或者利用决策性质推出最优决策的位置.(在本题中就是不断取出满足条件的队首元素). 可以说：单调队列维护的是可能是最优决策的决策集合.

题目大意

  一个n*m的网格图，有一些障碍点，用2*3或3*2的骨牌覆盖，求最多能放多少张骨牌.

提示

  m特别小，想到状压.

代码

分析

  很容易想到状态压缩. 关键就是如何表示出状态. 仅仅只用0,1表示状态是不够的，需要用3进制表示状态. 若第i位为0，则表示当前行的第i位和它上面的那个位置没有被占，若为1，则表示上面的位置被占，当前位置没有被占，若为2，则表示当前位置被占了. 为什么要这么表示状态？因为骨牌的数量与骨牌能不能放有关，若在第i行第j列放，则考虑骨牌能否以(i,j)为左下角. 这样必须要记录上面两行的信息. 因为考虑的是骨牌的下边界，所以从第i行转移到第i+1行时，2 —> 1, 1 —> 0,所有的骨牌都往上挪一格. 利用上一行的信息可以知道上上一行的信息.
  这样的状态数高达$3^10$,若枚举上一行的状态和当前行的状态，时间上承受不了. 注意到有障碍格子的存在，会有很多无效状态. 这些无效状态中，一部分是当前行的状态与障碍格子冲突产生的，一部分是当前行的状态与上一行的状态冲突产生的. 利用刷表法处理掉无效状态，而不是枚举. 因为枚举相当于复杂度没变. 考虑从当前状态向下一层状态扩展，利用dfs搜出放骨牌的所有决策，然后更新.

总结

  处理无效状态可以利用刷表法，而不是枚举状态+判不合法. 后一种方法只能应用于对于每一行的状态而言，限制条件都是一样的情况.
  填表法可以用dfs转移(转移太复杂太多).

bzoj2337 [HNOI2011]XOR和路径

发表于 2018-06-03 | 分类于 算法竞赛进阶指南 |

中文题面

提示

  不要被样例解释给迷惑了.

代码

分析

  看了样例解释后一直在思考自环怎么解决，感觉走到终点的期望是若干个无限等比数列求和，不知所措.
  不要想这么复杂了，直接利用dp的思想解决. 因为是期望，所以倒推，令$f_i$表示从点i到终点的异或期望值. 这个状态是没办法转移的，因为期望不能进行异或运算, dp求期望一般都是利用期望的线性性相加得到. 这里对二进制的每一位单独求期望，因为每一位要么是0要么是1，只有两种关系. 假设第i个点的度数为$d_i$,若i到j的一条边的边权的第k位为1,那么$f_i += (1 - f_j) / d_i$,$f_i$表示i号点到终点第k位为1的期望. 对于边权的第k位为0的情况类似.
  因为是无向连通图，最后转移关系形成了一个环，需要高斯消元来解. 其中$f_n = 0$,那么就只有n-1个方程和n-1个未知数. 需要注意的是要先将第n个方程的第n项系数设置为1，右边设置为0，这样第n项解出来就是0了.

总结

  求期望最基本的方法就是倒推dp. 一般涉及到每个状态决策的概率计算问题，关系比较简单(0,1),则可以直接得到概率. 关系比较复杂的(xor),就只能通过其他方法转化为简单关系再来做. 当转移成环时，利用高斯消元解. 注意赋初值的方法.

代码

分析

  这道题非常容易想到区间dp. 按照区间dp的常规套路来，令$f_{i,j}$表示区间[i,j]的最大代价. 有些情况是转移不了的. 比如多个不相邻相同颜色的色块合并，转移只能枚举k,最多只能处理3个相同颜色的色块的合并.
  区间外会影响区间内，端点色块可能会和区间外的相同色块合并，二维状态只能维护区间内部的情况，如果要处理区间外对区间内的影响，就必须再多加一维. 令$f_{i,j,k}$表示区间[i,j],右端点j与其右边的k个相同色块合并的最大代价. 转移的时候考虑右端点色块，要么只和区间外的色块合并，要么和区间内的色块合并.

总结

  区间dp的常规状态表示只能处理区间内部的情况，对于区间外部的情况需要多加一维. 只需要考虑左端点or右端点向外扩张的情况即可.

分析

  吼题啊！学到了许多新姿势.
  首先很容易想到状态$f_{i,j,k,l}$表示选了i个大括号，j个中括号，k个小括号，深度为l的方案数. 表达式可以看作是许多独立的部分组成的，深度即为这些部分的深度的最大值.
  每次考虑当前括号，是放在前面的一个部分中还是新创一个部分？一个比较麻烦的问题是深度不好维护. 因为不知道最大值是哪一个部分. 难道要记录每一部分的深度？这样就成了状压dp了，复杂度是不能接受的,而且还不知道具体有多少部分.
  换一种计数方式,不在对括号计数，而是对部分计数. 每次选取一个整体计数，采取基准点计数法. 以第i对括号所在的部分为基准点. 这样还是没有解决维护深度的问题,因为还要枚举前面其它部分的深度是多少，复杂度是指数级的. 那么改变一下状态:令l表示深度≤l. 这样枚举当前部分的深度d,因为第i个括号已经被确定了，所以当前部分的方案数就是$f_{a,b,c,d - 1}$, a,b,c需要枚举. 其它部分的方案数就是$f_{l1 - a,l2 - b,l3 - c,d}$,前3维可能要-1，取决于第i个括号是哪种类型的括号.
  最后的答案就是$f_{l1,l2,l3,d} - f_{l1,l2,l3,d - 1}$，相当于将具体值变成了前缀.

小结

  计数对象的选择：对整体，可以用基准点计数法，对个体，看这个个体的决策.
  计数=类型的方案数可以变成计数≤类型的方案数. 范围相减得到具体答案.
