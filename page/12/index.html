发表于 2018-05-10 | 分类于 算法竞赛进阶指南 |

中文题面

Description
Freda发明了传呼机之后，rainbow进一步改进了传呼机发送信息所使用的信号。由于现在是数字、信息时代，rainbow发明的信号用N个自然数表示。为了避免两个人的对话被大坏蛋VariantF偷听T_T，rainbow把对话分成A、B、C三部分，分别用a、b、c三个密码加密。现在Freda接到了rainbow的信息，她的首要工作就是解密。Freda了解到，这三部分的密码计算方式如下：
在1~N这N个数中，等概率地选取两个数l、r，如果l>r，则交换l、r。把信号中的第l个数到第r个数取出来，构成一个数列P。
A部分对话的密码是数列P的xor和的数学期望值。xor和就是数列P中各个数异或之后得到的数； xor和的期望就是对于所有可能选取的l、r，所得到的数列的xor和的平均数。
B部分对话的密码是数列P的and和的期望，定义类似于xor和。

样例输出2
0.667 0.222 0.889

分析

  这道题虽然披上了期望的外衣，实质上就是求出所有区间的xor,and,or和.
  涉及二进制运算的问题一般怎么快速求和？ 对每一位分别求和，若第i位有cnt种方案满足其为1，则它对答案的贡献为$2^i * cnt$.将每一位的贡献加起来就是答案了.
  涉及到所有区间的值，通常的方法是固定一个端点，令另一个端点快速移动. and和or运算都具有“单调性”，可以很容易地确定对答案有贡献的区间的个数. 具体方法是：设当前处理的是二进制的第p位，枚举区间的右端点r. 若第r个数的第p位为1，则找到r左边第一个第p位为0的数，记其位置为pos,那么左端点在[pos + 1,r - 1]的区间的and值都对答案有影响. 对于or操作同理.
  比较难处理的是xor操作. 有一个性质：在xor操作中，遇到0异或值不变.遇到1会变成对应的数.那么序列就可以被分成若干段，每一段的端点由1个0和1个1组成. 对这些段进行黑白染色. 若当前的右端点是第r位，那么要么是左端点在黑色段的区间产生贡献，要么是左端点在白色段的区间产生贡献.具体怎么维护见代码.
  上面所提到的区间都没有涉及l = r的情况. 这是因为当l = r时，区间出现的概率是$1 / n^2$,而l != r时，区间出现的概率是$2 / n^2$,故要单独处理l=r的情况.

总结

  xor操作求区间和分成一段一段来维护.

分析

  在DAG上求期望，显然是期望dp. 我一开始想的做法是dfs,到点u后枚举其后继节点v. 若u的出度为k,则$f_v += (f_u + w) / k$. 这种做法有问题，因为dfs是一条道走到黑，可能点v的f值还没有更新完，就用v去更新其后继节点了. 那么改成拓扑排序，结果还是错的. 因为一个有一个原则反向求期望，正向求概率.
  分析样例中的图，2—>3这条边的权值应该乘上1/2（走这条边的概率），而实际计算中是没有乘的. 这是因为在dp的时候没有考虑走到点u的概率是多少，从而转移默认了走u—>v这条边的概率是1.由此可知：如果正向求期望，会错误地计算很多不合法的状态. 一种解决方法是求出到达每个点的概率，那么这个点出边的边权就要乘上这个概率. 还有一种解决方法是构造反向图，倒着转移. 即假设到达v的状态合法，那么从u转移过去的状态一定合法. 由此可以总结出求期望的一般方法：要么是正向求概率乘权值，要么是倒着求期望.倒过来求要注意状态和问题的转化.

IPSC2016C Counting Swaps

发表于 2018-05-10 | 分类于 算法竞赛进阶指南 |

中文题面

给定你一个 1∼n 的排列 ${p_i}$ ，可进行若干次操作，每次选择两个整数 x,y，交换 $p_x,p_y$。
请你告诉穰子，用最少的操作次数将给定排列变成单调上升的序列 1,2,…,n，有多少种方式呢？请输出方式数对 ${10^9+9}$ 取模的结果。
输入格式
第一行一个整数 T 代表数据组数。

每一组测试数据，第一行是一个整数 n 代表排列中的元素个数，第二行 n 个整数，是这个排列。

输入数据中测试数据间也许存在空行，请务必注意。

输出格式
T 行，一行一个整数，代表这组测试数据的答案。

样例一
input
1
3
2 3 1
output
3
explanation
至少需要两步，有三种操作方式。

先换 2,3，再换 3,1。
先换 2,1，再换 3,2。
先换 3,1，再换 2,1。
数据范围与约定
对于 50% 的数据，1≤n≤10。
对于 100% 的数据，1≤n≤100000，T=100。

分析

  真是神题一道.
  题目先是限制了操作步数最少，在此基础上求方案数. 一步一步来. 先考虑怎么让操作步数最少. 我一开始以为是逆序对数，其实并不是. 逆序对数要求交换操作只能是相邻的元素,而本题中可以是任意元素. 将序列转化到图上去思考.
  每个元素的最终位置我们是知道的. 对于第i个位置上的数$p_i$,从i向$p_i$连一条边. 可以发现最终构出来的图会形成许多的环,每个点只会有一条出边. 最终局面就是图上只有自环.单独考虑每一个环，任意交换环上两个点的出边，可以发现环被分成了两个环，边的总数不会变. 继续分下去，一定都会变成自环，统计操作次数，可以发现一个结论：把一个长度为n的环变成n个自环，最少需要n-1次操作.
  知道了最少操作次数，同时也知道了怎么操作：单独考虑每一个环，每次任选两个点交换出边，一定可以将大环分成两个小环. 那么我们可以统计出将一个大环分成两个小环游多少种方案数，在来考虑怎么分小环,以此来计数.
  上面只考虑了将环分成哪两种的方案数，没有考虑具体怎么分. 实际上根据分的环的大小不同，方案数也是不同的，因此要用递推的方法.记$f_i$为把长度为n的环变成n个自环的方案数，枚举分成的一个小环的长度x,那么另一个小环的长度就是n-x.记将环分成长度为x,y的两个小环的方案数为$T_{x,y}$,那么$f_n = ΣT(x,y) * f_x * f_y *?$,这个问号是我们要考虑的.
  每一种分法，顺序不同，不会影响结果，但是会影响计数的结果. $f_x$的步数为x - 1,$f_y$的步数为y-1,那么总的步数就是n - 2. $f_x$的x-1步取n-1中的哪x-1步？这x-1步先进行哪一步，后进行哪一步都是我们要考虑的,根据多重排列数公式:$(n - 2)! / [(x - 1)! * (y-1)!]$.这就是问号那一部分的式子.
  如何算$T_{x,y}$?,找规律啦！对于一个长度为n的环，如果x == y,那么n必定是偶数，长度为x的环的起点可以是大环中的左半部分(如果是有半部分，因为是一个环，并且x=y,就会造成重复统计)，所以此时$T_{x,y} = n/2$,否则$T_{x,y} = n$.
  要求出所有的环的f值，复杂度是$O(n^2)$的，这个时候就要打表找规律啦！$f_n = n^{n - 2}$.

总结

  1.最优策略下的计数问题，先要找到最优策略是什么，再来计数. 关键是要找到最优策略的特征!
  2.当取值不同而使得方案数不同时，枚举这个取值，定义函数来计算方案数.
  3.考虑清楚在分解成子问题计算方案数时，是否要考虑分配的方案数.
  4.和jloi成绩公布那道题结合起来看.

分析

  神题一道.
  有一个结论：每个数的次数都小于等于63 / k,因为题目保证了答案不超过$2^63$,那么对于$k ≥ 3$的情况，直接$O(2^n)$爆搜即可. 主要是处理$k=1$和$k=2$的情况.
  求期望，关键在于求权值以及对应的概率. 权值很好算，就是枚举二进制位，看其是否为1. 关键是要怎么算概率？对于k = 1的情况. 若n个数中至少有一个数的二进制第i位为1，则异或值的第i位有$1/2$的概率为1,否则概率为0. 假设$a_j$的第i位为1，那么出现奇数次$a_j$和偶数次$a_j$ 子集的概率是一样的. 故概率为$1/2$.
  对于k = 2的情况. 枚举两个二进制位p,q. 若第p位或第q位在这n个数中都为0，那么对答案没有贡献. ​ 若这n个数中的每一个的第p位和第q位要么同时为1，要么同时为0，则可以转化为k=1的那种情况. p,q有贡献的概率是$1/2$. 否则，p位异或起来为1的概率为$1/2$,q位同样也为$1/2$,总概率是$1/4$. 可能会有特殊情况，详情见代码.
  对于$k ≥ 3$的情况，先求出线性基，然后暴力枚举选哪些基底异或起来，统计的答案. 需要注意的是中间过程因为k太大会爆long long. 一个非常巧妙的方法是将一个数拆开表示. 最后的结果是$ans / 2^{cnt}$,cnt为线性基的基底数量,那么我们可以维护两个变量a,b. 在乘方的时候b先乘上异或出来的结果，若大于$2^{cnt}$了，就把大的那一部分给a. 就相当于把一个数x拆成了a+b,乘法则利用乘法分配律运算. 最后的答案若b不为0，则小数部分要输出0.5(虽然我也不知道为什么).

总结

  1.分情况做.
  2.集合的小定理. 选定一些元素，出现在子集偶数个和奇数个的概率相等.
  3.将数字拆开来存.

提示

  遇到最大异或问题，用什么方法解决？ 怎么考虑所有路径？

代码

分析

  神题一道.
  要求最大异或值，我一开始是想着用Trie来做. Trie只能求两个单点的最大异或值，那么能不能预处理出1到每个点的路径的异或值和n到每个点路径的异或值？因为路径可能有多条，自然是不行的.
  如果没有环能不能dp呢？也不行. 设计的dp状态无法满足最优子结构性.
  除了这两种做法也就只有线性基了.
  一个结论：1到n的路径的异或和一定由一条路径的异或和和一堆环异或和异或而成,也就是说：如果选出了一条从1到n的路径，那么一定可以选出一条路径和一堆环，使得异或值与之相等. 如果选出了一条路径和一堆环，一定有一条从1到n的路径的异或值与之相等. 根据异或的抵消性可以证明:
  我们先走环，再走路径. 如果答案包含了一个环的异或值，我们可以从1号点走到环上遍历一圈再走回来. 这样从起点到环的路径的异或值被抵消了，正好保留了环的异或值. 也就是说，我一定可以将环的异或值放到答案中. 最后再走路径. 如果路径和环重叠了，那么重叠的那一部分会被抵消，环剩下的部分和剩下的路径又能组合成一条新路径. 也就是说，通过异或，我们一定可以走一条从1到n的路径.
  那么具体算法就是先随便找一条从1到n的路径. 再dfs预处理出所有的环的异或值（利用环的两个端点到1的异或值），最后高斯消元，线性基求最大值即可.

分析

  根据hdu3949,我们可以通过将Q分解成二进制，从而计算Q是第几小的. 具体方法是：维护一个变量cnt. 若从大到小第i个线性基存在于Q中，那么$cnt += 2^i$.
但是这道题的计数并没有这么简单,异或空间的异或值可能有重复的.
  一定能从$a_1,a_2,……,a_t$中找出t个线性无关的整数，构成异或空间的一个基。不在基底的整数有n-t个，从这n-t个整数中选出若干个，显然有$2^{n-t}$种方案. 对于任意一种选法，设选取的整数异或起来得到x,把基底能表示出的$2^t$个互不相同的整数与x分别异或，又得到$2^t$个整数,这些整数互不相同. 因此可以得到一个结论：不去重异或集合就是去重异或集合中的$2^t$个整数个重复$2^{n-t}$次形成的 根据这个结论，最后的答案就是$cnt * 2^{n-t} + 1$.

翻译

从n个数中选取若干个数，异或起来. 求所得的第k小的值.

代码

分析

  线性基套路题.
  首先找出这n个数中有用的数，即基底. 将每一个数看作m位二进制数，构造一个n行m列的矩阵. 对这个异或矩阵做高斯消元，最后得到的非0行即为矩阵的秩，也就是有用的那些数. 为什么呢？因为对矩阵做初等行变换矩阵的秩是不会发生改变的，当把矩阵变成相抵标准型后，没用的数都被消完了，只剩下基.
  一个定理：主元所在位为1的整数是唯一的,什么意思呢？在相抵标准型中，每一个有用的数都有一个位置(i,j),满足该位置系数非0，第j列的其它位置都是0，第i行的第1~j-1列都是0. 也就是说，在$b_1,b_2,……,b_t$中，有且仅有$b_i$的第$c_i$位为1，其它数的$c_i$位为0.并且j随i变大而递增,也就是说，基底越来越小. 利用这一点可以求得第k小的数.
  根据基的定义，这个数一定可以由基线性表示出来.j是单调递增的，所以选$b_i$一定比不选$b_i$得到的数要大. 将要查询的数k变成转换成二进制数. 若k-1的第i位为0，那么第t-i个基底就要选. (每个基底有选or不选两种可能). 两个二进制数一一对应. 若$k > 2^t$,则无解.
  有一个问题：题目不允许选一个数2次，而异或空间中是可能的. 从$a_i$中选出几个数来异或有可能得到0. 实际上在上述运算中，如果不可能异或出0，那么就直接对k - 1 + 1 = k求.(跳过掉基底异或的结果为0的情况).怎么判断$a_i$能否异或出0呢？看最后的矩阵中是否有一行为0即可. 因为做高斯消元实际上就是每次选不同的两行异或嘛.

总结

  1.xor问题一定要转化为二进制数再去思考.
  2.高斯消元后的主元下标单调性，基与行之间的关系.
  3.xor问题有0的情况.
