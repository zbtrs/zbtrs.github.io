###英文题面
Description

In Frobnia, a far-away country, the verdicts in court trials are determined by a jury consisting of members of the general public. Every time a trial is set to begin, a jury has to be selected, which is done as follows. First, several people are drawn randomly from the public. For each person in this pool, defence and prosecution assign a grade from 0 to 20 indicating their preference for this person. 0 means total dislike, 20 on the other hand means that this person is considered ideally suited for the jury.

###翻译
每个人有属性$a_i$和$b_i$,要求从n个人中选出m个人使得它们的$|Σa_i - b_i|$最小，若有多种选择，应当使$Σa_i + b_i$最大.

###代码

###分析
  很容易看出这是一个背包问题,将每个人的$a_i - b_i$和$a_i + b_i$看作两个维度.维护可行性背包$f_{i,j,k}$,最后枚举j和k的值即可. j与k分别维护a的总和与b的总和.
  这种方法因为$a_i$和$b_i$的总和可能会比较大而TLE. 不过这种做法启发我们：既然我们可以把$a_i$和$b_i$记作状态，最后枚举它们来判断可行性，那么我们也可以记录$Σa_i - b_i$到状态中,最后枚举这一维判断可行性即可. 那么$Σa_i + b_i$怎么维护呢？我们可以将$Σa_i+b_i$看作物品的价值，一开始将f数组初始化为-1,若枚举到的状态的f值不为-1，则说明合法. 根据状态中的差与状态表示的和，即可解出$Σa_i$与$Σb_i$.
  几个问题：
  1.绝对值如何处理？将下标整体移20 * m位即可，将20 * m当作0值.
  2.如何输出方案？经典做法，开相同大小的数组记录当前状态是从哪一个状态转移而来的.
  3.题目要求每个人只能被选一次，如何确保这一点？不能采取逆序枚举的方法，应当递归在路径数组中判断.算是一种新方法吧.

###提示
  由前缀能想到什么？

###代码

###分析
  看到前缀，很容易想到将所有字符串都放到Trie树上处理. 于是每一个单词都对应了一个点. 用一个前缀代替字符串，实际上就是在Trie树中，将点向其祖先方向上挪. 为了使所有前缀互不相同，每个位置最多只能被一个点占据. 问题就转化为了：在一棵树上，可以将一个点向其祖先方向上挪，不能有多个点在同一位置，问最后所有点的深度和最小是多少.
  我一开始的想法是每次挪深度最大的一个点，在Trie树上倍增找到深度最低的没有被占用的点. 这个贪心的想法很容易举出反例来. 但由此很容易想到正确的解法：肯定要将深度最大的点挪到上面去. 这个做法错就错在处理不同子树的处理上，如果先将深度最大的点挪到最上面去，那么就有可能会占据其它子树的位置,所以我们应该把这些子树合起来考虑. 递归，在以u为根节点的子树中找到深度最大的一个点，将其放到根节点上(若根节点为空).然后将这些子树合并. 利用multiset维护,合并两个multiset用启发式合并.(第一次知道原来函数的返回类型可以是STL容器).

###经验
  单单考虑一个点会对其它子树的决策造成影响，解决方法是将所有可能有影响的点合起来一起考虑，在树上多采用自底向上更新的方法.

Codeforces 965 C. Greedy Arkady

发表于 2018-05-19 | 分类于 计划

###题目大意
  k个人，n个糖果. 从1号开始分，每个人分x个糖果，然后下一轮继续这样分，知道糖果总数不足k. 确定x的值，使得第一个人分得的糖果数最多，要求x≤M，并且每个人被分的次数都不应该>D.

###提示
  写出第一个人分得糖果的表达式.

###代码

###分析
  第一个人能分$(k / x+ k - 1) / k$次，每一次分得的糖果数为x,故分得的总糖果数为$(k / x + k - 1) / k * x$. 要在满足限制的条件下使得这个表达式的值最大, 其实并不太好处理. 因为当x变大时，左半部分减小，右半部分增大，总体的值不知道是增大还是减小. 既然不好确定，那么我们就固定其中一部分，利用另一部分的单调性来求最值. 枚举第一个人分糖的次数，也就是左半部分的值. 固定下来后，整体的值随着x的增大而增大，二分x, 只有当左半部分的值等于我们枚举得到的值时才统计进入答案.

###经验
  当有多部分的值同时影响答案时，先固定其中若干部分的取值，再利用单调性or单峰性求另外若干部分的最值，以求得整体的最值. 若这些部分都涉及到同一变量的取值，那么只有在这些值等于枚举的值时才统计答案.

线性dp总结

发表于 2018-05-17 | 分类于 算法竞赛进阶指南

###1.基本处理方法
  1.找维度：限制、个数、长度.
  2.找状态表示：最优值、个数、看问题要求什么.
  3.找决策：刷表or填表.

###2.通用技巧
  1.排序or其它算法解决题目限制.
  2.决策集合的决策有限制条件，并且决策集合的大小只增不减，可以用一个变量维护代表元.
  3.优化状态. 维度上能否互相导出. 若可以则省略掉可以被导出的状态，只需要保留状态的“秩”.
  4.网格图的阶段；行/列，格子.
  5.元素相对大小不变，采用整体加减法，考虑边界来递推. (整体加减法不会影响元素的相对大小)
  6.用与答案数组相同大小的数组表示每一维是从哪一个状态转移过来的，从而可以递归输出方案.

###3.特定技巧
  1.中位数调整法.
  2.凸图形的轮廓线表示法.

#####翻译
给定长度为n的序列A,构造一个长度为N的序列B,满足：
1.B非严格单调. 2.最小化$Σ|A_i - B_i|$

#####代码

#####分析
  一个定理：一定存在一种构造B的方案，使得B中的数值在A中都出现过.
  证明：利用数学归纳法，N=1时显然成立. 假设N = k - 1时，可以构造出序列$B_1 ~ B_{k - 1}$,当N = k时，若$B_k ≤ A_k$,直接令$B_k = A_k$即可. 否则，假设存在一个j,使得$B_j,B_{j + 1},…,B_k$的数值都为v. 根据中位数原理，假设$A_j,A_{j + 1},…,A_k$的中位数为mid,若$mid ≥ B_{j - 1}$,则直接令$B_j,B_{j + 1},…,B_k$都为mid,否则就将这一段与$B_{j - 1}$,那一段合并，求新的中位数. 直到满足条件为止. 这种方法称为中位数调整法.
  那么考虑设计状态：$f_{i,j}$表示A的前i个数中，第i个数选择了B的第j小的数的答案. $f_{i,j} = min(f_{i-1,k},|A_i - A’_j|)$. 决策集合只增不减，依然可以$O(n^2)$处理.

m个饼干给n个人，对于第i个人，若有x个人得到的饼干比它多，那么它会产生$x * g_i$的愤怒值，求怎么分饼干使得这些人的总愤怒值最小.

#####分析
  已经有多少个人获得了饼干、已经发了多少个饼干应该作为阶段，用$f_{i,j}$表示.
  因为每个人得多少块饼干没有限制，而g值有区别，可以想到将所有人按照g值从大到小排序. g值大的分配的饼干就多. 那么排序后，这些人得到的饼干数就是非严格单调递减的. 状态的转移需要知道第i - 1个人的饼干数. 那么就要额外再记录一维，显然不行，考虑一下转化.
  若第i个人获得的饼干数>1,那么可以将所有人的饼干数-1，当前状态从这个状态转移而来. 因为所有人的相对大小不变，所以答案不会变. 否则就枚举从哪一个人开始到第i个人的饼干数都是1. 因为它们的饼干数都是1,就相当于知道了第i-1个人的饼干数，也可以快速确定有多少个人的饼干比它们多. (只有这些人的饼干数都是1).那么状态转移方程就是:$f_{i,j} = min(f_{i,j - i},f_{k,j - (i - k) + kΣg_p})$.
  相对大小不变的问题可以通过缩放到边界1来进行处理.
