poj3585 Accumulation Degree

发表于 2018-05-20 | 分类于 算法竞赛进阶指南 |

英文题面

Description

Trees are an important component of the natural landscape because of their prevention of erosion and the provision of a specific ather-sheltered ecosystem in and under their foliage. Trees have also been found to play an important role in producing oxygen and reducing carbon dioxide in the atmosphere, as well as moderating ground temperatures. They are also significant elements in landscaping and agriculture, both for their aesthetic appeal and their orchard crops (such as apples). Wood from trees is a common building material.

Trees also play an intimate role in many of the world’s mythologies. Many scholars are interested in finding peculiar properties about trees, such as the center of a tree, tree counting, tree coloring. A(x) is one of such properties.

A(x) (accumulation degree of node x) is defined as follows:

Each edge of the tree has an positive capacity.
The nodes with degree of one in the tree are named terminals.
The flow of each edge can’t exceed its capacity.
A(x) is the maximal flow that node x can flow to other terminal nodes.
Since it may be hard to understand the definition, an example is showed below:

A(1)=11+5+8=24
Details: 1->2 11
　 1->4->3 5
　 1->4->5 8(since 1->4 has capacity of 13)
A(2)=5+6=11
Details: 2->1->4->3 5
　 2->1->4->5 6
A(3)=5
Details: 3->4->5 5
A(4)=11+5+10=26
Details: 4->1->2 11
　 4->3 5
　 4->5 10
A(5)=10
Details: 5->4->1->2 10
The accumulation degree of a tree is the maximal accumulation degree among its nodes. Here your task is to find the accumulation degree of the given trees.

Input

The first line of the input is an integer T which indicates the number of test cases. The first line of each test case is a positive integer n. Each of the following n - 1 lines contains three integers x, y, z separated by spaces, representing there is an edge between node x and node y, and the capacity of the edge is z. Nodes are numbered from 1 to n.
All the elements are nonnegative integers no more than 200000. You may assume that the test data are all tree metrics.

Output

For each test case, output the result on a single line. 　

Sample Input

1
5
1 2 11
1 4 13
3 4 5
4 5 10
Sample Output

26
Source

South Central China 2008 hosted by NUDT

一棵树，可以任选一个点作为源点，度数为1的点作为汇点，每条边有容量，求最大流.

提示

  如果给定了源点要怎么做？

代码

分析

  如果给定了源点root,那么只需要树形dp即可求出答案. 即$f_u = Σmin\{f_v,w_{u,v}\}$v是u的子节点. 涉及到换根操作，通常的处理方法是自顶向下再做一次更新. 一开始根是x,将根换成y,那么y的最大流就是y流向其子树的最大流和x流向y的最大流. 开两个数组维护.
  两次更新操作顺序不一样. 第一次是先递归，再用子节点更新.第二次是先更新，再递归. 注意特判度数为1的点.

金字塔

发表于 2018-05-20 | 分类于 算法竞赛进阶指南 |

中文题面

虽然探索金字塔是极其老套的剧情，但是这一队
探险家还是到了某金字塔脚下。经过多年的研究，科
学家对这座金字塔的内部结构已经有所了解。首先，
金字塔由若干房间组成，房间之间连有通道。如果把
房间看做节点，通道看做边的话，整个金字塔呈现一
个有根树结构，节点的子树之间有序，金字塔有唯一
的一个入口通向树根。并且，每个房间的墙壁都涂有
若干种颜色的一种。
探险队员打算进一步了解金字塔的结构，为此，他们使用了一种特殊设计的机器人。这
种机器人会从入口进入金字塔，之后对金字塔进行深度优先遍历。机器人每进入一个房间（无
论是第一次进入还是返回），都会记录这个房间的颜色。最后，机器人会从入口退出金字塔。
显然，机器人会访问每个房间至少一次，并且穿越每条通道恰好两次（两个方向各一次），
然后，机器人会得到一个颜色序列。但是，探险队员发现这个颜色序列并不能唯一确定金字
塔的结构。现在他们想请你帮助他们计算，对于一个给定的颜色序列，有多少种可能的结构
会得到这个序列。由于结果可能会非常大，你只需要输出答案对 $10^9$ 取模之后的值。
输入格式
输入文件包含一行，含有一个字符串，表示机器人得到的颜色序列。
输出格式
输出一个整数表示答案。
样例输入
ABABABA
样例输出
5

提示

  在树上dfs得到的一个序列，能想到什么？

代码

分析

  由在树上dfs形成的一个序列可以想到dfs序. 因为dfs是一个一个子树遍历的，所以一个子树在所形成的序列上是连续的. 由数据范围很容易想到区间dp,即令$f_{l,r}$表示子串S[l,r]对应着多少种可能的金字塔结构. 根据区间dp的一般转移形式，很容易想到$f_{l,r} = Σf_{i,k} * f_{k + 1,j}$.这种转移方式是错的.
  考虑序列ABABABA,两种方案A|BAB|A|B|A和A|B|A|BAB|A是一样的，因为它们都对应一个A延伸出3个B. 为什么会错呢？因为k只是我们枚举的一个分割点, 每一层枚举一个分割点，到最后可能就会有多种决策产生同一种局面，而他们选择的分割点相同，只是选择的次序不同罢了,也就是说：我们考虑的是排列的情况，而题目要求组合的情况. 要去重就必须要在每一层中确定选择几个分割点，并且还要确定这几个分割点的位置. 这样就有点搜索的意味了，会TLE.
  因为题目说子树是有序的，那么我们只需要让每一步的不同决策可能产生的不同局面不同即可.在第一棵子树上下手. 只考虑第一棵子树是由哪些点组成的，将k作为划分点. 如果k不相同，那么第一棵子树的大小也就不相同，整棵树也就不同了，就可以计数了. 在转移的时候，要先考虑只延伸一棵子树的情况. 因为遍历完第一棵子树后要回到根，所以在s[k] = s[r]时才能转移,在转移的时候一定要考虑状态的合法性！

总结

  dp计数问题中，若多种决策会产生同一种最终局面，就必须改变决策，使每一层的决策所导出的局面都是不同的. 转移时要考虑状态的合法性(定义).

poj1179 Polygon

发表于 2018-05-19 | 分类于 算法竞赛进阶指南 |

英文题面

Description

Polygon is a game for one player that starts on a polygon with N vertices, like the one in Figure 1, where N=4. Each vertex is labelled with an integer and each edge is labelled with either the symbol + (addition) or the symbol * (product). The edges are numbered from 1 to N.

On the first move, one of the edges is removed. Subsequent moves involve the following steps:
�pick an edge E and the two vertices V1 and V2 that are linked by E; and
�replace them by a new vertex, labelled with the result of performing the operation indicated in E on the labels of V1 and V2.
The game ends when there are no more edges, and its score is the label of the single vertex remaining.

Consider the polygon of Figure 1. The player started by removing edge 3. After that, the player picked edge 1, then edge 4, and, finally, edge 2. The score is 0.

Write a program that, given a polygon, computes the highest possible score and lists all the edges that, if removed on the first move, can lead to a game with that score.
Input

Your program is to read from standard input. The input describes a polygon with N vertices. It contains two lines. On the first line is the number N. The second line contains the labels of edges 1, …, N, interleaved with the vertices’ labels (first that of the vertex between edges 1 and 2, then that of the vertex between edges 2 and 3, and so on, until that of the vertex between edges N and 1), all separated by one space. An edge label is either the letter t (representing +) or the letter x (representing *).

3 <= N <= 50
For any sequence of moves, vertex labels are in the range [-32768,32767].
Output

Your program is to write to standard output. On the first line your program must write the highest score one can get for the input polygon. On the second line it must write the list of all edges that, if removed on the first move, can lead to a game with that score. Edges must be written in increasing order, separated by one space.
Sample Input

4
t -7 t 4 x 2 x 5
Sample Output

33
1 2
Source

IOI 1998

翻译

一个图，点上标有数字，边上表有运算符号，第一轮选择一条边删掉，然后每次选两个相邻的点，将其合并为1个点，点的数字为两个点原来的数字做所连的边上的运算得到的数字. 问怎样操作能使得最后剩下的那一个数尽量大.

提示

  求最值、环形、数字的数量每次减少1，能想到什么？

代码

分析

  首先这道题很容易看出来要用区间dp做. 对于环，有两种处理方法：要么枚举删掉哪一条边，要么任意选取一条边删掉并复制一份接在后面变成一条长链. 比较好的是第二种方法，因为在这条新链上考虑一条长为n的子链，就相当于割掉了一条边，并且起点已知，不用再额外话$O(n)$的时间去枚举割掉哪条边.
  状态转移方程也很显然，就是令$f_{i,j}$为区间[i,j]的最大值，枚举中间点k,$f_{i,j} = max\{f_{i,k} op f_{k + 1,j}\}$ . 最后枚举起点i为哪一个即可. 事实上这种做法是错误的.因为有负数的情况，该状态设计不具备最优子结构性.那么还要维护最小值，加一维$f_{i,j,0/1}$表示区间[i,j]的最大值/最小值. 分别维护即可.

总结

  环的两种处理方法，以及数的乘除运算求最值问题要考虑负数与0的情况.

poj1015 Jury Compromise

发表于 2018-05-19 | 分类于 算法竞赛进阶指南 |

英文题面

Description

In Frobnia, a far-away country, the verdicts in court trials are determined by a jury consisting of members of the general public. Every time a trial is set to begin, a jury has to be selected, which is done as follows. First, several people are drawn randomly from the public. For each person in this pool, defence and prosecution assign a grade from 0 to 20 indicating their preference for this person. 0 means total dislike, 20 on the other hand means that this person is considered ideally suited for the jury.
Based on the grades of the two parties, the judge selects the jury. In order to ensure a fair trial, the tendencies of the jury to favour either defence or prosecution should be as balanced as possible. The jury therefore has to be chosen in a way that is satisfactory to both parties.
We will now make this more precise: given a pool of n potential jurors and two values di (the defence’s value) and pi (the prosecution’s value) for each potential juror i, you are to select a jury of m persons. If J is a subset of {1,…, n} with m elements, then D(J ) = sum(dk) k belong to J
and P(J) = sum(pk) k belong to J are the total values of this jury for defence and prosecution.
For an optimal jury J , the value |D(J) - P(J)| must be minimal. If there are several jurys with minimal |D(J) - P(J)|, one which maximizes D(J) + P(J) should be selected since the jury should be as ideal as possible for both parties.
You are to write a program that implements this jury selection process and chooses an optimal jury given a set of candidates.

The input file contains several jury selection rounds. Each round starts with a line containing two integers n and m. n is the number of candidates and m the number of jury members.
These values will satisfy 1<=n<=200, 1<=m<=20 and of course m<=n. The following n lines contain the two integers pi and di for i = 1,…,n. A blank line separates each round from the next.
The file ends with a round that has n = m = 0.
Output

For each round output a line containing the number of the jury selection round (‘Jury #1’, ‘Jury #2’, etc.).
On the next line print the values D(J ) and P (J ) of your jury as shown below and on another line print the numbers of the m chosen candidates in ascending order. Output a blank before each individual candidate number.
Output an empty line after each test case.
Sample Input

4 2
1 2
2 3
4 1
6 2
0 0
Sample Output

Best jury has value 6 for prosecution and value 4 for defence:
2 3
Hint

If your solution is based on an inefficient algorithm, it may not execute in the allotted time.
Source

Southwestern European Regional Contest 1996

翻译

每个人有属性$a_i$和$b_i$,要求从n个人中选出m个人使得它们的$|Σa_i - b_i|$最小，若有多种选择，应当使$Σa_i + b_i$最大.

代码

分析

  很容易看出这是一个背包问题,将每个人的$a_i - b_i$和$a_i + b_i$看作两个维度.维护可行性背包$f_{i,j,k}$,最后枚举j和k的值即可. j与k分别维护a的总和与b的总和.
  这种方法因为$a_i$和$b_i$的总和可能会比较大而TLE. 不过这种做法启发我们：既然我们可以把$a_i$和$b_i$记作状态，最后枚举它们来判断可行性，那么我们也可以记录$Σa_i - b_i$到状态中,最后枚举这一维判断可行性即可. 那么$Σa_i + b_i$怎么维护呢？我们可以将$Σa_i+b_i$看作物品的价值，一开始将f数组初始化为-1,若枚举到的状态的f值不为-1，则说明合法. 根据状态中的差与状态表示的和，即可解出$Σa_i$与$Σb_i$.
  几个问题：
  1.绝对值如何处理？将下标整体移20 * m位即可，将20 * m当作0值.
  2.如何输出方案？经典做法，开相同大小的数组记录当前状态是从哪一个状态转移而来的.
  3.题目要求每个人只能被选一次，如何确保这一点？不能采取逆序枚举的方法，应当递归在路径数组中判断.算是一种新方法吧.

题目大意

  给n个不相同的字符串，要求将所有字符串用它的前缀代替，使得这些前缀互不相同，并且长度和最小.

提示

  由前缀能想到什么？

代码

分析

  看到前缀，很容易想到将所有字符串都放到Trie树上处理. 于是每一个单词都对应了一个点. 用一个前缀代替字符串，实际上就是在Trie树中，将点向其祖先方向上挪. 为了使所有前缀互不相同，每个位置最多只能被一个点占据. 问题就转化为了：在一棵树上，可以将一个点向其祖先方向上挪，不能有多个点在同一位置，问最后所有点的深度和最小是多少.
  我一开始的想法是每次挪深度最大的一个点，在Trie树上倍增找到深度最低的没有被占用的点. 这个贪心的想法很容易举出反例来. 但由此很容易想到正确的解法：肯定要将深度最大的点挪到上面去. 这个做法错就错在处理不同子树的处理上，如果先将深度最大的点挪到最上面去，那么就有可能会占据其它子树的位置,所以我们应该把这些子树合起来考虑. 递归，在以u为根节点的子树中找到深度最大的一个点，将其放到根节点上(若根节点为空).然后将这些子树合并. 利用multiset维护,合并两个multiset用启发式合并.(第一次知道原来函数的返回类型可以是STL容器).

经验

  单单考虑一个点会对其它子树的决策造成影响，解决方法是将所有可能有影响的点合起来一起考虑，在树上多采用自底向上更新的方法.

Codeforces 965 C. Greedy Arkady

发表于 2018-05-19 | 分类于 计划 |

题目大意

  k个人，n个糖果. 从1号开始分，每个人分x个糖果，然后下一轮继续这样分，知道糖果总数不足k. 确定x的值，使得第一个人分得的糖果数最多，要求x≤M，并且每个人被分的次数都不应该>D.

提示

  写出第一个人分得糖果的表达式.

代码

分析

  第一个人能分$(k / x+ k - 1) / k$次，每一次分得的糖果数为x,故分得的总糖果数为$(k / x + k - 1) / k * x$. 要在满足限制的条件下使得这个表达式的值最大, 其实并不太好处理. 因为当x变大时，左半部分减小，右半部分增大，总体的值不知道是增大还是减小. 既然不好确定，那么我们就固定其中一部分，利用另一部分的单调性来求最值. 枚举第一个人分糖的次数，也就是左半部分的值. 固定下来后，整体的值随着x的增大而增大，二分x, 只有当左半部分的值等于我们枚举得到的值时才统计进入答案.

经验

  当有多部分的值同时影响答案时，先固定其中若干部分的取值，再利用单调性or单峰性求另外若干部分的最值，以求得整体的最值. 若这些部分都涉及到同一变量的取值，那么只有在这些值等于枚举的值时才统计答案.

  这些限制条件中有一个是显然满足的：资源总和不小于$x_1$,和资源总和不小于$x_2$.关键就是选出$k_1$个资源≥$x_1 / k_1$的服务器. 在此基础上，还要让$k_1$尽量小,因为k越大，可选的数就越多，那么留给第二个任务的选择就更多了. 那么将服务器按照资源数从大到小排序,枚举$k_1$，找到最小的满足要求的. 接着按照相同的方法枚举$k_2$. 最后将两个任务交换，重复这个过程,为了避免出现满足第一个任务却不能满足第二个任务的情况.
