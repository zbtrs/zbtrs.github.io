poj1952 BUY LOW, BUY LOWER

发表于 2018-05-28 | 分类于 算法竞赛进阶指南 |

题目大意

  求最长严格下降子序列的长度及个数，两个子序列不同当且仅当元素不同，下标不是影响因素.

提示

  先考虑标准为：下标不同而序列不同时该怎么计数.伴随更新法.

代码

分析

  如果仅仅考虑下标不同而导致子序列不同，那么这道题就很好做. 统计个数利用伴随更新法，即若从最优解更新，则覆盖掉方案数，若从相同的最优解更新，则累加方案数.
  非常麻烦的一个问题是数值重复要怎么解决？有一点很容易想到：如果元素$a_i$前面有一个相同的元素，那么计数到前一个$a_i$就可以停止计数. 因为它们数值相同，可能造成区别的位置只有它们之间的位置. 对于序列654654,如果从第二个5向左边扩展，计数到第一个5就不继续进行计数了，但是还是会重复计数，因为65会被重复计数两次. 而第二个5的答案是从第二个6的答案转移而来的，于是可以想到:若一个数为最长下降子序列的第一个元素，并且它的前面有与他相同的数，那么这个数的方案数为0，这样用它来更新后面的数，方案数自然就都是0了. 非常奇妙.

总结

  限制枚举顺序，若要求数值上不同，则将所有的计数方案变到左边第一个数上. 利用伴随转移，可以将重复计数的答案赋值为0.

tyvj1124 花店橱窗

发表于 2018-05-28 | 分类于 算法竞赛进阶指南 |

中文题面

题目描述
每种花都有一个标识，假设杜鹃花的标识数为1，秋海棠的标识数为2，康乃馨的标识数为3，所有的花束在放入花瓶时必须保持其标识数的顺序，即：杜鹃花必须放在秋海棠左边的花瓶中，秋海棠必须放在康乃馨左边的花瓶中。如果花瓶的数目大于花束的数目。则多余的花瓶必须空置，且每个花瓶中只能放一束花。

每种花放在不同的瓶子里会产生不同的美观程度，美观程度可能是正数也可能是负数。

上述例子中，花瓶与花束的不同搭配所具有的美观程度，如下表所示：

                         花    瓶

                  1     2    3    4    5
   1 (杜鹃花)     7    23   -5  -24   16
   2 (秋海棠)     5    21   -4   10   23
   3 (康乃馨)    -21    5   -4  -20   20


根据上表，杜鹃花放在花瓶2中，会显得非常好看；但若放在花瓶4中则显得十分难看。

为取得最大美观程度，你必须在保持花束顺序的前提下，使花束的摆放取得最大的美学值，并求出每种花应该摆放的花瓶的编号。

输入格式
第1行：两个整数F和V，表示xq和xz一共有F种花，V个花瓶。(1<=F<=V<=100)
第2行到第F+1行：每行有V个数，表示花摆放在不同花瓶里的美观程度值value。(美观程度和不超过maxint，美观程度有正有负。)
输出格式
输出有两行：第一行为输出最大美观程度和的值，第二行有F个数表示每朵花应该摆放的花瓶的编号。

提示
其实就是简单的DP，花店橱窗问题啦。
注意尽量靠前放啊!
样例数据
输入样例 #1
3 5
7 23 -5 -24 16
5 21 -4 10 23
-21 5 -4 -20 20
输出样例 #1
53
2 4 5

代码

分析

  很简单的dp啦,令$f_{i,j}$表示前i种花放在前j个花瓶的答案. 那么$f_{i,j} = max\{f_{i,j - 1},f_{i-1,j-1} + a_{i,j}\}$. 为了输出方案，再开一个同等大小的数组记录是从i-1转移过来的还是从i转移过来的。

教训

  一定要注意初始化时的边界问题！手推几组小数据看转移是否合法.

Codeforces 959 D. Mahmoud and Ehab and another array construction task

发表于 2018-05-28 | 分类于 计划 |

题目大意

  给定一个数列a,要求构造一个长度和a相同的数列b,满足：
  1.b的字典序≥a
  2.b的每个数≥2
  3.b的每一对数都是互质的.
  4.在满足上述条件的情况下b的字典序最小.

分析

  非常好的一道题目.
  若最优性与可行性结合在一起，那么在可行性上考虑最优性的顺序.什么意思呢？满足可行性肯定是前提. 如果枚举到第i个数，发现前面的第j个数与第i个数不互质，那么这两个数中至少要修改一个才行. 从最优性的顺序来考虑肯定是修改后面的数更好. 那么不断将第i个数变大，直到与前i-1个数满足条件. 如果说可行性限制了一个框架，那么最优性则是在框架中决定方向，寻找最优解.
  其实只需要修改一个数即可，这个数后面的数依次从小到大取没有被占用的质数. 这个时候字典序已经满足要求了. 那为什么要取质数呢？如果取了一个尽可能小的合数，与前面的数不互质，那么前面的数一定不包含这个合数的质因子，那么这个合数就可以用其最小的质因子代替.
  利用标记质因子法实现. 不能每枚举一个数都去和前面的数求gcd. 利用质因子求gcd不等于1的情况的解是一种常用的方法. 记住一点：如果第i个数确定了，才能将这个数的所有质因子打上标记,因为有可能只是在试探性地看这个数能不能填.

Codeforces 959 E. Mahmoud and Ehab and the xor-MST

发表于 2018-05-28 | 分类于 计划 |

题目大意

给定一个n个点的无向完全图，每个点的点权是该点的编号，每条边的边权是所连的两个点的点权的异或值，求最小生成树.

提示

  出数据时一般如何生成一棵树？

代码

分析

  在完全无向图上求最小生成树的方法是：对于第i个点，找第1个点到第i-1个点中，与i异或值最小的点，与i相连. 这样确保了每个点只有一个父亲节点,保证了是一棵树，同时因为取的是异或值最小的点，所以答案是最优的.
  关键问题是怎么计算. 我一开始想放到Trie树中，后来一看n有$10^{12}$,果断放弃. 那么很明显应该是找规律+数学公式计算. 手动打表找规律可以发现：异或值为1数每隔2个出现一次，为2的每隔4个出现一次，为$2^i$的每隔$2^{i+1}$个出现一次，而且答案只会是2的次幂. 那么枚举2的次幂，计算其出现了多少次即可.

Codeforces 959 F. Mahmoud and Ehab and yet another xor task

题目大意

n个数q次询问，每次询问序列的前l个数中，选若干个数出来异或和为x的方案数.

提示

  异或和问题的标准解法是什么？

代码

吐槽

  比赛的时候看错题了，以为是选出的子序列的前l个数……

分析

  标准的线性基计数题. 将前l个数插入到线性基中，若线性基的大小为cnt,并且x能被表示出来，那么答案为$2^{l - cnt}$，否则答案为0. 这是根据可重线性基的结果公式得到的.
  可多次询问，l总在变，那岂不是要每次询问都要将前l个数重新插入到线性基中？ 其实不必，一个数插入过一次后就不用再插了，将所有的询问按照左端点从小到大排序即可.

一类基准点划分计数dp

发表于 2018-05-27 | 分类于 算法竞赛进阶指南 |

方法总结

  划分对象：一个特定的对象所在的集合. 这个对象一般是取最”小”的，即“第一个经过”，“编号最小”等等. 利用dp处理划分出来的这么一个集合,需要考虑各种情况，剩下的直接利用数学公式算，特殊情况不用考虑,如果数学公式不能算，就用dp来算.

划分规律

  划分是呈递进式的，多数题目只需要进行一次划分，但是有的题目要进行多次划分. 一步一步划分，每一次划分在上一次的划分的基础上进行,表示的状态和划分条件越来越严格.

应用

  分析几道例题是如何应用上面的规律的.

1.金字塔

  划分对象：第一棵子树，满足对象最小性.
  划分规律：从整棵树变成了一棵子树，呈现递进式，因为子树可以通过dp计数，所以不需要再次划分.

2.cf559C

  划分对象：第一个经过的黑点.
  划分规律：从不经过黑点到经过黑点. 加上了限定条件.

3.poj1737

  划分对象：第一个节点所在的连通块.
  划分规律：从无向图变成连通块.

4.boss题

题目大意

  求n个点割边不超过m的无向连通图的数量，节点有标号.

分析

  因为多了割边的限制，所以dp有两维状态：$f_{i,j}$表示i个点组成的割边数量为j的无向连通图的数量. 如何划分？基准点肯定是编号最小的点所在的连通块，那么限制条件呢？ 与割边有关并且条件比无向连通图更严苛的是什么？双连通分量！那么就以编号最小的点所在的双连通分量作为基准点来考虑.
  双连通分量内部没有割边，所以只需要枚举划分成的双连通分量的点数即可. 剩下的点的条件就可以放松了，只需要是无向图即可. 需要枚举和划分出来的双连通分量有多少条割边相连. 令$g_{i,j,k}$表示i个点划分成j个连通分量有k条割边的方案数. 那么$f_{i,j} = Σ(f_{k,0}*C(i-1,k-1) *Σg_{i-k,x,j-x}*k^x)$, x是枚举划分出来的双连通分量有多少条割边与外界相连，每条割边有k个点可以选择. 一定要注意：编号最小的点是已经被选出来的，所以组合数上下标都要-1
  同理，计算$g_{i,j,k}$，基准点是编号最小的点所在的连通分量. $g_{i,j,k} = ΣΣf_{p,q} * C(i - 1,p - 1) * p * g_{i - p,j - 1,k - q}$. 注意划分出来的连通分量和一开始划分出来的双连通分量的割边的点的选择也要算入方案数中. 这一步算的是从连通分量选点的方案数.

poj1737 Connected Graph

发表于 2018-05-27 | 分类于 算法竞赛进阶指南 |

题目大意

求n个节点的无向连通图有多少个，节点有标号.

提示

  考虑以什么为基准进行划分来计数.

代码

分析

  连通图不容易计数，考虑计算不连通图的数量，再用总的图的数量减去不连通图的数量即为答案.
  以标号为1的节点所在的连通块为基准进行划分. 令$f_i$表示i个节点的无向连通图的个数，枚举选j个点构成第一个连通子图，那么方案数就是$f_j * C(i - 1,j - 1) * 2^{(i - j)(i - j - 1) / 2}$,即从剩下的i-1个点中选出j-1个点构成标号为1的节点所在的连通块. 剩下的点与第一个连通块都是不连通的，它们之间的连通情况任意. 总的方案数是$2^{i*(i - 1) / 2}$. $f_i = 2^{i*(i - 1) / 2} - Σf_j * C(i - 1,j - 1) * 2^{(i - j)(i - j - 1) / 2}$.
  比较毒瘤的是没有取模操作，需要手写高精度.

poj1037 A decorative fence

发表于 2018-05-27 | 分类于 算法竞赛进阶指南 |

题目大意

求字典序为第C小的波动排列.

提示

  想一想二分法求第k小的数的原理.

代码

分析

  诸如求第k小的具体是哪一个的问题，都采用步步逼近的方法. 不断调整，缩小范围，直到当前答案就是第k小的. 如果答案具有单调性，可以像二分一样从两边向中间缩小范围，否则就要dp求出各种情况的方案数，从单方面缩小范围，就像倍增一样.
  现在的问题就是如何dp处理出各种情况的方案数？这实质上是一个排列计数问题. 因为排列有一个非常好的性质：1到n中所有数仅出现一次，那么我们可以考虑第i个数放在哪个位置而不是第i个位置放哪个数. 后者一般要用状压dp记录当前已经用了哪些数.
   如果考虑第i个数放在哪个位置，是不好处理的，因为如果要这样考虑的话，就要考虑把第i个数插入到某个位置会对答案整体造成什么影响，显然这是不能直接计算的，应该采用递推的形式，即第二种考虑方式.
  第二种方法采用递推的形式，从左往右,依次确定第i个位置放哪一个数. 比较难解决的是数会重复这一问题. 若用状压的话，这一题的N比较大，又是多组数据，有TLE的危险. 正确的方法是改变记录的状态.令$f_{i,j,k}$表示当前已经处理到了第i个位置，第i个位置放的木板是从小到大第j个没被占用的木板，并且状态为k(0代表低位，1代表高位)的方案数. 这样考虑的就是数的相对大小，而且考虑的还是没有占用的数，就不会出现有两位出现相同的数这种情况了. 转移时要特别注意j的取值范围，取决于k和j表示的意义！ $f_{i,j,0} = Σf_{i - 1,p,1}(j≤p≤i-1)$,$f_{i,j,1} = Σf_{i-1,p,0}(1≤p≤j - 1)$.这里要保证k的合法性(比周围小or大)，而j表示的是未被占用的第j小的木板，举几个例子即可得出范围.

总结

  1.求第k小的具体值用逐步逼近法.
  2.排列dp计数问题的2种考虑方式，3种实现方法. 记住什么时候求方案数(一位一位递推)，什么时候求对整体的影响(插到某一个位置).

Codeforces 559 C. Gerald and Giant Chess

发表于 2018-05-27 | 分类于 算法竞赛进阶指南 |

题目大意

有一个h*w的棋盘，棋盘上有n个格子是黑色的，其它都是白色的. 求从左上角走到右下角不经过黑色格子的路径数，每次只能往下或往右走. n较小.

提示

  n比较小，以什么为基准进行计数？

代码

分析

  考虑到w,h巨大，而n很小. 所以对黑点进行计数. 只需要求出从起点到终点的所有路径减去经过黑点的路径即可.
  将终点当作第n+1个黑点. 令$f_i$表示不经过其它黑点到达第i个黑点的方案数. 那么$f_i = C(x_i - 1 + y_i - 1,x_i - 1) - Σf_j * C(x_i - x_j + y_i - y_j,x_i - x_j)$. 为什么满足这个递推式呢？ 递归地考虑状态表示的意义！f表示的是不经过其它黑点并到达当前点的方案数. 要统计不经过其它黑点并到达第i个黑点的方案数就要用总方案数减掉经过其它可能经过的黑点的方案数. 这时$f_j$的含义是从起点走到j,j是第一个经过的黑点的方案数. 这其实就相当于一个基准点. 经过了j则说明该路径一定经过黑点，至于后面怎么走随意. 因为经过的第一个黑点不同，所以对应的路径不同,而我们将所有可能经过的第一个黑点的情况都考虑了，所以路径数被不重不漏地统计了. 很奇妙，和金字塔那道题类似. 都是多种同类型决策，会造成重复计数时，考虑所有可能的“第一个”.
