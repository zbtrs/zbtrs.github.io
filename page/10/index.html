poj2912 Rochambeau

发表于 2018-05-13 | 分类于 算法竞赛进阶指南

###英文题面 Description

N children are playing Rochambeau (scissors-rock-cloth) game with you. One of them is the judge. The rest children are divided into three groups (it is possible that some group is empty). You don’t know who is the judge, or how the children are grouped. Then the children start playing Rochambeau game for M rounds. Each round two children are arbitrarily selected to play Rochambeau for one once, and you will be told the outcome while not knowing which gesture the children presented. It is known that the children in the same group would present the same gesture (hence, two children in the same group always get draw when playing) and different groups for different gestures. The judge would present gesture randomly each time, hence no one knows what gesture the judge would present. Can you guess who is the judge after after the game ends? If you can, after how many rounds can you find out the judge at the earliest?

Input

Input contains multiple test cases. Each test case starts with two integers N and M (1 ≤ N ≤ 500, 0 ≤ M ≤ 2,000) in one line, which are the number of children and the number of rounds. Following are M lines, each line contains two integers in [0, N) separated by one symbol. The two integers are the IDs of the two children selected to play Rochambeau for this round. The symbol may be “=”, “>” or “<”, referring to a draw, that first child wins and that second child wins respectively.

Output

There is only one line for each test case. If the judge can be found, print the ID of the judge, and the least number of rounds after which the judge can be uniquely determined. If the judge can not be found, or the outcomes of the M rounds of game are inconsistent, print the corresponding message.

Sample Input

3 3 0<1 1<2 2<0 3 5 0<1 0>1 1<2 1>2 0<2 4 4 0<1 0>1 2<3 2>3 1 0 Sample Output

Can not determine Player 1 can be determined to be the judge after 4 lines Impossible Player 0 can be determined to be the judge after 0 lines Source

###分析   遇到完全平方数，先将所有数分解质因数. 若一个数为完全平方数，则它的每一个质因子的次数都是偶数.若两个数的乘积是完全平方数，那么这两个数对应的质因子的次数要么同时是奇数，要么同时是偶数. 因为要求乘积的质因子的次数为偶数，若当前这个数的某个质因子的次数已经是偶数了，我们就不需要考虑这个质因子的影响,只需要考虑它的次数为奇数的质因子的影响.   将每个数换一种表示方法，比如180 = 2*2*3*3*5,把次数为偶数的质因子去掉，就变成了5. 对于一个已经确定的集合，该集合最少被分成的子集数=有多少种不同的表达方式. 将所有数的表达方式都存到一个数组中，离散化一下. 因为n只有5000，可以枚举连续段的左右端点,固定左端点，枚举右端点，同时统计答案.   需要考虑负数和0的情况. 负数的表示方法一定带一个负号，若连续的一段数只包含0，则最少划分成一个子集，若包含其他的数，0可以忽略！ ###经验教训   一开始想着把所有的表示方法状压一下，看来还是too young too simple. 感觉可以把这个表示方法理解成hash函数，将状压转换成hash,效率高很多.

1)$.   5.若d为n的约数，那么$Σφ(d) = n$.   因为$gcd(n,x) = gcd(n,n-x)$,即与n不互质的数成对出现，平均值为$n/2$，可以证明性质1，利用性质1可以求与一个数不互质的数的和.   利用第2,3,4条性质，可以在线性筛中求出$φ(n)$的值,性质二的证明可以利用上面提到的积性函数的求值方法：将a,b分解质因数. 性质3可以通过公式展开直接证明. 性质4则利用性质2证明.   对于第5条性质,令$g(n) = Σφ(d)$,证明$g(n)$是积性函数，利用积性函数的求值方法即可证明. 如何证明$g(n)$是积性函数？只需要证明$g(n)g(m) = g(nm)$即可，其中n,m互质. ###4.费马小定理的证明   费马小定理：若p是质数，对于任意整数a,有$a^p = a （mod p)$   由费马小定理可以导出欧拉定理$a^{φ(p)} = 1(mod p)$,由于$p$是质数， 所以$φ(p) = p - 1$,证明了欧拉定理即可证明费马小定理.   利用简化剩余系来证明. 若${a_1,a_2,...,a_{φ(n)}}$与${aa_1,aa_2,...,aa_{φ(n)}}$都能表示模n下的简化剩余系，那么$a^{φ(n)}a_1a_2...a_{φ(n)} = (aa_1)(aa_2)......(aa_{φ(n)}) = a_1a_2...a_{φ(n)}(modn)$,也就是将上述两个集合分别乘起来,因为a与n互质，所以a可以消掉，就得到了上面的定理，同时也证明了欧拉定理. 这一步利用了一个关键的性质：模n意义下，与n互质的数不影响整除性，故a可以消掉.   上面的结论都是在这两个集合是等价的前提下推导出来的，下面来证明这两个集合是等价的. 我们只需要两个集合的元素是一一对应的即可. 对于n原来的简化剩余系，若有$a_i,a_j$,$aa_i = aa_j(modn)$,那么a可以消掉，得到$a_i = a_j(modn)$. 当$a_i ≠a_j$ 时，$aa_i,aa_j$一定对应着不同的同余类. $a_i对应aa_i$,两个集合大小相等，那么这两个集合就是等价的. ###5.裴蜀定理的证明   先证明边界情况：b = 0时,x = 1,y = 0. 再对欧几里得算法的递归过程应用数学归纳法. 若$bx + (amodb)y = gcd(b,amodb)$利用余数的展开式，对式子进行变形，令$x' = y,y' = x - ###分析   sg函数的吼题啊！   我一开始的想法类似于二进制状压，将每一堆的魔法珠的数量都给存到一个状态中. 这样造成的后果是后继状态极多，状态不好表示，而且不好转移和还原. 没有想到每一堆魔法珠都是独立的....... 单独考虑每一堆魔法珠. 将其每一种决策的后继所有状态的异或值放到一个集合中，最后求该集合的mex,就能得到sg值了.   之前xor和mex一直傻傻分不清. 做完这题才算是有了一点新的认识. 单独考虑每一种决策，若一个决策能够分成若干个子问题，就将这些子问题的sg值异或起来，放到mex集合中. 也就是说：mex集合的大小取决于决策的个数,mex对应决策,xor对应将所有的子问题合并.

###中文题面 Description 　　打开了黑魔法师Vani的大门，队员们在迷宫般的路上漫无目的地搜寻着关押applepi的监狱的所在地。突然，眼前一道亮光闪过。“我，Nizem，是黑魔法圣殿的守卫者。如果你能通过我的挑战，那么你可以带走黑魔法圣殿的地图……”瞬间，队员们被传送到了一个擂台上，最初身边有一个容量为K的包包。 　　擂台赛一共有N项挑战，各项挑战依次进行。第i项挑战有一个属性ai，如果ai>=0，表示这次挑战成功后可以再获得一个容量为ai的包包；如果ai=-1，则表示这次挑战成功后可以得到一个大小为1 的地图残片。地图残片必须装在包包里才能带出擂台，包包没有必要全部装满，但是队员们必须把 【获得的所有的】地图残片都带走（没有得到的不用考虑，只需要完成所有N项挑战后背包容量足够容纳地图残片即可），才能拼出完整的地图。并且他们至少要挑战成功L次才能离开擂台。 　　队员们一筹莫展之时，善良的守卫者Nizem帮忙预估出了每项挑战成功的概率，其中第i项挑战成功的概率为pi%。现在，请你帮忙预测一下，队员们能够带上他们获得的地图残片离开擂台的概率。

Input 　　第一行三个整数N,L,K。 　　第二行N个实数，第i个实数pi表示第i项挑战成功的百分比。 　　第三行N个整数，第i个整数ai表示第i项挑战的属性值.

Output 　　一个整数，表示所求概率，四舍五入保留6 位小数。

Sample Input 样例输入1

3 1 0

10 20 30

-1 -1 2

样例输入2

5 1 2

36 44 13 83 63

-1 2 -1 2 1

Sample Output 样例输出1

0.300000

样例输出2

0.980387

HINT 　　若第三项挑战成功，如果前两场中某场胜利，队员们就有空间来容纳得到的地图残片，如果挑战失败，根本就没有获得地图残片，不用考虑是否能装下；若第三项挑战失败，如果前两场有胜利，没有包来装地图残片，如果前两场都失败，不满足至少挑战成功次（）的要求。因此所求概率就是第三场挑战获胜的概率。

对于 100% 的数据，保证0<=K<=2000，0<=N<=200，-1<=ai<=1000，0<=L<=N，0<=pi<=100。

###分析   挺好的一道题.   很容易想到这道题用概率dp来做. 如何设计状态？ 对答案有影响的变量是：当前进行到了第几项挑战、有几项挑战成功了、当前背包容量、收集了多少个地图残片. 如果将它们都保存在状态中，内存和时间都会爆. 考虑优化状态. 地图的数量是优先的，最多只会有n个，而背包容量可能会非常大，如果能不保存这一维就好了. 题目要求最后背包的容量>地图残片个数. 那么可以记录一个收益值，遇到背包，则加上其对应的容量，遇到地图就-1，最后只要检查收益值是否大于等于0即可.那么状态就设计为了$f_{i,j,k}$表示当前进行到了第i项挑战，有j项成功了，收益值为k的概率.   一些细节问题需要注意：收益值可能会出现负数，因此将收益值整体+n，那么以前的0收益值就对应了现在的n, 因为地图残片数量最多只会有n个，所以当k ≥ 2n时，可以将k看作2n,同理，k ≤ 0时，可以将其看作0. dp是由当前已知状态去更新位置状态，用刷表法. 内存可能会炸,(刚好开成200*200*400就不会炸)，利用滚动数组优化. 因为是用刷表法，并不是仅仅交换两个指针就完了. 要将转移到的指针的f数组给清空. 刷表法的滚动数组优化和填表法的还是有区别的.

###分析   直接计数gcd = 1的四元组的数量很难. 一个思路是求gcd != 1的四元组数量，用总数量减掉它就是答案了. 求gcd != 1的n元组问题有一个很经典的解法：容斥原理. 枚举所有可能出现在$a_i$的质因子p,求gcd为p的倍数的四元组数量. 可以先求出有多少个数是p的倍数，再利用组合公式可以计算出来. 可能会有四元组同时被p和q计数，那么就要减掉gcd同时是p和q的倍数的四元组的数量. 以此类推，奇+偶-即可. 容斥原理要分析清楚把状态空间分为哪几种集合，有哪些集合可能相交.找相交关系，不要找包含关系！

###中文题面 描述 在一片广袤无垠的原野上，散落着N块磁石。每个磁石的性质可以用一个五元组(x,y,m,p,r)描述，其中x,y表示其坐标，m是磁石的质量，p是磁力，r是吸引半径。若磁石A与磁石B的距离不大于磁石A的吸引半径，并且磁石B的质量不大于磁石A的磁力，那么A可以吸引B。 小取酒带着一块自己的磁石L来到了这篇原野的(x0,y0)处，我们可以视为磁石L的坐标为(x0,y0)。小取酒手持磁石L并保持原地不动，所有可以被L吸引的磁石将会被吸引过来。在每个时刻，他可以选择更换任意一块自己已经获得的磁石（当然也可以是自己最初携带的L磁石）在(x0,y0)处吸引更多的磁石。小取酒想知道，他最多能获得多少块磁石呢？

输入格式 第一行五个整数x0,y0,pL,rL,N，表示小取酒所在的位置，磁石L磁力、吸引半径和原野上散落磁石的个数。 接下来N行每行五个整数x,y,m,p,r，描述一块磁石的性质。

输出格式 输出一个整数，表示最多可以获得的散落磁石个数（不包含最初携带的磁石L）。

样例输入 0 0 5 10 5 5 4 7 11 5 -7 1 4 7 8 0 2 13 5 6 2 -3 9 3 4 13 5 1 9 9 样例输出 3 数据范围与约定 对于30%的数据，1<=N<=1000。 对于100%的数据，1<=N<=250000，-10^9<=x,y<=10^9，1<=m,p,r<=10^9。 ###提示   二维偏序问题怎么处理？ ###代码

###分析   首先可以想到基本的算法框架：建立一个队列，每次取出队首磁石，将能吸到的磁石全部放到队列里，最后统计队列中的元素个数. 关键问题就是如何快速判断哪些磁石能够被吸引.   本题中，磁石吸引条件包含两个维度：质量≤磁力、距离≤吸引半径.这是一个二维偏序问题. 通常的解法是排序+树状数组，但是这只能统计数量，我们想要知道构成二维偏序的是具体的哪一对. 两种解法：1.分块. 2.平衡树. 先对一维进行排序，再对另一维操作.   这道题比较简单的方法是分块. 处理方法是整体排序+块内排序.先将所有的磁石按照质量从小到大排序. 再将每一块内部的磁石按照距离从小到大排序. 在排序之前要记录下每一块内的磁石的最大质量. 当每一块的最大质量被确定下来后，若当前队首磁石的磁力≥当前块内的最大质量，那么这一个块内的磁石的质量对答案就没有影响了，只需要考虑距离. 而块内是按照距离从小到大排序的，可以对每一个块维护一个单调指针,这些单调指针一共最多会移动n步. 可以直接把每一个块的左端点当作指针.   若当前队首磁石的磁力≤当前块内的最大质量，那么只能暴力统计了. 将符合要求的放进队列中. 注意：这一个块的单调指针不会变动，也不需要重新排序. 因为拿走一些元素不影响已经排好的顺序. 为了避免将已入队的元素重复入队，使用vis数组标记.. ###套路   分块处理二维偏序问题，题目要求知道具体的值. 利用整体排序+块内排序的方法.

###分析   知道了矩形的长和宽，只需要知道矩形的一个边角的坐标即可确定这个矩形的位置. 为了避免坐标出现负数的情况，我们选取右上角.   对于一个星星，矩形的右上角在一个固定的区域内就能够覆盖这颗星星.边界情况比较难处理. 如果边界上不能有星星，那么这个固定的区域对应的横坐标和纵坐标的取值就是开区间，坐标可能取到浮点数. 那么将所有的星星往左移x,下移y$(0 <x,y<1)$,那么矩形顶点的坐标就是整数了(否则我们可以改变x,y的大小，将矩形顶点的坐标调整为整数). 因此，矩形右上角能取到的区域的左下角的点为(x,y),右上角的点为(x + w - 1,y + h - 1).   对每一个星星求得这么一个矩形. 这些矩形都有权值(星星的亮度). 矩形重叠的部分的权值是重叠的这些矩形的权值和. 现在的任务就是求权值和最大的一个区域.做到这已经不难想到用扫描线去求了，线段树维护最大值即可. ###总结   不要陷入思维的死胡同. 星星是产生贡献的，故从星星的角度去考虑，而不是考虑矩形应该放在哪.和hdu4864有异曲同工之妙.   若干矩形堆在一起求某个值是扫描线的应用.   矩形边界情况的处理.
