###分析   非常牛逼的一道题!   题目想让我们干什么？计数！有什么方法？暴力数学dp法！   先尝试数学法. 计数的步骤是先计算合法的相对排名的方案数，再计算在该排名下分数的方案数.排名可以用式子表示出来，但是无法计算在特定排名下分数的方案数.   那么尝试使用dp. 先定义状态$f_{i,j}$表示前i个科目中,有j个人被碾压的方案数. 考虑如何从$f_{i,j}$转移到$f_{i+1,k}$,会发现转移不过来. why？因为被碾压的条件太苛刻了，只考虑当前的科目是不能确定这$k-j$是否被碾压. 换一个状态表示：令$f_{i,j}$表示前i个科目中，有j个人不被碾压的方案数. 那么从$f_{i,j}$转移到$f_{i+1,k}$就可以只考虑第i个科目对人的影响. 令$t_1 = k - j$,表示从被碾压的人中选k个人变成新的不被碾压的人(第i个科目的分数比B高). 而第i个科目有$R_i - 1$个人不被B碾压，剩下的$t_2 = R_i - 1 - t_1$个人就要从之前不被B碾压的人中选. (不能从被碾压的人中选，否则会使得k变大,矛盾). 方案数就是$C(n - j - 1,t_1) * C(j,t_2)$. 如果令科目i的所有人得分的方案数为$s_i$,那么$f_{i + 1,k} = f_{i,j} * C(n - j - 1,t_1) * C(j,t_2) * s_i$.   下面来说一下$s_i$要怎么计算. 因为那$R_i - 1$个人的分数只要高于B的就好了，具体是多少分没有限制，剩下的人的分数要低于B的分数，只需要确定B的分数，就能得到方案数. 枚举B的分数x,则方案数为$Σ(U_i - x)^{R_i - 1} * x^{n - R_i} (1 ≤ x ≤ U_i)$, 因为$U_i$太大不好算. ** 事实上这是一个n-1次的多项式求和，答案一定可以表示为不超过n次的一个多项式. n+1个点即可确定一个n次多项式的唯一表示，利用拉格朗日插值法.**   先算出$y_1$到$y_{n+1}$的所有值，若中途碰到了上界$U_i$,返回，因为x是连续的，预处理出lsum和rsum. $lsum_i = lsum_{i - 1} * (U - i)$,rsum类似. 然后预处理出逆元的阶乘. 若$(n + 1 - i) %2 == 1$,则计算出来的答案是负数，需要取相反数.（至于为什么，看代码实现）. ###总结   理一下基本思路、为什么要这么做、怎么样更好地实现？   1.为什么dp?   因为数学式子无法在计算排名的情况下计算分数的方案数.   2.为什么状态的第二维定义为不能被超越的人数？   因为被超越的人的定义太苛刻了，不能只通过一个科目的分数来决定.   关于拉格朗日插值法：要求一个多项式的值，多项式的次数很小，但是要求的那一项很大，就可以利用拉格朗日插值法. 需要熟悉求和多项式可以用单个多项式来表示，n+1个点确定一个n次多项式，都涉及到次数的变化.   对于x连续的n次多项式，可以利用预处理前缀+后缀信息来快速插值.

前言

  花了5天时间把这一章搞完了，学习了一些数据结构的巧妙用法. ###1.栈与队列   普通的用法是个人都会. 但有一些比较高端的做法，就是用若干个栈or队列来维护信息.   1.如果要维护整体信息，这些数据结构分别维护某一段的信息，若将它们“拼”起来，则能得到整体的信息比如说求中位数，开两个对顶堆，每个堆维护一半，正好对应了一整段序列;光标的左右移动，开两个栈，左右移则分别对应栈顶元素的变化.   2.维护历史信息（或者说是需要维护的信息要保存历史信息）,用一个相同的数据结构来维护，两个数据结构同时操作比如说要维护一个栈，支持求最小值/最大值. 则另开栈维护最小值/最大值，插入一个元素x,即插入min(x,栈顶元素)或max(x,栈顶元素),弹出一个元素则对应栈也要弹出元素. 查询的时候直接输出栈顶元素即可.   3.用若干个低级数据结构来替代高级数据结构，可以优化时间复杂度例如noip2016蚯蚓. 常见的就是队列替代优先队列. ###2.链表   有一类问题可以用平衡树或堆来做，链表也可以轻松解决. 这类问题通常涉及到多次排序(用平衡树或堆的原因)，多次询问，能够时光倒流,用链表处理的原理是我们可以只需要对所有操作结束后的局面排一次序，利用链表容易执行删除操作的特性来进行时光倒流.   比如说：每次往一个序列中添数求中位数，我可以先把所有的数都添加进去，排好序，找到当前的中位数后删除，再来找下一个中位数，整个序列一直都是有序的. ###3.Hash   Hash分为两种,一种是字符串Hash,一种是Hash表. Hash表的Hash函数是由变量的特征决定的.你可以选择把特征值加起来mod一个数，也可以先乘后加再mod,由题目而定. 举个例子：一个雪花六个角，每个角都有一个数，要判断两个雪花能不能重合，设计的Hash函数就不能仅仅是将这六个数加起来，因为可能有两个雪花它们的六个数加起来相等，但实际不一样. 因此Hash函数应设计为：六个数的和+六个数的积%mod. mod一般取内存允许的范围内尽量大的质数，因为时间复杂度与n / 表头长度有关   字符串Hash就是对一段序列，一个区间进行Hash. 我个人认为它有一种线性递推的感觉. 可以将字符串Hash看作是一个特殊的Hash函数，它几乎不可能重复! 一般形式是：$hash[i] = (hash[i - 1] * p + a[i]) %mod$, 为了方便处理，将$hash$开成unsigned long long类型的，这样就不必mod了. P一般取131或13331.   利用字符串hash的递推公式，我们可以算出任意一段子串的hash值. 方法是：先求出前缀子串的hash值$f[i] = f[i - 1] * P + (S[i] - "a" + 1)$,任意区间的hash值为$f[r] - f[l - 1] * P^{r - l + 1}$. ###4.最小表示法   一个循环同构字符串，可以用它的最小表示法唯一表示出来. 如果两个循环同构字符串的最小表示法一样，那么这两个字符串是相等的. 如何求最小表示法？   1.破环成链，先将原字符串复制一份.   2.初始化i=1,j = 2.   通过向后扫描的方法，比较$B[i],B[j]$两个循环同构串，这里$B[i]$的意思是从第i个位置的n个字符表示的字符串.   如果扫描了n个字符后仍然相等，说明S只由1种字符构成，任意$B[i]$都是他的最小表示.   如果在i+k与j+k处发现不相等:   若$S[i + k] > S[j + k]$,令$i = i + k + 1$,若此时$i = j$,则令$i = i + 1$.   另一种情况也这么处理,变动的指针是j.   若$i > n$,则$B[j]$为最小表示，否则$B[i]$为最小表示. ###5.Trie   当看到有一个数/字符串对多个数/字符串询问时，想到将这多个被询问的数/字符串放到Trie上.   Trie的一个经典应用是xor,求任意两个数xor的最值、和之类的问题. 为什么强调两个？有的题目会让你选两个点，求这两个点所组成的区间的xor的值，因为xor具有相抵性，把“每个点表示当前点的数值“变成”每个点表示到当前点的前缀的数值“即可. ###6.哈夫曼树   详情见noi2015荷马史诗 ###7.习题   大多数习题都写了单独的题解，唯有一道题：括号画家. ####大致题意   有3中括号(,[,{,求一段最长的合法的括号序列. ####分析：   判断是否合法很简单，如何求最长的呢？维护一个栈，如果是左括号，则添加进栈中，如果是右括号：若栈顶的括号与之匹配，则说明栈顶到目前为止构成了一个合法的括号序列，更新答案，栈顶元素的左边可能有合法的括号序列能够连起来，取一个max即可. ####代码
