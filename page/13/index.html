CH Round

发表于 2018-05-12 | 分类于 算法竞赛进阶指南 |

中文题面

描述
在一片广袤无垠的原野上，散落着N块磁石。每个磁石的性质可以用一个五元组(x,y,m,p,r)描述，其中x,y表示其坐标，m是磁石的质量，p是磁力，r是吸引半径。若磁石A与磁石B的距离不大于磁石A的吸引半径，并且磁石B的质量不大于磁石A的磁力，那么A可以吸引B。
小取酒带着一块自己的磁石L来到了这篇原野的(x0,y0)处，我们可以视为磁石L的坐标为(x0,y0)。小取酒手持磁石L并保持原地不动，所有可以被L吸引的磁石将会被吸引过来。在每个时刻，他可以选择更换任意一块自己已经获得的磁石（当然也可以是自己最初携带的L磁石）在(x0,y0)处吸引更多的磁石。小取酒想知道，他最多能获得多少块磁石呢？

输入格式
第一行五个整数x0,y0,pL,rL,N，表示小取酒所在的位置，磁石L磁力、吸引半径和原野上散落磁石的个数。
接下来N行每行五个整数x,y,m,p,r，描述一块磁石的性质。

输出格式
输出一个整数，表示最多可以获得的散落磁石个数（不包含最初携带的磁石L）。

样例输入
0 0 5 10 5
5 4 7 11 5
-7 1 4 7 8
0 2 13 5 6
2 -3 9 3 4
13 5 1 9 9
样例输出
3
数据范围与约定
对于30%的数据，1<=N<=1000。
对于100%的数据，1<=N<=250000，-10^9<=x,y<=10^9，1<=m,p,r<=10^9。

提示

  二维偏序问题怎么处理？

代码

分析

  首先可以想到基本的算法框架：建立一个队列，每次取出队首磁石，将能吸到的磁石全部放到队列里，最后统计队列中的元素个数. 关键问题就是如何快速判断哪些磁石能够被吸引.
  本题中，磁石吸引条件包含两个维度：质量≤磁力、距离≤吸引半径.这是一个二维偏序问题. 通常的解法是排序+树状数组，但是这只能统计数量，我们想要知道构成二维偏序的是具体的哪一对. 两种解法：1.分块. 2.平衡树. 先对一维进行排序，再对另一维操作.
  这道题比较简单的方法是分块. 处理方法是整体排序+块内排序.先将所有的磁石按照质量从小到大排序. 再将每一块内部的磁石按照距离从小到大排序. 在排序之前要记录下每一块内的磁石的最大质量. 当每一块的最大质量被确定下来后，若当前队首磁石的磁力≥当前块内的最大质量，那么这一个块内的磁石的质量对答案就没有影响了，只需要考虑距离. 而块内是按照距离从小到大排序的，可以对每一个块维护一个单调指针,这些单调指针一共最多会移动n步. 可以直接把每一个块的左端点当作指针.
  若当前队首磁石的磁力≤当前块内的最大质量，那么只能暴力统计了. 将符合要求的放进队列中. 注意：这一个块的单调指针不会变动，也不需要重新排序. 因为拿走一些元素不影响已经排好的顺序. 为了避免将已入队的元素重复入队，使用vis数组标记..

套路

  分块处理二维偏序问题，题目要求知道具体的值. 利用整体排序+块内排序的方法.

翻译

每个星星都有亮度，用一个长为w,宽为h的矩形圈星星，要求圈的星星的亮度总和最大. 位于矩形边界上的星星不算.

分析

  已经知道了矩形的长和宽，还需要知道什么就能确定一个矩形？

代码

分析

  知道了矩形的长和宽，只需要知道矩形的一个边角的坐标即可确定这个矩形的位置. 为了避免坐标出现负数的情况，我们选取右上角.
  对于一个星星，矩形的右上角在一个固定的区域内就能够覆盖这颗星星.边界情况比较难处理. 如果边界上不能有星星，那么这个固定的区域对应的横坐标和纵坐标的取值就是开区间，坐标可能取到浮点数. 那么将所有的星星往左移x,下移y$(0 <x,y<1)$,那么矩形顶点的坐标就是整数了(否则我们可以改变x,y的大小，将矩形顶点的坐标调整为整数). 因此，矩形右上角能取到的区域的左下角的点为(x,y),右上角的点为(x + w - 1,y + h - 1).
  对每一个星星求得这么一个矩形. 这些矩形都有权值(星星的亮度). 矩形重叠的部分的权值是重叠的这些矩形的权值和. 现在的任务就是求权值和最大的一个区域.做到这已经不难想到用扫描线去求了，线段树维护最大值即可.

总结

  不要陷入思维的死胡同. 星星是产生贡献的，故从星星的角度去考虑，而不是考虑矩形应该放在哪.和hdu4864有异曲同工之妙.

bzoj3105 [cqoi2013]新Nim游戏

发表于 2018-05-12 | 分类于 算法竞赛进阶指南 |

中文题面

Description
传统的Nim游戏是这样的：有一些火柴堆，每堆都有若干根火柴（不同堆的火柴数量可以不同）。两个游戏者轮流操作，每次可以选一个火柴堆拿走若干根火柴。可以只拿一根，也可以拿走整堆火柴，但不能同时从超过一堆火柴中拿。拿走最后一根火柴的游戏者胜利。
本题的游戏稍微有些不同：在第一个回合中，第一个游戏者可以直接拿走若干个整堆的火柴。可以一堆都不拿，但不可以全部拿走。第二回合也一样，第二个游戏者也有这样一次机会。从第三个回合（又轮到第一个游戏者）开始，规则和Nim游戏一样。
如果你先拿，怎样才能保证获胜？如果可以获胜的话，还要让第一回合拿的火柴总数尽量小。

Input
第一行为整数k。即火柴堆数。第二行包含k个不超过109的正整数，即各堆的火柴个数。

Output

输出第一回合拿的火柴数目的最小值。如果不能保证取胜，输出-1。
Sample Input
6

5 5 6 6 5 5

Sample Output
21
HINT
k<=100

提示

  在普通的nim游戏中，先手必胜的条件是什么？

代码

分析

  尽量让先手取胜. 若第一轮操作后，存在若干堆石子，使得它们的数量的异或和为0，则先手必输. 因为后手可以取出其它的石子使得异或和变为0，则变成必输局面. 线性基中，无论取多少个数异或起来都不会等于0，故求出石子个数最多的线性基即可.按照石子个数从大到小排序，动态维护线性基即可.

翻译

解一个模线性方程组.

代码

分析

  解方程组一般都是用高斯消元解. 在模意义下高斯消元其实和在一般情况下高斯消元是差不多的。区别就是多一个取模运算，要考虑负数的情况. 在消元完后判断解的情况. 最后还原. 还原实际上就是在解一个模线性方程，利用扩展欧几里得算法求一组等于gcd的特解，然后求出最小正整数通解即可.
  这道题使我又掌握了一种新的高斯消元的模板. 以前写过3类模板：
  1.一定有解且仅有一组解的方程组.直接消就好了.
  2.贪心+高斯消元，不改变方程的位置,类似线性基的插入操作.
  3.不一定有且仅有一组解的xor方程组. 每次找到最大的那个数，找到其最高位，将其它的这一位为1的数与之异或即可.
  现在不是xor方程组，而且解的情况可能不唯一，于是换上一种新的模板：记录当前扫到了哪一行，扫到了哪一列. 不管扫到哪一列，一定要将当前行给消掉。为了保证方程组的系数都是整数，乘到最小公倍数来消元.

翻译

求一个矩阵的等比数列和.

分析

  求等比数列可以利用公式，也可以分治. 在本题中采用分治的方式. 一些问题需要注意：
  1.尽量减少不必要的取模.
  2.能不用long long就尽量不用long long.

After N turns the paper will be broken into N+1 pieces, and in the later turn the players can choose any piece to cut. If one player cuts out a piece of paper with a single grid, he wins the game. If these two people are both quite clear, you should write a problem to tell whether the one who cut first can win or not.
Input

The input contains multiple test cases. Each test case contains only two integers W and H (2 <= W, H <= 200) in one line, which are the width and height of the original paper.
Output

For each test case, only one line should be printed. If the one who cut first can win the game, print “WIN”, otherwise, print “LOSE”.

2 2
3 2
4 2
Sample Output

LOSE
LOSE
WIN

翻译

一张n*m的矩形网格纸，两名玩家轮流行动。在每一次行动中，可以任选一张矩形网格纸，沿着某一行或者某一列的格线，把它剪成两部分。首先剪出1*1的玩家获胜.问先手能否必胜.

代码

分析

  很显然是要用sg来做. 但是这道题和一般的博弈论的题不太一样. 这道题是谁不能动谁就赢了，而利用sg函数求解的博弈论的题要求谁不能动谁就输了. 需要对问题进行转化.
  遇到这种情况，我们先找到必败的边界状态，将其转化为题目的边界状态.比如这道题中，若剪成了2*2,2*3,3*2,那么接下来行动的人必败. 那么这三种情况的sg值为0.
  如何算sg值？ 根据原则多游戏异或，后继求mex,即$sg(n,m) = mex(\{sg(i,m) \^ sg(n-i,m)\} U \{sg(n,i) \^ sg(n,m - i)\})$.
  这么做可能会有问题：在dfs计算sg值的时候可能会走到边界1,1,这样就会得到错误的答案. 为了防止这种情况发生，强制让dfs走到新边界而不是原边界,只需要对决策做一点点改动即可.
