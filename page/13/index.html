poj1830 开关问题

发表于 2018-05-08 | 分类于 算法竞赛进阶指南

中文题面

Description

有N个相同的开关，每个开关都与某些开关有着联系，每当你打开或者关闭某个开关的时候，其他的与此开关相关联的开关也会相应地发生变化，即这些相联系的开关的状态如果原来为开就变为关，如果为关就变为开。你的目标是经过若干次开关操作后使得最后N个开关达到一个特定的状态。对于任意一个开关，最多只能进行一次开关操作。你的任务是，计算有多少种可以达到指定状态的方法。（不计开关操作的顺序）
Input

输入第一行有一个数K，表示以下有K组测试数据。
每组测试数据的格式如下：
第一行 一个数N（0 < N < 29）
第二行 N个0或者1的数，表示开始时N个开关状态。
第三行 N个0或者1的数，表示操作结束后N个开关的状态。
接下来 每行两个数I J，表示如果操作第 I 个开关，第J个开关的状态也会变化。每组数据以 0 0 结束。
Output

如果有可行方法，输出总数，否则输出“Oh,it’s impossible~!!” 不包括引号
Sample Input

2
3
0 0 0
1 1 1
1 2
1 3
2 1
2 3
3 1
3 2
0 0
3
0 0 0
1 0 1
1 2
2 1
0 0
Sample Output

4
Oh,it’s impossible~!!
Hint

第一组数据的说明：
一共以下四种方法：
操作开关1
操作开关2
操作开关3
操作开关1、2、3 （不记顺序）
Source

LIANGLIANG@POJ

提示

  列出每盏灯初始状态到终止状态的关系式.

代码

分析

  对于第i个开关，可以列出方程（以第1个为例）：
$a_{1,1}x_1  xor  a_{2,1}x_2  xor …… xor  a_{n,1}x_1 = s_1  xor  t_1$
  其中$a_{i,j} = 1$表示操作开关i对开关j有影响. $x_i$表示第i个开关有没有被操作. $s_i$表示第i盏灯的初始状态, $t_i$表示终止状态. 特殊地，我们令$a_{i,i} = 1$
  n元n个方程，挺好的. 题目没有说一定会存在唯一解，可能会出现无解或是多解的情况. 一般情况下，若有m个方程n个未知数，将方程看作系数向量+结果向量，若最大线性无关组的大小为k,则存在n-k个自由元. 若存在方程0 = x(x不为0)，则方程无解. 方程的解的个数和自由元的个数有关 在本题中，自由元可以任意取值0或1，若有cnt个自由元，则答案为$2^{cnt}$. 利用高斯消元求解.
  因为每个系数只有0,1两种取值，故可以用二进制数存储,当前扫到了第i行，若$sta_i = 0$,说明出现了自由元，答案为$2^{n - i + 1}$,若$sta_i = 1$,说明无解. 利用二进制数可以快速判断方程左边与右边的关系，并且能够快速按位异或，不用逐位操作. 高斯消元的具体步骤是找一个二进制表示下最大的数，找到它的最高位k,将其他的第k位不为0的数异或上这个最大的数. 类似普通方程的高斯消元，只不过加减法变成了异或.

总结

  1.遇到的新题型—xor方程组. 利用每一个变量的初始状态，最终状态，以及对它有影响的变量列方程. 方程的左边枚举j,对i是否有影响取决于是否操作j以及如果操作j是否会对i有影响. 方程的右边就是初始状态xor最终状态.
  2.二进制表示状态的方法很值得学习.
  3.高斯消元的一般步骤：找最高位不为0的，让其它最高位变成0，判断无解和自由元的情况.

bzoj2973 石头游戏

发表于 2018-05-08 | 分类于 算法竞赛进阶指南

中文题面

Description
石头游戏的规则是这样的。
石头游戏在一个n行m列的方格阵上进行。每个格子对应了一个编号在0~9之间的操作序列。
操作序列是一个长度不超过6且循环执行、每秒执行一个字符的字符串。它包括：
数字0~9：拿0~9个石头到该格子。
NWSE：把这个格子内所有的石头推到相邻的格子。
D：拿走这个格子的石头。
石头游戏的问题是：当这个石头游戏进行了t秒之后，所有方格中最多的格子有多少个石头。
注意：所有格子的操作同时执行。
Input
第一行三个整数n, m, t, act。
接下来n行，每行m个字符，表示每个格子对应的操作序列。
最后act行，每行一个字符串，表示从0开始的每个操作序列。
Output
一个整数：游戏进行了t秒之后，所有方格中最多的格子有多少个石头。
Sample Input
1 6 10 3

011112

1E

E

0

Sample Output
3

【样例解释】

这是另一个类似于传送带的结构。左边的设备0间隔地产生石头并向东传送。设备1向右传送，直到设备2。10秒后，总共产生了5个石头，2个在传送带上，3个在最右边。

【数据约定】

0≤n, m≤8。

1≤act≤10。

1≤t≤10^9。

提示

  1.每个点在固定时刻转移是相同的，由此能想到什么？
  2.t这么大，一般用什么方法优化转移？

代码

分析

  很显然是矩阵快速幂优化转移. 但是时间不同，转移矩阵就不唯一，不太好处理. 幸运的是每一个转移操作的长度最大为6，也就是说最多过60s,所有点的转移方式都会和之前某一秒的相同.
  遗憾的是我总是想着如何构造一个转移矩阵. 一开始想的是每一秒的转移矩阵该怎么构造，发现转移并不总是不变的，于是又想怎么构造一个矩阵表示60s的转移，还是不好构造.死活就想不到正解上去.
  正解是构造60个矩阵……分别对应第0~59s的转移. 状态矩阵的行数为1，列数为n*m+1,转移矩阵的行数和列数均为n*m+1, 相当于把n*m个元素放到一个序列上编号，状态矩阵增加了第0列，转移矩阵增加了第0行第0列，为了方便给某个位置增加石头. 令状态矩阵为ans,转移矩阵为t. 一开始$ans_{0,0} = 1$，其它元素为0. $t_{0,0} = 1$. $id(i,j)$表示位置为$(i,j)$的点的序号具体的构造如下：
  1.若在第T秒，第i行第j列的操作是一个数x,那么$t_{id(i,j),id(i,j)} = 1$,并且$t_{0,id(i,j)} = x$. 因为$ans_{0,0} = 1$,所以0这个点为给$(i,j)$x个石头.
  2.若操作是一个方向,以N为例，则$t_{id(i,j),id(i-1,j)} = 1$,其它的情况类似.
  那还有一种操作D怎么处理呢？其实不需要处理.因为下一秒$(i,j)$的点的石头数取决于转移到它的石头数. 如果$t_{id(i,j),id(i,j)} = 0$，那么它的石头是不会被保留的. 而一个点的3种类型的决策，只有决策为数字的点的石头才可能被保留. 所以不处理就相当于处理了.

总结

  矩阵快速幂的思维不能仅仅只局限在构造单一转移矩阵上. 可能会有多个转移矩阵. 一般的应用情形是转移次数多，但是转移方式有一个周期，周期很小.
  构造转移矩阵可以通过加行、列，实现对某一个位置的单独修改!

英文题面

Description

Chinese people think of ‘8’ as the lucky digit. Bob also likes digit ‘8’. Moreover, Bob has his own lucky number L. Now he wants to construct his luckiest number which is the minimum among all positive integers that are a multiple of L and consist of only digit ‘8’.

Input

The input consists of multiple test cases. Each test case contains exactly one line containing L(1 ≤ L ≤ 2,000,000,000).

The last test case is followed by a line containing a zero.

Output

For each test case, print a line containing the test case number( beginning with 1) followed by a integer which is the length of Bob’s luckiest number. If Bob can’t construct his luckiest number, print a zero.

Sample Input

8
11
16
0
Sample Output

Case 1: 1
Case 2: 2
Case 3: 0
Source

2008 Asia Hefei Regional Contest Online by USTC

翻译

类似88888这样的数，由n个8组成，求最少的n使得这个数是L的倍数.

提示

  写出式子，化简得到一个简单的同余式.

代码

分析

  先写出式子： $L | 8(10^x - 1) / 9$,将9乘过去: $9L | 8(10^x - 1)$.
  接下来怎么化简？不知道. 现在这个式子也无法求解，因为x的取值范围太大. 我们需要做的是将x的取值范围缩小. 需要先知道一个结论：
  若$a^x \% p == 1$,则x的最小取值一定是$φ(p)$的约数.证明如下：
  $φ(p) = kx + r$,$a^{kx + r} \% p == 1$则$a^{kx} * a^r \%p == 1$,而$a^{kx} \% p == 1$,所以$a^r \% p == 1$.若$r > 0$,$r$完全可以变成0来使得$x$更小，所以$r = 0$,成立.
  原式子中有$10^x - 1$，若将$-1$移到同余式的右边，则刚好可以用上这个结论.关键是如何消掉式子中的8.
  还有一个结论：互质的两个数不影响整除性，可以忽视其中一个数,可$9L$与$8$不一定互质，怎么办？ 左右两边同时除以$(9L,8)$即可，令$d = (9L,8)$,左边的$9L / d$与右边的$8 / d$互质，可以将$8 / d$拿走. 原式子变成了$(9L / d) | (10^x - 1)$. 可以利用上面的结论了.
  满心欢喜敲完代码交上去，发现WA了. Why? 因为对$9L / d$取模，模数可能大于int存储范围, 而快速幂可能就是两个大于int存储范围的数相乘，会爆long long,正确的方法是将快速幂中的乘法改成加法.

总结

  1.如何在整除式中越掉一个乘积？提公因数！
  2.一个扩展结论:模m意义下，与m互质的数对于整除性没有影响，也就是说：若$ac = bc (mod  m),(c,m) == 1—->a = c(mod  m)$
  3.模一个数等于1的次数最小值的取值范围.

算法竞赛进阶指南第三章

发表于 2018-05-08 | 分类于 算法竞赛进阶指南

前言

  历时5天终于干掉了这一章. 码量之巨，技巧之多，无不使人见之而走.

1.bitset的应用

  一般用于可达性统计. 想要统计一张图上某一个点对于其它点的到达情况，仅仅只关心它能否到达，可以用bitset统计. 如果自己用二进制状压在n很大的情况下是写不了的.

2.剪枝方法

  1.每次找分支最少的状态去搜，也就是找“最难符合要求”的状态去搜.
  2.若搜索的多种决策会导向同一种状态，应确定决策的先后顺序. 若一种决策的后继状态包含了另一种决策的后继状态，可以只选择前一种决策.
  3.可行性剪枝.
  4.最优化剪枝.
  5.记忆化，若能记忆化最好写成递推的形式(dp).

3.边权为0,1的BFS

  通过双端队列广搜来使得队列中的状态满足步数单调性. 若决策转移的代价为0，则将扩展状态放到队首，否则放到队尾.

4.两种DFS类型

  1.可行性dfs,返回类型为bool型.
  2.最优性dfs,要求出一个最优解.
  这两种dfs的区别是剪枝和回溯的不同. 对于可行性dfs,没有最优性剪枝. 若失败了才回溯（还原历史状态）. 对于最优性dfs,两种剪枝都可以有，并且不论成功与否，都要还原历史状态.

5.迭代加深搜索与DFS的关系

  1.迭代加深搜索实质上就是可行性dfs.
  2.与迭代加深搜索功能相对应的就是最优性dfs.

6.搜索方法的选择

  1.优先考虑最优化搜索算法(A*,IDA*).
  2.若期望步数比较小，状态能存下.优先选择BFS类搜索.
  3.若期望步数较大，决策单一，优先选择DFS类搜索.
  4.若搜索树有的节点深度非常大（搜不到底，或者要搜特别多次），选择迭代加深类搜索.
  5.若能优化(BFS优化成优先队列BFS,优先队列BFS优化成A*,迭代加深搜索优化成IDA*),选择优化算法.

分析

代码

分析

  先选定搜索方法. DFS肯定是不可以的，很容易走到死胡同. 很显然要用广搜.
  数字P比较大，决策也比较多，直接搜肯定会TLE. 考虑优化算法A*,维护一个优先队列，每次取出当前操作数+估值最小的状态扩展.
  如何设计估价函数？ 利用快速幂算法，想让一个数尽可能快地接近另一个数，肯定是不断地复制它自己. 所以对于一个状态(x,y),假设较大者是y,找到最小的整数k,使得$2^k * y > P$,k就是估价值.
  A*算法需要判重，因为x,y比较大，不能直接用一个二维vis数组存储，利用hash表来判重，设计hash函数$H(x,y) = (x + y + x*y) \% mod$. 模数选一个比较大的7位素数.
  这样就完了？交上去会MLE. 优先队列占用了太多的内存, 如果输出一下优先队列的大小，会发现最大数据中大小会达到80w,直接用STL中的优先队列是过不了的，要手写堆.(毒瘤！)
  然后会发现TLE. woc?为什么都这样了还会TLE? 可以加上一些剪枝：
  1.若$(x,y)$不整除$p$,说明当前状态不合法.
  2.根据第一条剪枝，若x,y中的任何一个等于0，则gcd会陷入死循环中，由此可以想到另一条剪枝：若$x = y = 0$,一定不可能最优
  3.一个不容易想到的剪枝.若$x = y$,当前状态一定不可能最优. 怎么说呢，与其让x,y一样，不如让更大者更大，更小者更小. 根据快速幂算法原理，这样是更优的.
  终于，卡时1000ms过了，真tm毒瘤！

分析
