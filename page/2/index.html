bzoj1791 [ioi2008]Island 岛屿

发表于 2018-06-13 | 分类于 算法竞赛进阶指南 |

中文题面

Description
你将要游览一个有N个岛屿的公园。从每一个岛i出发，只建造一座桥。桥的长度以Li表示。公园内总共有N座桥。尽管每座桥由一个岛连到另一个岛，但每座桥均可以双向行走。同时，每一对这样的岛屿，都有一艘专用的往来两岛之间的渡船。 相对于乘船而言，你更喜欢步行。你希望所经过的桥的总长度尽可能的长，但受到以下的限制。 • 可以自行挑选一个岛开始游览。 • 任何一个岛都不能游览一次以上。 • 无论任何时间你都可以由你现在所在的岛S去另一个你从未到过的岛D。由S到D可以有以下方法： o 步行：仅当两个岛之间有一座桥时才有可能。对于这种情况，桥的长度会累加到你步行的总距离；或者 o 渡船：你可以选择这种方法，仅当没有任何桥和/或以前使用过的渡船的组合可以由S走到D（当检查是否可到达时，你应该考虑所有的路径，包括经过你曾游览过的那些岛）。 注意，你不必游览所有的岛，也可能无法走完所有的桥。 任务 编写一个程序，给定N座桥以及它们的长度，按照上述的规则，计算你可以走过的桥的最大长度。 限制 2 <= N <= 1,000,000 公园内的岛屿数目。 1<= Li <= 100,000,000 桥i的长度。

分析

  这个题目描述，看的人真是捉急……
  只有两个连通块之间才能用渡船，否则只能步行. 注意到每个点都只会连出一条边，是基环树模型. 对每个连通块单独处理，总答案为所有连通块的答案和.
  怎么处理一个连通块上的答案？基环树要先把环给找出来，一般方法是拓扑排序. 对于无向图，只需要把入队的判定条件改成度数为1即可. 求出环上每个子树内部的答案，以及每条支链的最大值.现在就是要在环上找到两个点，使得$d_i + d_j + dist(i,j)$最大. 将环拆成链，枚举i,j是有范围限制的，利用单调队列优化.
  注意环的处理和数组之间的转换关系，在单调队列内不要再用d数组了，用a数组.还要注意二元环这种特殊情况!

区间与点匹配问题小结

发表于 2018-06-13 | 分类于 算法竞赛进阶指南 |

前言

  晚上想了一下一道题的贪心解法，结果想到很多与之类似的题，特地来记录一下.

概况

  这类题通常是给定若干个区间，若干个点，可能是无限个点. 两种类型：要么是点覆盖区间，要么是区间覆盖点，主要看最后是点对答案产生贡献还是区间对答案产生贡献.

1.一对一匹配

  一个区间只能和一个点匹配，并且点和区间都是给定的. 常用方法是对区间按照一个端点排序，然后贪心地选点. 或者直接上网络流.(复杂度略高).因为点匹配区间和区间匹配点是对称问题，下面从区间匹配点来考虑.

区间贡献为1型

  有若干个区间和若干个点，只有当点在某个区间内时，点才能和这个区间匹配，求最多有多少个区间能被匹配.

分析

  将所有的区间按照左端点从大到小排序，在不超过右端点的点中找到一个最靠右的点与该区间匹配. 根据决策包容性可以证明这种贪心方法是正确的. 大致原理是：左端点是从大到小排序的，当前选的点一定在以后所有区间的左端点的右边，左端点的条件被满足了，那么要满足右端点的条件. 显然，当前区间的决策越差越好，前提是要能满足当前区间的限制条件，那么留给后来的区间能选的点就更多了.
  出处：poj3614.

区间贡献不为1型

分析

  显然，若区间贡献为负数，那么这个区间可以不要. 剩下的可以跑最大费用最大流.

2.一对多匹配

  一对多匹配指的是：一个数在一个区间内部，那么就相当于这个数匹配了这个区间,因为一个区间可以匹配多个数，所以要对区间的容量进行讨论.

区间无容量型

  区间无容量指的是一个区间内只要有一个数即可,必须满足所有区间内至少有一个数，最少要用多少个数?

分析

  将所有区间按照左端点从小到大排序，记录一个变量pos. 一开始$pos = r_1$ ,扫到第i个区间时，若$pos ≥ l_i$,那么该区间可以被覆盖，同时更新$pos = min\{pos,r_i\}$,否则$pos = r_i$,就要多用一个点.
  还是贪心算法，原则是让尽量多的区间共用一个点.
  出处：poj1328.

区间有容量型

  每个区间内至少要有$c_i$个数，必须满足所有区间的要求，问最少要用多少个数？

分析

  将区间按照右端点从小到大排序，每次在第i个区间的右端点放数，直到第i个区间满足要求.
  这样做的原理是：一个个满足区间的要求，并且尽可能地对其他区间产生贡献.
  上面这道题好像也可以用这种方法，树状数组记录一下即可.
  出处：poj1201.

区间有代价型

  给点区间和点，每个点选or不选，选了第i个点，那么所有包含它的区间的贡献都会被计算，每个区间的贡献最多被计算一次，求最大贡献.

分析

  将点从小到大排序，用vector存每个点能被哪些区间覆盖. 令$f_i$表示前i个点，第i个点一定被选的最大贡献. $f_i$从$f_j$转移过来，代价是能覆盖i但是不能覆盖j的区间的贡献和. 增加一个虚拟点n+1,位置为无穷远，最后的答案为$f_{n + 1}$.

经典dp模型

  区间和点都必须是整点，点有无数个. 一般的状态表示为$f_{i,j}$表示前i个区间，右端点最远能延伸到j的答案. 分大小区间讨论j的变化来转移.
  详见：https://www.cnblogs.com/zbtrs/p/7751257.html

3.未解之谜—有容量有代价型

  说一下问题吧：一个数对应一个区间，一个区间至少有$c_i$个数时才会有贡献.
  或者是一对多,一个区间包含$c_i$个数时才会产生贡献. 想不出什么好的方法.

Byteotia城市有n个 towns m条双向roads. 每条 road 连接 两个不同的 towns ,没有重复的road. 所有towns连通。

Sample Output
8

8

16

提示

写出答案的表达式.

代码

分析

  最后答案是删掉所有边后形成的连通块大小两两相乘累加得到的结果.
  如果x不是割点，那么只有x与其它n-1个点不连通，所以答案为$2*(n - 1)$.
  如果x是割点，那么删掉x的所有边后，原图就会形成若干个连通块. 若x有k条边，那么最多就会有k+2个连通块,这些连通块有三种类型：
  1.x自身构成一个连通块.
  2.某些x的子节点v所在的搜索树的子树构成的连通块.
  3.其余点.
  考虑如何才能形成第二类连通块. 只有当$low_v ≥ pre_x$时,才行，这时v以及其子树内所有点都不能到x之前的点,就是一个独立的连通块. 如果$low_v < pre_x$呢？可以证明，所有这样的v及其子树形成一个连通块. 因为题目给的图是连通图，如果存在两个点$v_1,v_2$,它们连到x之前的点不连通，那么其中一个点一定在x之后被递归到，然而它们的pre值都小于x的pre值，矛盾.
  那么就比较好做了，在tarjan时记录下以每个点为根的子树的大小，对于第二类子节点，贡献是$size_v * (n - size_v)$,令$sum = Σsize_v$,那么第三类点的贡献就是$(n - sum - 1) * (sum + 1)$.

小结

  删除割点后形成连通块的种类要记住.

英文题面

Description

A network administrator manages a large network. The network consists of N computers and M links between pairs of computers. Any pair of computers are connected directly or indirectly by successive links, so data can be transformed between any two computers. The administrator finds that some links are vital to the network, because failure of any one of them can cause that data can’t be transformed between some computers. He call such a link a bridge. He is planning to add some new links one by one to eliminate all bridges.

You are to help the administrator by reporting the number of bridges in the network after each new link is added.

Input

The input consists of multiple test cases. Each test case starts with a line containing two integers N(1 ≤ N ≤ 100,000) and M(N - 1 ≤ M ≤ 200,000).
Each of the following M lines contains two integers A and B ( 1≤ A ≠ B ≤ N), which indicates a link between computer A and B. Computers are numbered from 1 to N. It is guaranteed that any two computers are connected in the initial network.
The next line contains a single integer Q ( 1 ≤ Q ≤ 1,000), which is the number of new links the administrator plans to add to the network one by one.
The i-th line of the following Q lines contains two integer A and B (1 ≤ A ≠ B ≤ N), which is the i-th added new link connecting computer A and B.

The last test case is followed by a line containing two zeros.

For each test case, print a line containing the test case number( beginning with 1) and Q lines, the i-th of which contains a integer indicating the number of bridges in the network after the first i new links are added. Print a blank line after the output for each test case.

代码

分析

  自始至终都感觉这道题非常奇怪.
  给出的图是一个连通图，这就比较好处理，每次加一条边，就可能将一个连通块变成一个双连通分量，那么用并查集来维护双连通分量. 在搜索树上处理，若加一条边(u,v)，则相当于将搜索树上u到v的路径上的桥全部去掉了，路径上所有的点合并成一个双连通分量，利用并查集进行合并. 注意合并不改变树的结构，只是在求LCA的时候可以一下子跳到连通块的顶部.
  细节：不要认为搜索树的深度是pre数组，要在tarjan算法中求每个点在搜索树中的深度. 求LCA也要注意，要先跳一边，再跳一边，最后一起跳.

小结

  合并操作不要改变树的形态!不光对这道题而言，对于所有的题，修改操作都不要随意改变数据结构的结构.

noip2012 疫情控制

发表于 2018-06-12 | 分类于 算法竞赛进阶指南 |

题目大意

一棵树上有若干个特殊位置，每个位置可以四处挪动，费用为边权，求最小费用使得1号点到所有叶子节点的位置上都有一个特殊位置.

代码

分析

  这道题充分体现了我是一个sb这一事实.
  算法是很容易想到的,每个点肯定是尽量往上跳，但是跳到根节点后的策略就有很多种，不好确定总时间. 对于确定具体数值的题，可以利用二分求解. 这道题的答案也具有二分性. 如果一个时间刚刚可以满足要求，那么更多的时间一定可以满足要求，更少的时间一定不能满足要求.
  考虑这么一个贪心策略：每个点在有限的时间内尽量往上跳，如果不能够跳到根节点，就停在能跳到的深度最小的节点，否则就跳到根节点的对应的儿子节点上. 接下来把有点存在并且不是根节点的儿子节点的点打上标记，从下往上更新，如果一个节点有儿子节点并且所有的儿子节点都打上了标记，那么这个点也打上标记. 这个过程实际上就是把标记上传：如果一个点的所有儿子都被占领了，那么这个点也一定被占领了.
  接下来考虑根节点的所有儿子节点. 如果一个儿子节点没有被打上标记，并且到达它的军队走到根节点再回到该点不能在剩下的时间内完成，那么这个点就留在原地，否则加入到数组中. 记录一个二元组(s,rest)，是根节点的哪一个儿子节点和剩下的时间. 将根节点的所有没有打上标记的儿子节点提出来，按照它们到根节点的边权大小从小到大排序，并将二元组按照rest从小到大排序，对应着分配. 如果最后根节点的儿子节点都能被分配到一个点，那么二分的答案可行，否则不可行.
  需要注意贪心算法各个过程的先后顺序：是先打标记，再上传，然后处理能跳到根节点的军队.

一些sb错误

  1.倍增的时候先跳点，再计算代价……这里一定要注意先后顺序！
  2.上传标记的时候发现一个儿子节点不合法就直接返回了… 一定要搜完！

1 2
2 3

3 4

代码

分析

Description
设T=(V, E, W) 是一个无圈且连通的无向图（也称为无根树），每条边带有正整数的权，我们称T为树网（treenetwork），其中V, E分别表示结点与边的集合，W表示各边长度的集合，并设T有n个结点。 路径：树网中任何两结点a,b都存在唯一的一条简单路径，用d(a,b)表示以a,b为端点的路径的长度，它是该路径上各边长度之和。我们称d(a,b)为a,b两结点间的距离。 一点v到一条路径P的距离为该点与P上的最近的结点的距离： d(v，P)=min{d(v，u)，u为路径P上的结点}。 树网的直径：树网中最长的路径称为树网的直径。对于给定的树网T，直径不一定是唯一的，但可以证明：各直径的中点（不一定恰好是某个结点，可能在某条边的内部）是唯一的，我们称该点为树网的中心。 偏心距ECC(F)：树网T中距路径F最远的结点到路径F的距离，即 。 任务：对于给定的树网T=(V, E,W)和非负整数s，求一个路径F，它是某直径上的一段路径（该路径两端均为树网中的结点），其长度不超过s（可以等于s），使偏心距ECC(F)最小。我们称这个路径为树网T=(V,E,W)的核（Core）。必要时，F可以退化为某个结点。一般来说，在上述定义下，核不一定只有一个，但最小偏心距是唯一的。 下面的图给出了树网的一个实例。图中，A-B与A-C是两条直径，长度均为20。点W是树网的中心，EF边的长度为5。如果指定s=11，则树网的核为路径DEFG（也可以取为路径DEF），偏心距为8。如果指定s=0（或s=1、s=2），则树网的核为结点F，偏心距为12。
Input
包含n行： 第1行，两个正整数n和s，中间用一个空格隔开。其中n为树网结点的个数，s为树网的核的长度的上界。设结点编号依次为1, 2, …, n。 从第2行到第n行，每行给出3个用空格隔开的正整数，依次表示每一条边的两个端点编号和长度。例如，“2 4 7”表示连接结点2与4的边的长度为7。 所给的数据都是正确的，不必检验。
Output
只有一个非负整数，为指定意义下的最小偏心距。
Sample Input
5 2

1 2 5

2 3 2

2 4 4

2 5 3

Sample Output
5
HINT
对于70%的数据，n<=200000
对于100%的数据：n<=500000, s<2^31, 所有权值<500

代码

分析

  原题$O(n^3)$就可以水过，不知道还有$O(n)$的算法，今天看了一看，真tm厉害.
  做这道题需要大胆猜测一个性质：在一个直径上求得的答案等于在所有直径上求得的答案. 假设直径的两个端点为p,q，可以枚举核，在直径的每个点上都做一次dfs,求出最远距离，取一下max就得到了偏心距,时间复杂度$O(n^3)$.
  继续分析，答案取决于核外的点到核上的任意一个点最小距离. 这是针对核外的某个点而言的. 为了使得答案尽量小，肯定是核越长越好，所以枚举核只需要枚举一个端点即可.时间复杂度$O(n^2)$
  题目要求的实质是让最大值最小，很容易想到二分. 二分偏心距的最大值，怎么判断可行性呢？判断的过程肯定是越简单越好，如果还是一个一个点去求偏心距，那么就不需要二分了. 需要利用到直径的性质. 假设核靠近p的一端是u,因为(p,q)是直径，所以(p,u)上延伸出去的分支到u的距离一定不大于p到u的距离. 否则p就不可能是直径的端点. 那么只需要枚举u，v的位置可以通过维护单调指针得到，判断两个端点到u,v的距离的最大值即可.时间复杂度$O(nlogsum)$
  还没完，根据上述分析，可以知道答案一定是以下几种情况：
  1.p,q到u,v的距离.
  2.(p,q)上的点能延伸到的最远距离.
  枚举u,v的位置是单调递增的，那么可以用单调队列维护.时间复杂度$O(n)$.
  还没完，之前维护的是(p,q)上的点能延伸到的最远距离. 事实上，根据(p,u)上延伸出去的分支到u的距离一定不大于p到u的距离这一性质，(p,q)之外的点能延伸到的最远距离不可能成为答案，因为最后取max,可以将直径上所有的点能延伸到的最远距离归为第二种情况. 就不需要用单调队列维护了. 只需要求直径的两个端点到核的两个端点的距离即可.
  其实我觉得正常人应该能一下子想到最后一步，写出答案的表达式即可. 只是直径的最长性可能会有点困难想到.
