poj3417 Network

发表于 2018-06-12 | 分类于 算法竞赛进阶指南 |

英文题面

Description

Yixght is a manager of the company called SzqNetwork(SN). Now she’s very worried because she has just received a bad news which denotes that DxtNetwork(DN), the SN’s business rival, intents to attack the network of SN. More unfortunately, the original network of SN is so weak that we can just treat it as a tree. Formally, there are N nodes in SN’s network, N-1 bidirectional channels to connect the nodes, and there always exists a route from any node to another. In order to protect the network from the attack, Yixght builds M new bidirectional channels between some of the nodes.

As the DN’s best hacker, you can exactly destory two channels, one in the original network and the other among the M new channels. Now your higher-up wants to know how many ways you can divide the network of SN into at least two parts.

Input

The first line of the input file contains two integers: N (1 ≤ N ≤ 100 000), M (1 ≤ M ≤ 100 000) — the number of the nodes and the number of the new channels.

Following N-1 lines represent the channels in the original network of SN, each pair (a,b) denote that there is a channel between node a and node b.

Following M lines represent the new channels in the network, each pair (a,b) denote that a new channel between node a and node b is added to the network of SN.

Output

Output a single integer — the number of ways to divide the network into at least two parts.

Sample Input

4 1
1 2
2 3
1 4
3 4
Sample Output

3
Source

POJ Monthly—2007.10.06, Yang Mu

代码

分析

  先把树建出来，可以发现新加的M条边是树上的返祖边，与树上的若干条边形成一个环. 如果砍掉环内的边，则第二步必须要砍掉这条返祖边，方案唯一. 如果一条树边被多个环包围，那么这条边是不能砍的. 如果一条边没有被任何环包围，那么砍掉这条边后可以任选一条返祖边砍掉. 因为每次都对一条链修改，可以利用树上差分算法解决. 即$f_x++$,$f_y++$,$f_{LCA(x,y)} -= 2$.

bzoj1999 [Noip2007]Core树网的核

发表于 2018-06-12 | 分类于 算法竞赛进阶指南 |

中文题面

Description
设T=(V, E, W) 是一个无圈且连通的无向图（也称为无根树），每条边带有正整数的权，我们称T为树网（treenetwork），其中V, E分别表示结点与边的集合，W表示各边长度的集合，并设T有n个结点。 路径：树网中任何两结点a,b都存在唯一的一条简单路径，用d(a,b)表示以a,b为端点的路径的长度，它是该路径上各边长度之和。我们称d(a,b)为a,b两结点间的距离。 一点v到一条路径P的距离为该点与P上的最近的结点的距离： d(v，P)=min{d(v，u)，u为路径P上的结点}。 树网的直径：树网中最长的路径称为树网的直径。对于给定的树网T，直径不一定是唯一的，但可以证明：各直径的中点（不一定恰好是某个结点，可能在某条边的内部）是唯一的，我们称该点为树网的中心。 偏心距ECC(F)：树网T中距路径F最远的结点到路径F的距离，即 。 任务：对于给定的树网T=(V, E,W)和非负整数s，求一个路径F，它是某直径上的一段路径（该路径两端均为树网中的结点），其长度不超过s（可以等于s），使偏心距ECC(F)最小。我们称这个路径为树网T=(V,E,W)的核（Core）。必要时，F可以退化为某个结点。一般来说，在上述定义下，核不一定只有一个，但最小偏心距是唯一的。 下面的图给出了树网的一个实例。图中，A-B与A-C是两条直径，长度均为20。点W是树网的中心，EF边的长度为5。如果指定s=11，则树网的核为路径DEFG（也可以取为路径DEF），偏心距为8。如果指定s=0（或s=1、s=2），则树网的核为结点F，偏心距为12。
Input
包含n行： 第1行，两个正整数n和s，中间用一个空格隔开。其中n为树网结点的个数，s为树网的核的长度的上界。设结点编号依次为1, 2, …, n。 从第2行到第n行，每行给出3个用空格隔开的正整数，依次表示每一条边的两个端点编号和长度。例如，“2 4 7”表示连接结点2与4的边的长度为7。 所给的数据都是正确的，不必检验。
Output
只有一个非负整数，为指定意义下的最小偏心距。
Sample Input

1 2 5

2 3 2

2 4 4

2 5 3

Sample Output

HINT
对于70%的数据，n<=200000
对于100%的数据：n<=500000, s<2^31, 所有权值<500

代码

分析

  原题$O(n^3)$就可以水过，不知道还有$O(n)$的算法，今天看了一看，真tm厉害.
  做这道题需要大胆猜测一个性质：在一个直径上求得的答案等于在所有直径上求得的答案. 假设直径的两个端点为p,q，可以枚举核，在直径的每个点上都做一次dfs,求出最远距离，取一下max就得到了偏心距,时间复杂度$O(n^3)$.
  继续分析，答案取决于核外的点到核上的任意一个点最小距离. 这是针对核外的某个点而言的. 为了使得答案尽量小，肯定是核越长越好，所以枚举核只需要枚举一个端点即可.时间复杂度$O(n^2)$
  题目要求的实质是让最大值最小，很容易想到二分. 二分偏心距的最大值，怎么判断可行性呢？判断的过程肯定是越简单越好，如果还是一个一个点去求偏心距，那么就不需要二分了. 需要利用到直径的性质. 假设核靠近p的一端是u,因为(p,q)是直径，所以(p,u)上延伸出去的分支到u的距离一定不大于p到u的距离. 否则p就不可能是直径的端点. 那么只需要枚举u，v的位置可以通过维护单调指针得到，判断两个端点到u,v的距离的最大值即可.时间复杂度$O(nlogsum)$
  还没完，根据上述分析，可以知道答案一定是以下几种情况：
  1.p,q到u,v的距离.
  2.(p,q)上的点能延伸到的最远距离.
  枚举u,v的位置是单调递增的，那么可以用单调队列维护.时间复杂度$O(n)$.
  还没完，之前维护的是(p,q)上的点能延伸到的最远距离. 事实上，根据(p,u)上延伸出去的分支到u的距离一定不大于p到u的距离这一性质，(p,q)之外的点能延伸到的最远距离不可能成为答案，因为最后取max,可以将直径上所有的点能延伸到的最远距离归为第二种情况. 就不需要用单调队列维护了. 只需要求直径的两个端点到核的两个端点的距离即可.
  其实我觉得正常人应该能一下子想到最后一步，写出答案的表达式即可. 只是直径的最长性可能会有点困难想到.

提示

  在树上加一条边会怎样？不加边的计算公式和加边后的计算公式分别是什么样的？本质是在统计什么？

代码

分析

  如果不加边，那么任意一条边都会被经过两次，模拟dfs的递归过程，可以发现每条边都要被遍历一次和回溯一次，所以所有边一定会被经过两次.
  如果加一条边，就相当于把一条链变成了一个环，n个点n条边，就是环+外向树，环上的所有边都只会被经过一次，外向树上的边都会被经过两次. 分析加边与不加边每条边对答案的贡献的变化，可以得知：加边实际上就是将一条链上的所有边的贡献由2变为了1. 那么当k=1时，找到最长链—树的直径即可.
  当k=2时，可以继续再找一条链，但是如果这条链和上一次找的链有重合的部分那么重合的部分的贡献就变回了2,和异或一样. 这个时候如果想着枚举链的端点，看这两个端点在环上的距离就不行了，情况太多了. 我们需要找到一种通用型的算法，这个算法可以处理贡献变回2的情况. 考虑加边带来的收益的本质，就是将一些边的贡献由2变成了1，那么我们可以将原链上的所有边的边权变成-1，那么再一次所求得的最长链中如果包含了边权为-1的边，就相当于：不仅该链中一条边的贡献不会减少，原链中一条边的贡献也不会减少，正好符合要求.

总结

  考虑计数方法的本质.

最小环问题

发表于 2018-06-10 | 分类于 算法竞赛进阶指南 |

两种类别

  1.无向图,输出路径.
  2.有向图,输出路径.

环的本质

  两个点之间的最短路+次短路.

无向图

Description

There is a travel agency in Adelton town on Zanzibar island. It has decided to offer its clients, besides many other attractions, sightseeing the town. To earn as much as possible from this attraction, the agency has accepted a shrewd decision: it is necessary to find the shortest route which begins and ends at the same place. Your task is to write a program which finds such a route.

In the town there are N crossing points numbered from 1 to N and M two-way roads numbered from 1 to M. Two crossing points can be connected by multiple roads, but no road connects a crossing point with itself. Each sightseeing route is a sequence of road numbers y_1, …, y_k, k>2. The road y_i (1<=i<=k-1) connects crossing points x_i and x_{i+1}, the road y_k connects crossing points x_k and x_1. All the numbers x_1,…,x_k should be different.The length of the sightseeing route is the sum of the lengths of all roads on the sightseeing route, i.e. L(y_1)+L(y_2)+…+L(y_k) where L(y_i) is the length of the road y_i (1<=i<=k). Your program has to find such a sightseeing route, the length of which is minimal, or to specify that it is not possible,because there is no sightseeing route in the town.
Input

The first line of input contains two positive integers: the number of crossing points N<=100 and the number of roads M<=10000. Each of the next M lines describes one road. It contains 3 positive integers: the number of its first crossing point, the number of the second one, and the length of the road (a positive integer less than 500).
Output

There is only one line in output. It contains either a string ‘No solution.’ in case there isn’t any sightseeing route, or it contains the numbers of all crossing points on the shortest sightseeing route in the order how to pass them (i.e. the numbers x_1 to x_k from our definition of a sightseeing route), separated by single spaces. If there are multiple sightseeing routes of the minimal length, you can output any one of them.
Sample Input

5 7
1 4 1
1 3 300
3 1 10
1 2 16
2 3 100
2 5 15
5 3 20
Sample Output

1 3 5 2
Source

CEOI 1999

分析

  floyd算法的经典应用. 用floyd算法做这道题，需要明白floyd算法的原理.实质上是动态规划，将阶段放在最外层. $f_{k,i,j}$表示经过编号不大于k的点从i到j的最短路是多少.
  因为环就是两个点之间的最短路+次短路，利用迭代更新求最小环. 即用$d_{i,j} + d_{i,k} + d_{k,j}$更新. 环可以表示成由编号不超过k的节点构成的一条路径和经过节点k的一条路径构成. 因为对称性，这样做不会影响结果. 为什么这么考虑？如果不这样做可能会导致$d_{i,j}$和$d_{i,k} + d_{k,j}$是同一条路径.

代码

有向图

  有向图的环一定是从原点出发，然后再回到原点，有向图跑最短路用dijkstra算法，因为dijkstra算法是处理单源最短路用的，所以要枚举环的起点s. 一开始令d[s] = 0,加入优先队列中扩展，然后将d[s]设置为inf,当s第二次被取出时，d[s]即为所求.

小结

  环的本质是两个点之间两条不同的路径. 无向图上因为两条路径方向可能相同，故用floyd处理，对路径进行分类，避免路径重复. 有向图上因为两条路径方向相反，所以可以特殊处理枚举起点进行dijkstra算法.

bzoj2200 [Usaco2011 Jan]道路和航线

发表于 2018-06-09 | 分类于 算法竞赛进阶指南 |

中文题面

Description
Farmer John正在一个新的销售区域对他的牛奶销售方案进行调查。他想把牛奶送到T个城镇 (1 <= T <= 25,000)，编号为1T。这些城镇之间通过R条道路 (1 <= R <= 50,000，编号为1到R) 和P条航线 (1 <= P <= 50,000，编号为1到P) 连接。每条道路i或者航线i连接城镇A_i (1 <= A_i <= T)到B_i (1 <= B_i <= T)，花费为C_i。对于道路，0 <= C_i <= 10,000;然而航线的花费很神奇，花费C_i可能是负数(-10,000 <= C_i <= 10,000)。道路是双向的，可以从A_i到B_i，也可以从B_i到A_i，花费都是C_i。然而航线与之不同，只可以从A_i到B_i。事实上，由于最近恐怖主义太嚣张，为了社会和谐，出台 了一些政策保证：如果有一条航线可以从A_i到B_i，那么保证不可能通过一些道路和航线从B_i回到A_i。由于FJ的奶牛世界公认十分给力，他需要运送奶牛到每一个城镇。他想找到从发送中心城镇S(1 <= S <= T) 把奶牛送到每个城镇的最便宜的方案，或者知道这是不可能的。
Input

  • 第1行：四个空格隔开的整数: T, R, P, and S 第2到R+1行：三个空格隔开的整数（表示一条道路）：A_i, B_i 和 C_i 第R+2到R+P+1行：三个空格隔开的整数（表示一条航线）：A_i, B_i 和 C_i
    Output
  • 第1到T行：从S到达城镇i的最小花费，如果不存在输出”NO PATH”。
    Sample Input

6 3 3 4

1 2 5

3 4 5

5 6 10

3 5 -100

4 6 -100

1 3 -10

样例输入解释：

一共六个城镇。在1-2，3-4，5-6之间有道路，花费分别是5，5，10。同时有三条航线：3->5，

4->6和1->3，花费分别是-100，-100，-10。FJ的中心城镇在城镇4。

Sample Output

NO PATH

NO PATH

5

0

-95

-100

样例输出解释：

FJ的奶牛从4号城镇开始，可以通过道路到达3号城镇。然后他们会通过航线达到5和6号城镇。

但是不可能到达1和2号城镇。

提示

  spfa会被卡，dijkstra不能处理负边权. 注意题目给的图的性质.

代码

分析

  因为题目给出的图的特殊性质，如果将所有有向边拿走，剩下的点构成了若干个连通块，将连通块看作一个点，加入有向边后就构成了一个有向无环图，由此可以得到算法：块内单独处理，块外当作整体做拓扑排序.
  块内单独处理可以用dijkstra. dijkstra算法支持多源，将一个连通块内的所有点都加入到优先队列中，即可更新一个连通块的答案. 如果遇到有向边，则将有向边所连的块的入度—,如果入度为0，则加入到队列中，做拓扑排序. 负边权虽然在dijkstra中被用到了，但是到达点u时路径长度已经是最短的了，所以更新后v的路径长度也一样是最短的.
  那么算法流程就出来了：
  1.建两个队列，一个是dijkstra用，一个是拓扑排序用.
  2.取出拓扑排序队列中的连通块，将该连通块内所有的点都加入到dijkstra的队列中，进行更新. 每一次dijkstra算法都只加入当前连通块内的点到优先队列中.
  3.在dijkstra算法的过程中，如果遇到有向边，则该边所连的连通块度数—,为0则加入到拓扑排序队列中.
  几个优化：
  1.如果每次都扫哪些连通块入度为0非常耗时间，故用数组S存新加入的入度为0的连通块.
  2.时间戳.

Codeforces 917 D. Stranger Trees

发表于 2018-06-09 | 分类于 计划 |

题目大意

给定一棵n个点的树，求与该树恰好有k条边重合的n个点的树有多少种.

提示

  对”恰好”进行转化. 想一想无向完全图怎么计算生成树个数.

代码

分析

  非常神的一道题，简直是计数问题的巅峰！
  找问题的突破口. 恰好k个非常不好处理，要确保恰好就必须知道选哪k条边，这样才能保证生成的图是一棵树. 结合noi2011陨石的秘密一题，可以想到把“等号”变成“不等号”.即求出某个范围内的答案，然后直接相减或者利用容斥原理得到答案.
  假设已经选出了k条边，那么就剩下n-k个连通块，剩下的边随便连. 连通块内部是不能连边的，否则就形成了环，不是树. 那么可以将每个连通块都看作一个点. 计算n-k个点的完全图的生成树个数，可以用prufer序列来做. 利用prufer序列计数的原理是通过一种变换将n个点的生成树与长为n-2的序列一一映射. 统计生成树的个数只需要统计序列个数即可. 因为将所有连通块都看成了点，所以序列长度变成了m-2,每个位置有n中选择，故方案数为$n^{m - 2}$.
  这样计数有一个问题.这种计数方式对于每一个连通块，只计算了它连向儿子节点的方案，没有计算它连向父亲节点的方案. 对于两个连通块连边的方案，应该是两个连通块大小的乘积，不仅要考虑一个连通块选择接受节点的方案数，还要考虑一个连通块选择传出节点的方案数. 那么还要乘上$πsize_i$,即所有连通块的大小的乘积,要加上每一种方案的$πsize_i$.
  这就又是一个难点了，怎么样才能知道将一棵树分成m个连通块的所有方案的$πsize_i$的和呢？限制很多，利用dp求解. 考虑dp需要知道哪些信息：
  1.当前是哪一个节点.
  2.已经构成了多少个连通块.
  3.当前点所在的连通块大小为多少.
  用$f_{i,j,k}$来表示.对于每一个子节点，分选/不选两种决策来转移.需要注意一点：状态的最后一维k是在变动的，只有确定了i不与父节点形成一个连通块时，k才能被计入答案中. 转移以及最后求答案都要注意到这一点.
  令$ans_i$表示有k条边与原树重合的方案数,下面着重讲一讲容斥原理.
  一开始求出来的$ans_i$表示的是至少有k条边与原树重合的方案数. 为了保留这个至少，采取从小到大更新的方式. 根据容斥原理的基本形式一减一加即可. 需要注意的是加减操作需要乘上组合数. 因为上述过程计数得到的结果不同当且仅当选的边的顺序不同，举个例子：本来有(1,2),(2,3)两条边，先选(1,2)再选(2,3)和先选(2,3)再选(1,2)是相同的方案，而上述方法则会认为这是不同的方案. 手推一下就能发现容斥原理时乘上组合数才是正确的计算方法.

总结

  1.从等于号到不等号的转变.
  2.特殊考虑，其它随意的思想，由此可以得到一般情况的计数公式.
  3.容斥原理一定要考虑清楚容斥的顺序!!!有的容斥原理并不只是简单的加加减减，还要考虑计数时认为两种方案不同的标准是什么.
