noip2012 疫情控制

发表于 2018-06-12 | 分类于 算法竞赛进阶指南 |

题目大意

一棵树上有若干个特殊位置，每个位置可以四处挪动，费用为边权，求最小费用使得1号点到所有叶子节点的位置上都有一个特殊位置.

代码

分析

  这道题充分体现了我是一个sb这一事实.
  算法是很容易想到的,每个点肯定是尽量往上跳，但是跳到根节点后的策略就有很多种，不好确定总时间. 对于确定具体数值的题，可以利用二分求解. 这道题的答案也具有二分性. 如果一个时间刚刚可以满足要求，那么更多的时间一定可以满足要求，更少的时间一定不能满足要求.
  考虑这么一个贪心策略：每个点在有限的时间内尽量往上跳，如果不能够跳到根节点，就停在能跳到的深度最小的节点，否则就跳到根节点的对应的儿子节点上. 接下来把有点存在并且不是根节点的儿子节点的点打上标记，从下往上更新，如果一个节点有儿子节点并且所有的儿子节点都打上了标记，那么这个点也打上标记. 这个过程实际上就是把标记上传：如果一个点的所有儿子都被占领了，那么这个点也一定被占领了.
  接下来考虑根节点的所有儿子节点. 如果一个儿子节点没有被打上标记，并且到达它的军队走到根节点再回到该点不能在剩下的时间内完成，那么这个点就留在原地，否则加入到数组中. 记录一个二元组(s,rest)，是根节点的哪一个儿子节点和剩下的时间. 将根节点的所有没有打上标记的儿子节点提出来，按照它们到根节点的边权大小从小到大排序，并将二元组按照rest从小到大排序，对应着分配. 如果最后根节点的儿子节点都能被分配到一个点，那么二分的答案可行，否则不可行.
  需要注意贪心算法各个过程的先后顺序：是先打标记，再上传，然后处理能跳到根节点的军队.

一些sb错误

  1.倍增的时候先跳点，再计算代价……这里一定要注意先后顺序！
  2.上传标记的时候发现一个儿子节点不合法就直接返回了… 一定要搜完！

中文题面

Description
小 C 最近学了很多最小生成树的算法，Prim 算法、Kurskal 算法、消圈算法等等。 正当小 C 洋洋得意之时，小 P 又来泼小 C 冷水了。小 P 说，让小 C 求出一个无向图的次小生成树，而且这个次小生成树还得是严格次小的，也就是说： 如果最小生成树选择的边集是 EM，严格次小生成树选择的边集是 ES，那么需要满足：(value(e) 表示边 e的权值) 这下小 C 蒙了，他找到了你，希望你帮他解决这个问题。

Input
第一行包含两个整数N 和M，表示无向图的点数与边数。 接下来 M行，每行 3个数x y z 表示，点 x 和点y之间有一条边，边的权值为z。

Output
包含一行，仅一个数，表示严格次小生成树的边权和。(数据保证必定存在严格次小生成树)

Sample Input

1 2 1

1 3 2

2 4 3

3 5 4

3 4 3

4 5 6

HINT
数据中无向图无自环； 50% 的数据N≤2 000 M≤3 000； 80% 的数据N≤50 000 M≤100 000； 100% 的数据N≤100 000 M≤300 000 ，边权值非负且不超过 10^9 。

提示

非严格次小生成树怎么求？在此基础上再记录一个什么信息即可？

代码

分析

  求非严格次小生成树的方法是：先求出原图的一棵最小生成树，然后依次尝试加入非最小生成树内的边，一定会形成一个环，在环上找到最大的边权，用当前边替换掉该边权，可以得到一棵生成树，记录下所有可能的生成树的答案，取min即为所求.至于怎么求，仿照求LCA的过程，倍增时记录下解即可.
  因为题目中要求的次小生成树是严格的，那么可能出现替换的边的权值和原边的权值一样的情况. 为了处理这种情况，记录一下与最大值不同的第二大值即可. 更新的时候先考虑更新最大值，再来考虑更新次大值.

poj3417 Network

发表于 2018-06-12 | 分类于 算法竞赛进阶指南 |

英文题面

Description

Yixght is a manager of the company called SzqNetwork(SN). Now she’s very worried because she has just received a bad news which denotes that DxtNetwork(DN), the SN’s business rival, intents to attack the network of SN. More unfortunately, the original network of SN is so weak that we can just treat it as a tree. Formally, there are N nodes in SN’s network, N-1 bidirectional channels to connect the nodes, and there always exists a route from any node to another. In order to protect the network from the attack, Yixght builds M new bidirectional channels between some of the nodes.

As the DN’s best hacker, you can exactly destory two channels, one in the original network and the other among the M new channels. Now your higher-up wants to know how many ways you can divide the network of SN into at least two parts.

Input

The first line of the input file contains two integers: N (1 ≤ N ≤ 100 000), M (1 ≤ M ≤ 100 000) — the number of the nodes and the number of the new channels.

Following N-1 lines represent the channels in the original network of SN, each pair (a,b) denote that there is a channel between node a and node b.

Following M lines represent the new channels in the network, each pair (a,b) denote that a new channel between node a and node b is added to the network of SN.

Output

Output a single integer — the number of ways to divide the network into at least two parts.

Sample Input

4 1
1 2
2 3
1 4
3 4
Sample Output

3
Source

POJ Monthly—2007.10.06, Yang Mu

代码

分析

  先把树建出来，可以发现新加的M条边是树上的返祖边，与树上的若干条边形成一个环. 如果砍掉环内的边，则第二步必须要砍掉这条返祖边，方案唯一. 如果一条树边被多个环包围，那么这条边是不能砍的. 如果一条边没有被任何环包围，那么砍掉这条边后可以任选一条返祖边砍掉. 因为每次都对一条链修改，可以利用树上差分算法解决. 即$f_x++$,$f_y++$,$f_{LCA(x,y)} -= 2$.

bzoj1999 [Noip2007]Core树网的核

发表于 2018-06-12 | 分类于 算法竞赛进阶指南 |

中文题面

Description
设T=(V, E, W) 是一个无圈且连通的无向图（也称为无根树），每条边带有正整数的权，我们称T为树网（treenetwork），其中V, E分别表示结点与边的集合，W表示各边长度的集合，并设T有n个结点。 路径：树网中任何两结点a,b都存在唯一的一条简单路径，用d(a,b)表示以a,b为端点的路径的长度，它是该路径上各边长度之和。我们称d(a,b)为a,b两结点间的距离。 一点v到一条路径P的距离为该点与P上的最近的结点的距离： d(v，P)=min{d(v，u)，u为路径P上的结点}。 树网的直径：树网中最长的路径称为树网的直径。对于给定的树网T，直径不一定是唯一的，但可以证明：各直径的中点（不一定恰好是某个结点，可能在某条边的内部）是唯一的，我们称该点为树网的中心。 偏心距ECC(F)：树网T中距路径F最远的结点到路径F的距离，即 。 任务：对于给定的树网T=(V, E,W)和非负整数s，求一个路径F，它是某直径上的一段路径（该路径两端均为树网中的结点），其长度不超过s（可以等于s），使偏心距ECC(F)最小。我们称这个路径为树网T=(V,E,W)的核（Core）。必要时，F可以退化为某个结点。一般来说，在上述定义下，核不一定只有一个，但最小偏心距是唯一的。 下面的图给出了树网的一个实例。图中，A-B与A-C是两条直径，长度均为20。点W是树网的中心，EF边的长度为5。如果指定s=11，则树网的核为路径DEFG（也可以取为路径DEF），偏心距为8。如果指定s=0（或s=1、s=2），则树网的核为结点F，偏心距为12。
Input
包含n行： 第1行，两个正整数n和s，中间用一个空格隔开。其中n为树网结点的个数，s为树网的核的长度的上界。设结点编号依次为1, 2, …, n。 从第2行到第n行，每行给出3个用空格隔开的正整数，依次表示每一条边的两个端点编号和长度。例如，“2 4 7”表示连接结点2与4的边的长度为7。 所给的数据都是正确的，不必检验。
Output
只有一个非负整数，为指定意义下的最小偏心距。
Sample Input
5 2

1 2 5

2 3 2

2 4 4

2 5 3

Sample Output
5
HINT
对于70%的数据，n<=200000
对于100%的数据：n<=500000, s<2^31, 所有权值<500

代码

分析

  原题$O(n^3)$就可以水过，不知道还有$O(n)$的算法，今天看了一看，真tm厉害.
  做这道题需要大胆猜测一个性质：在一个直径上求得的答案等于在所有直径上求得的答案. 假设直径的两个端点为p,q，可以枚举核，在直径的每个点上都做一次dfs,求出最远距离，取一下max就得到了偏心距,时间复杂度$O(n^3)$.
  继续分析，答案取决于核外的点到核上的任意一个点最小距离. 这是针对核外的某个点而言的. 为了使得答案尽量小，肯定是核越长越好，所以枚举核只需要枚举一个端点即可.时间复杂度$O(n^2)$
  题目要求的实质是让最大值最小，很容易想到二分. 二分偏心距的最大值，怎么判断可行性呢？判断的过程肯定是越简单越好，如果还是一个一个点去求偏心距，那么就不需要二分了. 需要利用到直径的性质. 假设核靠近p的一端是u,因为(p,q)是直径，所以(p,u)上延伸出去的分支到u的距离一定不大于p到u的距离. 否则p就不可能是直径的端点. 那么只需要枚举u，v的位置可以通过维护单调指针得到，判断两个端点到u,v的距离的最大值即可.时间复杂度$O(nlogsum)$
  还没完，根据上述分析，可以知道答案一定是以下几种情况：
  1.p,q到u,v的距离.
  2.(p,q)上的点能延伸到的最远距离.
  枚举u,v的位置是单调递增的，那么可以用单调队列维护.时间复杂度$O(n)$.
  还没完，之前维护的是(p,q)上的点能延伸到的最远距离. 事实上，根据(p,u)上延伸出去的分支到u的距离一定不大于p到u的距离这一性质，(p,q)之外的点能延伸到的最远距离不可能成为答案，因为最后取max,可以将直径上所有的点能延伸到的最远距离归为第二种情况. 就不需要用单调队列维护了. 只需要求直径的两个端点到核的两个端点的距离即可.
  其实我觉得正常人应该能一下子想到最后一步，写出答案的表达式即可. 只是直径的最长性可能会有点困难想到.

bzoj1912 [Apio2010]patrol 巡逻

发表于 2018-06-11 | 分类于 算法竞赛进阶指南 |

中文题面


Input
第一行包含两个整数 n, K(1 ≤ K ≤ 2)。接下来 n – 1行，每行两个整数 a, b， 表示村庄a与b之间有一条道路(1 ≤ a, b ≤ n)。
Output
输出一个整数，表示新建了K 条道路后能达到的最小巡逻距离。
Sample Input
8 1

1 2

3 1

3 4

5 3

7 5

8 5

5 6
Sample Output
11
HINT
10%的数据中，n ≤ 1000, K = 1；
30%的数据中，K = 1；
80%的数据中，每个村庄相邻的村庄数不超过 25；
90%的数据中，每个村庄相邻的村庄数不超过 150；
100%的数据中，3 ≤ n ≤ 100,000, 1 ≤ K ≤ 2。

提示

  在树上加一条边会怎样？不加边的计算公式和加边后的计算公式分别是什么样的？本质是在统计什么？

代码

分析

  如果不加边，那么任意一条边都会被经过两次，模拟dfs的递归过程，可以发现每条边都要被遍历一次和回溯一次，所以所有边一定会被经过两次.
  如果加一条边，就相当于把一条链变成了一个环，n个点n条边，就是环+外向树，环上的所有边都只会被经过一次，外向树上的边都会被经过两次. 分析加边与不加边每条边对答案的贡献的变化，可以得知：加边实际上就是将一条链上的所有边的贡献由2变为了1. 那么当k=1时，找到最长链—树的直径即可.
  当k=2时，可以继续再找一条链，但是如果这条链和上一次找的链有重合的部分那么重合的部分的贡献就变回了2,和异或一样. 这个时候如果想着枚举链的端点，看这两个端点在环上的距离就不行了，情况太多了. 我们需要找到一种通用型的算法，这个算法可以处理贡献变回2的情况. 考虑加边带来的收益的本质，就是将一些边的贡献由2变成了1，那么我们可以将原链上的所有边的边权变成-1，那么再一次所求得的最长链中如果包含了边权为-1的边，就相当于：不仅该链中一条边的贡献不会减少，原链中一条边的贡献也不会减少，正好符合要求.

总结

  考虑计数方法的本质.

提示

  给出一堆变量之间的大小关系，问至少多少次后知道这些变量关系的大小顺序，或至少多少次后知道这些关系有矛盾，或者大小关系不确定.

代码

分析

  主要考察对拓扑排序的理解，各种情况的判断以及逻辑关系的判断. 因为拓扑排序并不是按照给定的关系顺序来处理的，所以要读一条处理一条.用vis数组标记哪些变量已经出现过了，如果拓扑排序后，在拓扑序里的点的个数<已经出现的点的个数，说明存在矛盾. 如果在一轮拓扑排序中，有1个以上入度为0的点，说明大小关系不确定，如果n个点都出现在拓扑序里的，则大小关系确定. 注意这里面的逻辑关系以及优先级.
