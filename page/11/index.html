线性dp总结

发表于 2018-05-17 | 分类于 算法竞赛进阶指南 |

1.基本处理方法

  1.找维度：限制、个数、长度.
  2.找状态表示：最优值、个数、看问题要求什么.
  3.找决策：刷表or填表.

2.通用技巧

  1.排序or其它算法解决题目限制.
  2.决策集合的决策有限制条件，并且决策集合的大小只增不减，可以用一个变量维护代表元.
  3.优化状态. 维度上能否互相导出. 若可以则省略掉可以被导出的状态，只需要保留状态的“秩”.
  4.网格图的阶段；行/列，格子.
  5.元素相对大小不变，采用整体加减法，考虑边界来递推. (整体加减法不会影响元素的相对大小)
  6.用与答案数组相同大小的数组表示每一维是从哪一个状态转移过来的，从而可以递归输出方案.

3.特定技巧

  1.中位数调整法.
  2.凸图形的轮廓线表示法.

3.poj3666

英文题面

Description

A straight dirt road connects two fields on FJ’s farm, but it changes elevation more than FJ would like. His cows do not mind climbing up or down a single slope, but they are not fond of an alternating succession of hills and valleys. FJ would like to add and remove dirt from the road so that it becomes one monotonic slope (either sloping up or down).

You are given N integers A1, … , AN (1 ≤ N ≤ 2,000) describing the elevation (0 ≤ Ai ≤ 1,000,000,000) at each of N equally-spaced positions along the road, starting at the first field and ending at the other. FJ would like to adjust these elevations to a new sequence B1, . … , BN that is either nonincreasing or nondecreasing. Since it costs the same amount of money to add or remove dirt at any position along the road, the total cost of modifying the road is

|A1 - B1| + |A2 - B2| + … + |AN - BN |
Please compute the minimum cost of grading his road so it becomes a continuous slope. FJ happily informs you that signed 32-bit integers can certainly be used to compute the answer.

Input

  • Line 1: A single integer: N
  • Lines 2..N+1: Line i+1 contains a single integer elevation: Ai

Output

  • Line 1: A single integer that is the minimum cost for FJ to grade his dirt road so it becomes nonincreasing or nondecreasing in elevation.

Sample Input

7
1
3
2
4
5
3
9
Sample Output

3
Source

USACO 2008 February Gold

翻译

给定长度为n的序列A,构造一个长度为N的序列B,满足：
1.B非严格单调. 2.最小化$Σ|A_i - B_i|$

代码
分析

  一个定理：一定存在一种构造B的方案，使得B中的数值在A中都出现过.
  证明：利用数学归纳法，N=1时显然成立. 假设N = k - 1时，可以构造出序列$B_1 ~ B_{k - 1}$,当N = k时，若$B_k ≤ A_k$,直接令$B_k = A_k$即可. 否则，假设存在一个j,使得$B_j,B_{j + 1},…,B_k$的数值都为v. 根据中位数原理，假设$A_j,A_{j + 1},…,A_k$的中位数为mid,若$mid ≥ B_{j - 1}$,则直接令$B_j,B_{j + 1},…,B_k$都为mid,否则就将这一段与$B_{j - 1}$,那一段合并，求新的中位数. 直到满足条件为止. 这种方法称为中位数调整法.
  那么考虑设计状态：$f_{i,j}$表示A的前i个数中，第i个数选择了B的第j小的数的答案. $f_{i,j} = min(f_{i-1,k},|A_i - A’_j|)$. 决策集合只增不减，依然可以$O(n^2)$处理.

英文题面

According to top-secret A-country plans, I-country is divided into N*M equal squares, each square contains some oil resources. They want to occupy all the territory of I-country, but the UN (United Nations) will allow them to occupy only K squares. Of course, A-country want to get control over as many oil as possible, but, they will have to guard all their territory. So, they need their territory to be easy-controlled, i.e. from any square to any it must be possible to get moving only along two directions (selected from the next list: left, right, up, down; for different squares pairs of directions may differ).
You are to write a program, which determinies, what squares will be occupyed by A-country. If there are several solutions, you may output any.

Input
On the first line of input there are 3 integer numbers N,M,K (1<=N,M<=15, 0<=K<=N*M). Next N lines contains M integers each, which are the number of oil resource on that square. Each of this numbers lies in range of 0 to 1000.

Next you should output K pairs of numbers —- coordinates of the squares which will be occupied by A-country. The first coordinate is number of row (top to bottom, starting from 1), second is number of column (left to right, starting from 1).

Sample test(s)

Input

10 20 30
40 2 3

Output
Oil : 100
1 1
1 2
1 3
2 1

翻译

n*m的矩阵中，每个格子有一个权值，要寻找一个包含K个格子的凸连通块，使得这个连通块内的格子的权值和最大，求出这个权值和，并输出连通块的方案.

提示

  网格图用什么表示阶段？
  如何表示凸多边形？试试观察看能否得出规律？

代码
分析

  挺神的一道题.
  以行为阶段，需要关注的信息是：当前已经处理完了几行，已经选出了多少个格子，当前行的左端点和右端点分别是哪一个，左右轮廓.
  关键的问题是如何维护左右轮廓. 注意到一个凸连通块，其左侧的点的列数一定是先递减后递增的，右侧的点的列数一定是先递增后递减的. 那么维护左右轮廓只需要知道当前轮廓是递增还是递减即可.那么状态表示也就出来了：$f_{i,j,l,r,x,y}$表示前i行选了j个格子，第i行的左端点是l,右端点是r,左轮廓的单调性是x,右轮廓的单调性是y的答案. 转移什么的讨论一下这个状态从哪几种状态转移而来即可.
  如何输出解?一个比较常用的方法是开若干个和答案数组一样大小的数组，记录由上一层转移而来的状态的每一维.更新答案的时候同时更新这些信息，最后递归一遍就可以求出解来了.
  这种方法空间占用非常大. 需要将数组大小尽可能压缩. 比较牛逼的处理方法是将记录转移路径的int数组改成char数组. 因为每一维都很小，不会爆char.从int到char和从char到int实际上就是两种相反的映射.

6.未知来源的题

题目描述

m个饼干给n个人，对于第i个人，若有x个人得到的饼干比它多，那么它会产生$x * g_i$的愤怒值，求怎么分饼干使得这些人的总愤怒值最小.

分析

  已经有多少个人获得了饼干、已经发了多少个饼干应该作为阶段，用$f_{i,j}$表示.
  因为每个人得多少块饼干没有限制，而g值有区别，可以想到将所有人按照g值从大到小排序. g值大的分配的饼干就多. 那么排序后，这些人得到的饼干数就是非严格单调递减的. 状态的转移需要知道第i - 1个人的饼干数. 那么就要额外再记录一维，显然不行，考虑一下转化.
  若第i个人获得的饼干数>1,那么可以将所有人的饼干数-1，当前状态从这个状态转移而来. 因为所有人的相对大小不变，所以答案不会变. 否则就枚举从哪一个人开始到第i个人的饼干数都是1. 因为它们的饼干数都是1,就相当于知道了第i-1个人的饼干数，也可以快速确定有多少个人的饼干比它们多. (只有这些人的饼干数都是1).那么状态转移方程就是:$f_{i,j} = min(f_{i,j - i},f_{k,j - (i - k) + kΣg_p})$.
  相对大小不变的问题可以通过缩放到边界1来进行处理.

翻译

求区间[l,r]的f(l’,r’)的最大值. $l ≤ l’ ≤ r’ ≤ r$

提示

  询问特别多，区间总长度却非常小，甚至可以用数组存下，能想到什么？

代码

分析

  因为n只有5000,我们可以把$f_{l,r}$预处理出来，然后做一次dp,O(1)可完成查询操作，难点就在于怎么预处理.
  观察f函数，它是由相邻的两项异或而组成一个新的序列. 联想到杨辉三角，将$f_{l,r}$看作杨辉三角的最后一行，往上递推，一定能递推到一行只剩一个数. 预处理f函数的关键在于找到规律. 没有直接能表达出$f_{l,r}$的式子，就要找递推式. f函数构成的杨辉三角的每一个数是由它下面的数和它下面的右边的数异或得到的. 那么可以得到：$f_{l,r} = f_{l + 1,r} \^ f_{l,r - 1}$. 用记忆化搜索，即可$O(n^2)$预处理得到所有的f值. 接下来的dp就很容易了.

经验

  一个函数是递归定义的，并且与每一对相邻的数有关，那么可以联想到杨辉三角.

Codeforces 984 C. Finite or not?

发表于 2018-05-17 | 分类于 计划 |

英文题面

翻译

问p/q在b进制下是不是有限小数.

提示

  先考虑十进制.

代码

分析

  一个结论：在十进制下，若p/q化成最简分数后，q的质因子只有2和5，那么p/q就是一个有限小数，否则就是无限小数.
  证明：在十进制下，一个有限小数一定可以表示成$p/10^k$的形式,而$10^k$只含有2和5两种质因子，故q要能变成$10^k$,就必须只能含有2和5. 同样的，对于b进制下的p/q,若最简分数q的质因子在b中都出现过，那么p/q就可以表示成一个有限小数.
  怎么知道q的质因子在b中是否都出现过呢？ 求出它的所有质因子？复杂度太高了！可以利用唯一分解定律和gcd. 因为$gcd(a,b) = p_1^{min(a_1,b_1)}p_2^{min(a_2,b_2)}……p_k^{min(a_k,b_k)}$. 求q与b的gcd,并将q除去它. 一直除下去，知道gcd = 1,那么q中含有的b的质因数一定都被除完了，剩下的就是b没有的质因数，只需要判断q是否等于1即可.

算法竞赛进阶指南第五章

发表于 2018-05-16 | 分类于 算法竞赛进阶指南 |

前言

  历时3天时间完成了这一章的学习. 有一些数据结构有很好的应用.

1.并查集

  无向图连通情况，传递关系，位置占用(连成一片),带权并查集，扩展域并查集. 当位置连成一片可以跳过时，可以利用并查集维护每个点往右边跳能跳到哪个点.(跳过一整段连续的区域).
  几个经典的用并查集维护的关系：
  1.对立集合.
  2.同类集合.

2.树状数组

  树状数组区间修改+区间查询,思想是将整体求和转化为统计单点贡献.

3.线段树

  线段树维护一段数的gcd,利用定理$gcd(x,y) = gcd(x,y - x)$,维护序列首位元素以及差分数组.

Description

Your friend, Jackson is invited to a TV show called SuperMemo in which the participant is told to play a memorizing game. At first, the host tells the participant a sequence of numbers, {A1, A2, … An}. Then the host performs a series of operations and queries on the sequence which consists:

REVERSE x y: reverse the sub-sequence {Ax … Ay}. For example, performing “REVERSE 2 4” on {1, 2, 3, 4, 5} results in {1, 4, 3, 2, 5}
REVOLVE x y T: rotate sub-sequence {Ax … Ay} T times. For example, performing “REVOLVE 2 4 2” on {1, 2, 3, 4, 5} results in {1, 3, 4, 2, 5}

Codeforces 979 D.Kuro and GCD and XOR and SUM

英文题面

翻译

两种操作，要么往集合中插入一个数，要么给定x,k,s,求集合中的一个v,使得$k | gcd(x,v),x + v ≤ s$并且x异或v的值要求最大.

提示

  两个数异或最大能想到什么？

代码

分析

  由两个数异或得到的值最大可以想到Trie树(不要想线性基，那是多个数异或得到的值最大). 关键是限制不好处理. 如何才能保证gcd被k整除？与x的和≤s?
  对于第二个限制，我们可以在Trie树上记录当前节点子树叶节点的最小值，若要往下一层走，则先判断最小值+x是否≤s. 是则可以往下走. 对于第一个限制，要使得x与v的gcd被k整除，那么x,v一定都要是k的倍数. 先判断x是不是k的倍数，若不是，直接返回-1. v也要是k的倍数，在一棵Trie树上不好处理，因为在没有走到叶子节点以前不能判断这个数的具体组成. 那么就用10w棵Trie树,第i棵Trie树存的是被i整除的数,每次询问只需要在第k棵Trie树上查询即可.
  事实上这道题还可以用比较巧妙的暴力过掉：对k进行分类处理. 若$k > 100$,则直接找k的倍数，看它与x的异或值，更新答案. 若$k < 100$,则从s到1枚举异或得到的答案，看v是否满足答案. 跑的比std还快，不科学..

总结

  难点是对限制的处理. 不等式限制可以用极值法解决，而倍数关系则要对数分类. 要想到这个数在以后的哪些查询中可能会被考虑到.
