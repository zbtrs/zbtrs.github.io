poj1934 Trip

发表于 2018-05-30 | 分类于 算法竞赛进阶指南 |

英文题面

Description

Alice and Bob want to go on holiday. Each of them has planned a route, which is a list of cities to be visited in a given order. A route may contain a city more than once.
As they want to travel together, they have to agree on a common route. None wants to change the order of the cities on his or her route or add other cities. Therefore they have no choice but to remove some cities from the route. Of course the common route should be as long as possible.
There are exactly 26 cities in the region. Therefore they are encoded on the lists as lower case letters from ‘a’ to ‘z’.
Input

The input consists of two lines; the first line is Alice’s list, the second line is Bob’s list.
Each list consists of 1 to 80 lower case letters with no spaces inbetween.
Output

The output should contain all routes that meet the conditions described above, but no route should be listed more than once. Each route should be printed on a separate line. There is at least one such non-empty route, but never more than 1000 different ones. Output them in ascending order.
Sample Input

abcabcaa
acbacba
Sample Output

ababa
abaca
abcba
acaba
acaca
acbaa
acbca
Source

CEOI 2003

翻译

  按字典序从小到大打印所有的LCS.

提示

  打印所有的，只能用什么方法？

代码

分析

  先用dp求出LCS是多少，再来考虑如何输出答案.
  因为题目要输出所有解，不是输出个数，因此不能依靠dp记录路径. 只能通过搜索的方法. 从后往前搜. 令当前在第一个字符串内扫的位置为pos1,在第二个字符串内扫的位置为pos2,若s1[pos1] = s2[pos2],则在(pos1 - 1,pos2 - 1)位置继续搜，并记录s1[pos]这个字符. 否则选择就有两种了，要么是(pos1 - 1,pos2)或(pos1,pos2 - 1). 如果构造两个全是a的字符串，这种算法就超时了，需要对第二种决策进行优化.
  容易发现，累计答案当且仅当s1[pos1] = s2[pos2]时，若当前两个字符不相同，则跳到前面相同的字母即可,同时还要保证是从最优的位置转移过来的. 用两个数组l1[i][j]记录s1[i]及其以前的字符中最晚出现字符j的位置,l2类似，每次枚举字符i,往前面最优的决策中跳.
  为了方便处理，利用string记录路径，set排序去重. 注意：最后字符串一定要翻转！！！

小结一下

  题目要输出所有解时，只能用搜索了，没别的方法.
  考虑清楚什么情况下才会对答案有影响. 若指针会进行多次无意义的跳动，则预处理出从每个位置能跳到的下一个有效位置是哪一个. 当然，也可以用倍增法来优化.

Codeforces 909 F. AND-permutations

发表于 2018-05-30 | 分类于 计划 |

题目大意

构造两个长度均为n的排列，第一个排列要求$p_i != i$并且$p_i and i = 0$,第二个排列要求$p_i != i$并且$p_i and i != 0$.

提示

  找基准点

代码

分析

  有一个结论：对于每一个正整数x,总能找到一个比他小的并且每一位都与之相反的一个数. 如果把数位取反这种操作看作一种映射，那么变换前和变换后的数是一一映射的. 若两个数每一位都相反，那么这两个数and起来就等于0.
  怎么才能找到比x小并且每一位都与之相反的数呢？ 利用基准点—x最高位的1. 假设x最高位的1是第k位，若$x = 2^k + i$,那么对应的数就是$2^k - i - 1$. 其实就相当于将$2^k$当作对称轴.那么对于第一问，若n是偶数，动态更新对称轴，并更新答案. 注意：不要去枚举k然后去枚举i,正确的做法是枚举x,找最高位k并更新对应的数，因为原则：优先处理对答案有直接影响的量. 若n是奇数,因为不能两两配对，所以构造不出来.
  对于第二问，只要求一位相同即可. 还是找基准点—最高位k. 将所有最高位为k的分成一组(一定是连续的)，将这些数同时右移一位(类似循环排列)即可. 需要特殊处理的是1,最高位为第1位的数只有1，而1右移一位还是1，不合法，可以手动推出：在n < 6时，构造不出合法的解，当n = 6时，可以构造，n = 7时，前7个数正好可以用完(放在前7个位置)，那么剩下的位置就可以用之前的构造算法了. 需要注意的是若n是2的整数次幂是构造不出来的，因为最后一定会存在k,使得最高位为k的数只有1个，这个数没法挪动.

总结

  二进制数问题以最高位为基准点.
  两个数每一位都不相同的结论.
  优先处理对答案有直接影响的量.

提示

  较难解决的博弈论问题的常用方法，关键是如何表示状态？

代码

分析

  拿到一道博弈论题，先看看能不能找到规律，这道题中显然是找不到规律的. 如果不行，再来看能不能转化成有向图游戏利用sg函数求解. 在这道题中，如果一个人不能移动，那么这个人获胜，不符合用sg函数的条件. 那么只有用最后一种方法了：状态转移法.

Codeforces 919 D. Substring

发表于 2018-05-29 | 分类于 计划 |

题目大意

给定a,b,p,x,求$n*a^n=b(modp)$,(1≤n≤x),n的个数.

提示

  怎样才能使得统计的范围变小？

代码

分析

  最朴素的做法是从1到x枚举n,这样复杂度就与x有关. 观察题目的数据范围，发现p的数据范围刚好符合要求. 做法肯定是与p有关. 联想到bsgs算法，我们可以把x换一种方式表示：$x = y*(p - 1) + i$, (i ≤ p - 1),这样只需要枚举i,统计y的个数即可. 而根据费马小定理，$a^{p-1} = 1(mod p)$. 所以可以将$a^{p - 1}$约掉，最后求出y的式子$y = (b - i*a^i)((p - 1)*a^i)^{-1}(mod p)$,y的上限知道(n不能超过x),下限就是这个同余式右边一项，那么y的个数自然也很好统计了.

总结

  利用费马小定理将同余式指数的枚举的复杂度变成与P相关.

poj1952 BUY LOW, BUY LOWER

发表于 2018-05-28 | 分类于 算法竞赛进阶指南 |

题目大意

  求最长严格下降子序列的长度及个数，两个子序列不同当且仅当元素不同，下标不是影响因素.

提示

  先考虑标准为：下标不同而序列不同时该怎么计数.伴随更新法.

代码

分析

  如果仅仅考虑下标不同而导致子序列不同，那么这道题就很好做. 统计个数利用伴随更新法，即若从最优解更新，则覆盖掉方案数，若从相同的最优解更新，则累加方案数.
  非常麻烦的一个问题是数值重复要怎么解决？有一点很容易想到：如果元素$a_i$前面有一个相同的元素，那么计数到前一个$a_i$就可以停止计数. 因为它们数值相同，可能造成区别的位置只有它们之间的位置. 对于序列654654,如果从第二个5向左边扩展，计数到第一个5就不继续进行计数了，但是还是会重复计数，因为65会被重复计数两次. 而第二个5的答案是从第二个6的答案转移而来的，于是可以想到:若一个数为最长下降子序列的第一个元素，并且它的前面有与他相同的数，那么这个数的方案数为0，这样用它来更新后面的数，方案数自然就都是0了. 非常奇妙.

总结

  限制枚举顺序，若要求数值上不同，则将所有的计数方案变到左边第一个数上. 利用伴随转移，可以将重复计数的答案赋值为0.

tyvj1124 花店橱窗

发表于 2018-05-28 | 分类于 算法竞赛进阶指南 |

中文题面

题目描述
每种花都有一个标识，假设杜鹃花的标识数为1，秋海棠的标识数为2，康乃馨的标识数为3，所有的花束在放入花瓶时必须保持其标识数的顺序，即：杜鹃花必须放在秋海棠左边的花瓶中，秋海棠必须放在康乃馨左边的花瓶中。如果花瓶的数目大于花束的数目。则多余的花瓶必须空置，且每个花瓶中只能放一束花。

每种花放在不同的瓶子里会产生不同的美观程度，美观程度可能是正数也可能是负数。

上述例子中，花瓶与花束的不同搭配所具有的美观程度，如下表所示：

                         花    瓶

                  1     2    3    4    5
   1 (杜鹃花)     7    23   -5  -24   16
   2 (秋海棠)     5    21   -4   10   23
   3 (康乃馨)    -21    5   -4  -20   20


根据上表，杜鹃花放在花瓶2中，会显得非常好看；但若放在花瓶4中则显得十分难看。

为取得最大美观程度，你必须在保持花束顺序的前提下，使花束的摆放取得最大的美学值，并求出每种花应该摆放的花瓶的编号。

输入格式
第1行：两个整数F和V，表示xq和xz一共有F种花，V个花瓶。(1<=F<=V<=100)
第2行到第F+1行：每行有V个数，表示花摆放在不同花瓶里的美观程度值value。(美观程度和不超过maxint，美观程度有正有负。)
输出格式
输出有两行：第一行为输出最大美观程度和的值，第二行有F个数表示每朵花应该摆放的花瓶的编号。

提示
其实就是简单的DP，花店橱窗问题啦。
注意尽量靠前放啊!
样例数据
输入样例 #1
3 5
7 23 -5 -24 16
5 21 -4 10 23
-21 5 -4 -20 20
输出样例 #1
53
2 4 5

代码

分析

  很简单的dp啦,令$f_{i,j}$表示前i种花放在前j个花瓶的答案. 那么$f_{i,j} = max\{f_{i,j - 1},f_{i-1,j-1} + a_{i,j}\}$. 为了输出方案，再开一个同等大小的数组记录是从i-1转移过来的还是从i转移过来的。

教训

  一定要注意初始化时的边界问题！手推几组小数据看转移是否合法.
