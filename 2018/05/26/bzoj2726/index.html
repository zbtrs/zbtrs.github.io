bzoj2726 [SDOI2012]任务安排(斜率优化模板的选择)

发表于 2018-05-26 | 分类于 算法竞赛进阶指南 |

中文题面

Description
机器上有N个需要处理的任务，它们构成了一个序列。这些任务被标号为1到N，因此序列的排列为1,2,3…N。这N个任务被分成若干批，每批包含相邻的若干任务。从时刻0开始，这些任务被分批加工，第i个任务单独完成所需的时间是Ti。在每批任务开始前，机器需要启动时间S，而完成这批任务所需的时间是各个任务需要时间的总和。注意，同一批任务将在同一时刻完成。每个任务的费用是它的完成时刻乘以一个费用系数Fi。请确定一个分组方案，使得总费用最小。
Input
第一行两个整数，N,S。
接下来N行每行两个整数，Ti,Fi。
Output
一个整数，为所求的答案。
Sample Input
5 1

1 3

3 2

4 3

2 3

1 4

Sample Output
153

代码

分析

  很显然这是一道dp题. 要知道当前划分到了哪一位，已经划分了几段，状态中必须记录两个信息：$f_{i,j}$表示当前划分到了第i位，已经划分了j段的总费用.
这个dp显然是过不了的，光是状态数就已经爆了，还没有算上转移的复杂度.
  其实划分问题都只用保留第一维. 但是要计算S的影响，必须要知道当前划分了多少段，只知道划分到了哪里不能推出来. 于是就要用上一种叫做费用提前计算的方法. 若对于当前枚举到的状态，不知道有多少前继状态影响到它，但知道它影响哪些后继状态，可以将它对后继状态的影响记入状态中. 那么$f_i = min\{f_j + sumT_i * (sumC_i - sumC_j) + S * (sumC_N - sumC_j)\}$,最后一部分即为状态i对后继状态的影响.
  对dp转移进行优化. 转移方程内的多项式包含i和j的乘积项，因此不能用单调队列优化，”只能”用斜率优化. 关于斜率优化我有两种不同的模板，一种是博客园上的，一种是书上介绍的，下面说一说在什么情况下用哪种模板比较好：
  1.如果是普通情况，当然是博客园上的模板更好，因为斜率可能会出现不存在的情况. 使用第一种模板时要记住：斜率递增，符号相反，斜率递减，符号相同.
  2.如果斜率不一定递增，但是横坐标是递增的，那么用第二种模板. 保存整个凸包，不进行队首的操作，每次二分查找合适的斜率位置.
  3.其它情况都用第二种模板 + cdq分治. 先递归处理左边的部分，然后维护水平序凸包，对右边的斜率也进行排序. 两个单调指针扫描转移即可.
  对于本题而言，属于第二种情况.
