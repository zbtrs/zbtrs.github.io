bzoj1150 [CTSC2007]数据备份Backup

发表于 2018-05-02 | 分类于 算法竞赛进阶指南

###中文题面 Description 　　你在一家 IT 公司为大型写字楼或办公楼（offices）的计算机数据做备份。然而数据备份的工作是枯燥乏味 的，因此你想设计一个系统让不同的办公楼彼此之间互相备份，而你则坐在家中尽享计算机游戏的乐趣。已知办公 楼都位于同一条街上。你决定给这些办公楼配对（两个一组）。每一对办公楼可以通过在这两个建筑物之间铺设网 络电缆使得它们可以互相备份。然而，网络电缆的费用很高。当地电信公司仅能为你提供 K 条网络电缆，这意味 着你仅能为 K 对办公楼（或总计2K个办公楼）安排备份。任一个办公楼都属于唯一的配对组（换句话说，这 2K 个办公楼一定是相异的）。此外，电信公司需按网络电缆的长度（公里数）收费。因而，你需要选择这 K 对办公 楼使得电缆的总长度尽可能短。换句话说，你需要选择这 K 对办公楼，使得每一对办公楼之间的距离之和（总距 离）尽可能小。下面给出一个示例，假定你有 5 个客户，其办公楼都在一条街上，如下图所示。这 5 个办公楼分 别位于距离大街起点 1km, 3km, 4km, 6km 和 12km 处。电信公司仅为你提供 K=2 条电缆。 　　上例中最好的配对方案是将第 1 个和第 2 个办公楼相连，第 3 个和第 4 个办公楼相连。这样可按要求使用 K=2 条电缆。第 1 条电缆的长度是 3km-1km=2km ，第 2 条电缆的长度是 6km-4km=2km。这种配对方案需要总长 4km 的网络电缆，满足距离之和最小的要求。 Input 第一行包含整数n和k 其中n（2≤n≤100000）表示办公楼的数目，k（1≤k≤n/2）表示可利用的网络电缆的数目。 接下来的n行每行仅包含一个整数（0≤s≤1000000000）,表示每个办公楼到大街起点处的距离。 这些整数将按照从小到大的顺序依次出现。 Output 输出应由一个正整数组成，给出将2K个相异的办公楼连成k对所需的网络电缆的最小总长度。

###分析   首先可以观察样例猜出一个结论：电线一定架在相邻的两个房子间. 那么差分一下，记$d[i] = a[i + 1] - a[i]$,现在的任务就是从$d$中选出k个数来，使得它们之间两两不相邻.   显然是可以用dp来做的. 令$f[i][j][0/1]$表示前i个点中选了j个点，第i个点选/不选的最小代价. 不过k太大，不行. 数据这么大，还能用什么办法呢？二分？不会判定;贪心？有不能选相邻点的限制，不好做.   正解就是贪心！很容易想到每次都选最小值，然后把相邻两个点给去掉，接着选最小值，可以用优先队列来实现. 事实上这样是错的. 令最小值的点为$x$,$x$左边的点为$l$,右边的点为$r$,能选$x$的前提是$l$和$r$都没有被选. 有可能选了$l$和$r$的其中一个，或者$l$和$r$都选比选$x$要优.这就是决策了，贪心没办法直接处理决策.好在我们可以忽略其中一种情况：选了$l$和$r$中的某一个，因为只选一个的话还不如选$x$. 可还是有两种决策，怎么办？   贪心虽然不能直接处理决策，但它可以反悔！，我可以先选其中一个看上去优的决策，如果不好，我就选另一种决策将其抵消掉. 在本题中，每次先选最小的点$x$,然后删掉点$l,x,r$,将点$x$的权值变成$d[l] + d[r] - d[x]$,如果再次选到了$x$这个点，就将一开始选择点$x$的代价$d[x]$给抵消掉了，相当于选了$l,r$,选的点的个数依旧是$2$，删了$4$个点.最后正好能删掉$2k$个点.   边界怎么处理？贪心的策略是每次选权值最小的点,如果这个点正好是左端点or右端点该怎么办呢？假设删的点$x$是左端点，如果按照上面说的那样处理的话，最后删的点的数量就会小于$2k$,出现这种原因是因为可能会选择删掉$x$右边的点而不是删掉$x$. 其实仔细想想就能发现，这种情况在贪心中绝对不会被考虑到，因为$x$是最小值，又在最左边，受到限制的点数最少，要删肯定是删它. 为了避免下一步删掉其右边的点造成最终删掉的点数不对，我们将$d[0]$和$d[n + 1]$设置为$INF$. ###一些智障错误   1.链表删边后$pre,next$数组的变换顺序弄反了. 应先边相邻的点，再变自己.   2.忘了删掉点$x$后还会合并出点$x$,以致于$x$相邻的点的$pre,next$弄错.   3.只变了优先队列中保存的结构体的值，没有变对应位置上的值. ###总结   dp的复杂度太高要想到优化或者是其他算法. 一般而言如果数据庞大到一维dp都做不了，那么肯定是其他的巧妙方法. 贪心和dp的关联挺大的，一个是单一决策，一个是多种决策.   贪心可以利用决策作用效果的抵消性来反悔，从而实现“贪心选择”来取代“决策转移”.
