bzoj4559 [JLoi2016]成绩比较

发表于 2018-05-07 | 分类于 计划 |

中文题面

Description
G系共有n位同学，M门必修课。这N位同学的编号为0到N-1的整数，其中B神的编号为0号。这M门必修课编号为0到M-
1的整数。一位同学在必修课上可以获得的分数是1到Ui中的一个整数。如果在每门课上A获得的成绩均小于等于B获
得的成绩，则称A被B碾压。在B神的说法中，G系共有K位同学被他碾压（不包括他自己），而其他N-K-1位同学则没
有被他碾压。D神查到了B神每门必修课的排名。这里的排名是指：如果B神某门课的排名为R，则表示有且仅有R-1
位同学这门课的分数大于B神的分数，有且仅有N-R位同学这门课的分数小于等于B神（不包括他自己）。我们需要
求出全系所有同学每门必修课得分的情况数，使其既能满足B神的说法，也能符合D神查到的排名。这里两种情况不
同当且仅当有任意一位同学在任意一门课上获得的分数不同。你不需要像D神那么厉害，你只需要计算出情况数模1
0^9+7的余数就可以了。
Input
第一行包含三个正整数N,M,K，分别表示G系的同学数量（包括B神），必修课的数量和被B神碾压的同学数量。第二
行包含M个正整数，依次表示每门课的最高分Ui。第三行包含M个正整数，依次表示B神在每门课上的排名Ri。保证1
≤Ri≤N。数据保证至少有1种情况使得B神说的话成立。N<=100,M<=100,Ui<=10^9
Output
仅一行一个正整数，表示满足条件的情况数模10^9+7的余数。

Sample Input
3 2 1

2 2

1 2
Sample Output
10

提示

  选用哪种方式计数？

代码

分析

  非常牛逼的一道题!
  题目想让我们干什么？计数！有什么方法？暴力数学dp法！
  先尝试数学法. 计数的步骤是先计算合法的相对排名的方案数，再计算在该排名下分数的方案数.排名可以用式子表示出来，但是无法计算在特定排名下分数的方案数.
  那么尝试使用dp. 先定义状态$f_{i,j}$表示前i个科目中,有j个人被碾压的方案数. 考虑如何从$f_{i,j}$转移到$f_{i+1,k}$,会发现转移不过来. why？因为被碾压的条件太苛刻了，只考虑当前的科目是不能确定这$k-j$是否被碾压. 换一个状态表示：令$f_{i,j}$表示前i个科目中，有j个人不被碾压的方案数. 那么从$f_{i,j}$转移到$f_{i+1,k}$就可以只考虑第i个科目对人的影响. 令$t_1 = k - j$,表示从被碾压的人中选k个人变成新的不被碾压的人(第i个科目的分数比B高). 而第i个科目有$R_i - 1$个人不被B碾压，剩下的$t_2 = R_i - 1 - t_1$个人就要从之前不被B碾压的人中选. (不能从被碾压的人中选，否则会使得k变大,矛盾). 方案数就是$C(n - j - 1,t_1) * C(j,t_2)$. 如果令科目i的所有人得分的方案数为$s_i$,那么$f_{i + 1,k} = f_{i,j} * C(n - j - 1,t_1) * C(j,t_2) * s_i$.
  下面来说一下$s_i$要怎么计算. 因为那$R_i - 1$个人的分数只要高于B的就好了，具体是多少分没有限制，剩下的人的分数要低于B的分数，只需要确定B的分数，就能得到方案数. 枚举B的分数x,则方案数为$Σ(U_i - x)^{R_i - 1} * x^{n - R_i} (1 ≤ x ≤ U_i)$, 因为$U_i$太大不好算. 事实上这是一个n-1次的多项式求和，答案一定可以表示为不超过n次的一个多项式. n+1个点即可确定一个n次多项式的唯一表示，利用拉格朗日插值法.
  先算出$y_1$到$y_{n+1}$的所有值，若中途碰到了上界$U_i$,返回，因为x是连续的，预处理出lsum和rsum. $lsum_i = lsum_{i - 1} * (U - i)$,rsum类似. 然后预处理出逆元的阶乘. 若$(n + 1 - i) \%2 == 1$,则计算出来的答案是负数，需要取相反数.（至于为什么，看代码实现）.

总结

  理一下基本思路、为什么要这么做、怎么样更好地实现？
  1.为什么dp?
  因为数学式子无法在计算排名的情况下计算分数的方案数.
  2.为什么状态的第二维定义为不能被超越的人数？
  因为被超越的人的定义太苛刻了，不能只通过一个科目的分数来决定.
  关于拉格朗日插值法：要求一个多项式的值，多项式的次数很小，但是要求的那一项很大，就可以利用拉格朗日插值法. 需要熟悉求和多项式可以用单个多项式来表示，n+1个点确定一个n次多项式，都涉及到次数的变化.
  对于x连续的n次多项式，可以利用预处理前缀+后缀信息来快速插值.
