bzoj2457 [BeiJing2011]双端队列

发表于 2018-05-01 | 分类于 算法竞赛进阶指南

###中文题面
Description
Sherry现在碰到了一个棘手的问题，有Ｎ个整数需要排序。
Sherry手头能用的工具就是若干个双端队列。

她需要依次处理这N个数，对于每个数，Sherry能做以下两件事：
1．新建一个双端队列，并将当前数作为这个队列中的唯一的数；
2．将当前数放入已有的队列的头之前或者尾之后。

对所有的数处理完成之后，Sherry将这些队列排序后就可以得到一个非降的序列。
Input
第一行包含一个整数N，表示整数的个数。接下来的N行每行包含一个整数Di，其中Di表示所需处理的整数。
Output
其中只包含一行，为Sherry最少需要的双端队列数。
Sample Input

6

3

6

0

9

6

3

Sample Output

2
HINT
100%的数据中N≤200000。

Source
Day1

###提示
  1.能转换成判定性问题求解吗？
  2.双端队列如何给一堆数排序？（这些数应该满足什么条件才能被排序？）

###程序

###分析
  一个思路是考虑每个元素放在哪，要不要新开一个双端队列？这样是做不下去的，因为无法预料到最终的局面.
  既然如此，那么从最终的局面入手. 比如样例$3,6,0,9,6,3$,最终的局面就是$0,3,3,6,6,9$,因为最后的序列是由若干个双端序列内的数拼起来的，所以考虑能把最终序列最少划分成几段.
  接下来的一步就比较重要了：能否划分成一段不仅仅取决于数的大小，还取决于数的下标. 把数的下标序列给写出来，就是$3,1,6,5,2,4$,那么什么样的下标序列才能被划分成一段呢？
  一段单谷能被划分成一段（先严格递减，再严格递增）.比如$3,1,6$和$5,2,4$.为什么呢？ 因为位于递减序列的数可以在队首放，位于递增序列的数可以在队尾放,求出单谷的个数即可.

###坑点
  若有相同的元素，则取最小下标值和最大下标值.
  最后一段即使构成不了单谷，也应算1次.

###总结
  挺好的一道题. 由结果反推，转化为判定性问题. 下标与双端队列的关系是关键.
