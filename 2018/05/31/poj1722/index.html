poj1722 SUBTRACT

发表于 2018-05-31 | 分类于 算法竞赛进阶指南

###英文题面 Description

We are given a sequence of N positive integers a = [a1, a2, ..., aN] on which we can perform contraction operations. One contraction operation consists of replacing adjacent elements ai and ai+1 by their difference ai-ai+1. For a sequence of N integers, we can perform exactly N-1 different contraction operations, each of which results in a new (N-1) element sequence.

Precisely, let con(a,i) denote the (N-1) element sequence obtained from [a1, a2, ..., aN] by replacing the elements ai and ai+1 by a single integer ai-ai+1 :

con(a,i) = [a1, ..., ai-1, ai-ai+1, ai+2, ..., aN] Applying N-1 contractions to any given sequence of N integers obviously yields a single integer. For example, applying contractions 2, 3, 2 and 1 in that order to the sequence [12,10,4,3,5] yields 4, since : con([12,10,4,3,5],2) = [12,6,3,5]

con([12,6,3,5] ,3) = [12,6,-2]

con([12,6,-2] ,2) = [12,8]

con([12,8] ,1) = [4]

Given a sequence a1, a2, ..., aN and a target number T, the problem is to find a sequence of N-1 contractions that applied to the original sequence yields T. Input

The first line of the input contains two integers separated by blank character : the integer N, 1 <= N <= 100, the number of integers in the original sequence, and the target integer T, -10000 <= T <= 10000. The following N lines contain the starting sequence : for each i, 1 <= i <= N, the (i+1)st line of the input file contains integer ai, 1 <= ai <= 100. Output

Output should contain N-1 lines, describing a sequence of contractions that transforms the original sequence into a single element sequence containing only number T. The ith line of the output file should contain a single integer denoting the ith contraction to be applied. You can assume that at least one such sequence of contractions will exist for a given input. Sample Input

5 4 12 10 4 3 5 Sample Output

2 3 2 1 Source

###分析   我一开始就被合并操作给迷惑了，加上n的范围是100，让我想到了区间dp. 区间dp其实也可以，但是数不好表示,除非再开一维.   这道题是很容易想到整体的算法框架的--先求可行性dp,然后通过dp的结果逆推得到答案.关键就是如何记录状态使得状态能够“吻合”题目中的合并操作以及能推出解来？找到操作的本质：实际上所有数都是不变的，只是在一些位置加上括号和减号. 对样例进行操作，不把数合并，而是拆分成减法的形式，可以发现最后所有数的前面要么是加号要么是减号. 也就是说：我们要给n-1个数的前面添加符号，使得它们运算的结果等于T. 求出答案后再转化为操作.   那么可行性dp就出来了：令$f_{i,j}$表示前i个数是否能表示出j. 两种转移：$f_{i - 1,j - a_i}$或$f_{i - 1,j + a_i}$,可以根据前继状态的可行性推出每个数前的系数.先从前往后处理前面符号是加号的数，然后不断操作1即可. 因为前面是加号的数必须通过偶数次减法才行，从前往后处理符号是加号的数就把这些数合并成了若干连续的段，最后不断地操作1就能使得它们前面的符号是加号了. ###坑点   第二个数前面的符号一定是减号，要特殊处理边界！
