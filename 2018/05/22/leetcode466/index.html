###题目大意 定义conn(s,n)为n个字符串s首位相接连成的字符串. 给定s1,n1,s2,n2.求一个最大的整数m,满足conn(conn(s2,n2),m)能由conn(s1,n1)生成. 一个字符串能被另一个字符串生成当且仅当能从第一个字符串中删去某些字符得到第二个字符串. ###代码

###分析   这道题如果能想到用dp做基本上就成功了.   首先化简公式:conn(conn(s2,n2),m) = conn(s2,n2 * m),令n2 *m = M,求得最大的一个M后除以n2就是答案了. 那么问题就转变成了求满足conn(s2,M) = conn(s1,n1)的M的最大值. 我一开始的想法是看一个s1能生成多少个s2,然后利用数学方法求解. 这样会有一个问题：s1可能会有一部分没用完，数学方法显然不能针对这样的特殊情况进行处理. 但这正好也能让我们想到dp，需要记录当前的起始点是哪一个. 要求M的最大值，可以想到将M变成状态的值,那么$f_{i,j}$表示当前的起始点是i,s1已经用了j个字符(≤s1*n1),能得到的最大的M是多少. 由于s1*n1过大，这种方法行不通. 事实上M可能会比较小，至少从数量级上看是比s1*n1小得多，那么可以想到一种常用的方法对状态进行优化：当状态多，状态表示的值少时，可以将它们交换. 那么$f_{i,j}$表示起始点是i,已经拼了j个s2所用的s1的字符数量. 最后只需要从大到小枚举j,找到第一个$f_{i,j} ≤ s1 * n1$的j即可.状态转移也很显然：$f_{i,j} = f_{i,j - 1} + f_{(i + f_{i,j - 1}) % |s1|,1}$.   这样转移会非常慢，考虑到一个一个拼s2和两两拼s2最后的结果都是一样的，可以倍增优化转移. 令$f_{i,j}$表示起始点为i,拼了$2^j$个s2所用的s1的字符的数量. 转移和上面基本上是一样的. 最后统计答案就从大到小枚举j,若$cnt + f_{i,j} ≤ n1 * s1$,那么$cnt += f_{i,j}$,并且$ans += 2^j$. ###总结   要能从小样例中看出问题的答案可由相同的子问题求解得到，于是想到dp. 若写出来的dp方程一维是线性增长的，并且方程比较简单. (表示的状态一般是第几个，和时多少),那么可以考虑用倍增优化.
