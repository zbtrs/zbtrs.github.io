Codeforces 965 E. Short Code

发表于 2018-05-19 | 分类于 计划

###题目大意   给n个不相同的字符串，要求将所有字符串用它的前缀代替，使得这些前缀互不相同，并且长度和最小. ###提示   由前缀能想到什么？ ###代码

###分析   看到前缀，很容易想到将所有字符串都放到Trie树上处理. 于是每一个单词都对应了一个点. 用一个前缀代替字符串，实际上就是在Trie树中，将点向其祖先方向上挪. 为了使所有前缀互不相同，每个位置最多只能被一个点占据. 问题就转化为了：在一棵树上，可以将一个点向其祖先方向上挪，不能有多个点在同一位置，问最后所有点的深度和最小是多少.   我一开始的想法是每次挪深度最大的一个点，在Trie树上倍增找到深度最低的没有被占用的点. 这个贪心的想法很容易举出反例来. 但由此很容易想到正确的解法：肯定要将深度最大的点挪到上面去. 这个做法错就错在处理不同子树的处理上，如果先将深度最大的点挪到最上面去，那么就有可能会占据其它子树的位置,所以我们应该把这些子树合起来考虑. 递归，在以u为根节点的子树中找到深度最大的一个点，将其放到根节点上(若根节点为空).然后将这些子树合并. 利用multiset维护,合并两个multiset用启发式合并.(第一次知道原来函数的返回类型可以是STL容器). ###经验   单单考虑一个点会对其它子树的决策造成影响，解决方法是将所有可能有影响的点合起来一起考虑，在树上多采用自底向上更新的方法.
