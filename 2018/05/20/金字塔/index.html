金字塔

发表于 2018-05-20 | 分类于 算法竞赛进阶指南

###中文题面
虽然探索金字塔是极其老套的剧情，但是这一队
探险家还是到了某金字塔脚下。经过多年的研究，科
学家对这座金字塔的内部结构已经有所了解。首先，
金字塔由若干房间组成，房间之间连有通道。如果把
房间看做节点，通道看做边的话，整个金字塔呈现一
个有根树结构，节点的子树之间有序，金字塔有唯一
的一个入口通向树根。并且，每个房间的墙壁都涂有
若干种颜色的一种。
探险队员打算进一步了解金字塔的结构，为此，他们使用了一种特殊设计的机器人。这
种机器人会从入口进入金字塔，之后对金字塔进行深度优先遍历。机器人每进入一个房间（无
论是第一次进入还是返回），都会记录这个房间的颜色。最后，机器人会从入口退出金字塔。
显然，机器人会访问每个房间至少一次，并且穿越每条通道恰好两次（两个方向各一次），
然后，机器人会得到一个颜色序列。但是，探险队员发现这个颜色序列并不能唯一确定金字
塔的结构。现在他们想请你帮助他们计算，对于一个给定的颜色序列，有多少种可能的结构
会得到这个序列。由于结果可能会非常大，你只需要输出答案对 $10^9$ 取模之后的值。
输入格式
输入文件包含一行，含有一个字符串，表示机器人得到的颜色序列。
输出格式
输出一个整数表示答案。
样例输入
ABABABA
样例输出
5

###提示
  在树上dfs得到的一个序列，能想到什么？

###分析
  由在树上dfs形成的一个序列可以想到dfs序. 因为dfs是一个一个子树遍历的，所以一个子树在所形成的序列上是连续的. 由数据范围很容易想到区间dp,即令$f_{l,r}$表示子串S[l,r]对应着多少种可能的金字塔结构. 根据区间dp的一般转移形式，很容易想到$f_{l,r} = Σf_{i,k} * f_{k + 1,j}$.这种转移方式是错的.
  考虑序列ABABABA,两种方案A|BAB|A|B|A和A|B|A|BAB|A是一样的，因为它们都对应一个A延伸出3个B. 为什么会错呢？因为k只是我们枚举的一个分割点, 每一层枚举一个分割点，到最后可能就会有多种决策产生同一种局面，而他们选择的分割点相同，只是选择的次序不同罢了,也就是说：我们考虑的是排列的情况，而题目要求组合的情况. 要去重就必须要在每一层中确定选择几个分割点，并且还要确定这几个分割点的位置. 这样就有点搜索的意味了，会TLE.
  因为题目说子树是有序的，那么我们只需要让每一步的不同决策可能产生的不同局面不同即可.在第一棵子树上下手. 只考虑第一棵子树是由哪些点组成的，将k作为划分点. 如果k不相同，那么第一棵子树的大小也就不相同，整棵树也就不同了，就可以计数了. 在转移的时候，要先考虑只延伸一棵子树的情况. 因为遍历完第一棵子树后要回到根，所以在s[k] = s[r]时才能转移,在转移的时候一定要考虑状态的合法性！

###总结
  dp计数问题中，若多种决策会产生同一种最终局面，就必须改变决策，使每一层的决策所导出的局面都是不同的. 转移时要考虑状态的合法性(定义).
