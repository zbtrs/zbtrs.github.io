poj2185 Milking Grid

发表于 2018-05-03 | 分类于 算法竞赛进阶指南

###英文题面
Description

Every morning when they are milked, the Farmer John’s cows form a rectangular grid that is R (1 <= R <= 10,000) rows by C (1 <= C <= 75) columns. As we all know, Farmer John is quite the expert on cow behavior, and is currently writing a book about feeding behavior in cows. He notices that if each cow is labeled with an uppercase letter indicating its breed, the two-dimensional pattern formed by his cows during milking sometimes seems to be made from smaller repeating rectangular patterns.

Help FJ find the rectangular unit of smallest area that can be repetitively tiled to make up the entire milking grid. Note that the dimensions of the small rectangular unit do not necessarily need to divide evenly the dimensions of the entire milking grid, as indicated in the sample input below.

Input

  • Line 1: Two space-separated integers: R and C

  • Lines 2..R+1: The grid that the cows form, with an uppercase letter denoting each cow’s breed. Each of the R input lines has C characters with no space or other intervening character.
    Output

  • Line 1: The area of the smallest unit from which the grid is formed
    Sample Input

2 5
ABABA
ABABA
Sample Output

2
Hint

The entire milking grid can be constructed from repetitions of the pattern ‘AB’.
Source

USACO 2003 Fall

###翻译
给一个n*m的矩形，求一个面积最小的矩形，能通过复制、平移的方式全部覆盖.(边界处不需要铺满)

###提示
  1.二维问题通常怎么处理？
  2.矩形的面积和什么有关？

###代码（丑陋无比）

###分析
  二维的肯定要变成一维来处理嘛. 矩形的面积=长乘宽，那么对长和宽分别做kmp求最小循环节就可以了？
  我一开始想的是对第一行和第一列求最小循环节，后来发现不对劲，可能会有数据：
          aaaaa
          aacaa
          aaaaa
  其实也很好解决，对于每一行求最小循环节，再对于每一列求最小循环节，分别取最小公倍数嘛.
  又一次成功地WA了，在discuss中看到一组数据：
          aaabcaaa
          abababab
  如果求最小公倍数，最后的答案是16，事实上答案是12. 原因是行上的处理有问题，不一定是每一行选最小循环节最后得到的lcm最小，每一个字符串可能会有多个循环节. 问题就变成从n个位置中的每一个位置选1个可能的数，使得它们的lcm最小.
  似乎变成了一道不可解的爆搜题. 其实搞清楚lcm到底表示什么含义，就能想到正确解法了. 显然lcm要在每一行中都出现，那么开一个桶，将每一行可能的循环节在桶的相应位置+1，最后从小到大枚举，找到第一个值为n的数.
  如何找到串s的所有符合题意的长度？
  只需要对s进行一次kmp特征值计算，如果发现后面有多余的后缀，就把后缀再拿来做一次kmp，不断重复，直到没有后缀。这个过程就可以记录所有符合题意的串了。
   比如aaabcaaa，先求一遍kmp，得到5，再对后缀aaa求一遍，得到1(即6,7,8)。
  又比如aabaabaabaa，先求一遍kmp，得到3(即3,6,9)，再对后缀aa求一遍，得到1，(即10,11)。
  真正的算法不难实现，难的是边界的处理.
  一种更好的做法是hash+kmp.考虑放矩形的过程，肯定是一个接着一个拼上去的.中间肯定会出现分割线. 那么放的小矩形的两边一定是相同的.即左面和右面的右边一列肯定是相同的. 所以将每一列/行hash,求最小循环节即可.
