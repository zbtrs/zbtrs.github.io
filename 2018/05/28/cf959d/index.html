Codeforces 959 D. Mahmoud and Ehab and another array construction task

发表于 2018-05-28 | 分类于 计划 |

题目大意

  给定一个数列a,要求构造一个长度和a相同的数列b,满足：
  1.b的字典序≥a
  2.b的每个数≥2
  3.b的每一对数都是互质的.
  4.在满足上述条件的情况下b的字典序最小.

分析

  非常好的一道题目.
  若最优性与可行性结合在一起，那么在可行性上考虑最优性的顺序.什么意思呢？满足可行性肯定是前提. 如果枚举到第i个数，发现前面的第j个数与第i个数不互质，那么这两个数中至少要修改一个才行. 从最优性的顺序来考虑肯定是修改后面的数更好. 那么不断将第i个数变大，直到与前i-1个数满足条件. 如果说可行性限制了一个框架，那么最优性则是在框架中决定方向，寻找最优解.
  其实只需要修改一个数即可，这个数后面的数依次从小到大取没有被占用的质数. 这个时候字典序已经满足要求了. 那为什么要取质数呢？如果取了一个尽可能小的合数，与前面的数不互质，那么前面的数一定不包含这个合数的质因子，那么这个合数就可以用其最小的质因子代替.
  利用标记质因子法实现. 不能每枚举一个数都去和前面的数求gcd. 利用质因子求gcd不等于1的情况的解是一种常用的方法. 记住一点：如果第i个数确定了，才能将这个数的所有质因子打上标记,因为有可能只是在试探性地看这个数能不能填.
