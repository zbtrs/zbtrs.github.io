Hdu3949 XOR

发表于 2018-05-09 | 分类于 算法竞赛进阶指南

###英文题面 Problem Description XOR is a kind of bit operator, we define that as follow: for two binary base number A and B, let C=A XOR B, then for each bit of C, we can get its value by check the digit of corresponding position in A and B. And for each digit, 1 XOR 1 = 0, 1 XOR 0 = 1, 0 XOR 1 = 1, 0 XOR 0 = 0. And we simply write this operator as ^, like 3 ^ 1 = 2,4 ^ 3 = 7. XOR is an amazing operator and this is a question about XOR. We can choose several numbers and do XOR operatorion to them one by one, then we get another number. For example, if we choose 2,3 and 4, we can get 2^3^4=5. Now, you are given N numbers, and you can choose some of them(even a single number) to do XOR on them, and you can get many different numbers. Now I want you tell me which number is the K-th smallest number among them.

Input First line of the input is a single integer T(T<=30), indicates there are T test cases. For each test case, the first line is an integer N(1<=N<=10000), the number of numbers below. The second line contains N integers (each number is between 1 and 10^18). The third line is a number Q(1<=Q<=10000), the number of queries. The fourth line contains Q numbers(each number is between 1 and 10^18) K1,K2,......KQ.

Output For each test case,first output Case #C: in a single line,C means the number of the test case which is from 1 to T. Then for each query, you should output a single line contains the Ki-th smallest number in them, if there are less than Ki different numbers, output -1.

Sample Input 2 2 1 2 4 1 2 3 4 3 1 2 3 5 1 2 3 4 5

Sample Output Case #1: 1 2 3 -1 Case #2: 0 1 2 3 -1

Hint

If you choose a single number, the result you get is the number you choose. Using long long instead of int because of the result may exceed 2^31-1. Author elfness Source 2011 Multi-University Training Contest 11 - Host by UESTC Recommend xubiao ###翻译 从n个数中选取若干个数，异或起来. 求所得的第k小的值. ###代码

###分析   线性基套路题.   首先找出这n个数中有用的数，即基底. 将每一个数看作m位二进制数，构造一个n行m列的矩阵. 对这个异或矩阵做高斯消元，最后得到的非0行即为矩阵的秩，也就是有用的那些数. 为什么呢？因为对矩阵做初等行变换矩阵的秩是不会发生改变的，当把矩阵变成相抵标准型后，没用的数都被消完了，只剩下基.   一个定理：主元所在位为1的整数是唯一的,什么意思呢？在相抵标准型中，每一个有用的数都有一个位置(i,j),满足该位置系数非0，第j列的其它位置都是0，第i行的第1~j-1列都是0. 也就是说，在$b_1,b_2,......,b_t$中，有且仅有$b_i$的第$c_i$位为1，其它数的$c_i$位为0.并且j随i变大而递增,也就是说，基底越来越小. 利用这一点可以求得第k小的数.   根据基的定义，这个数一定可以由基线性表示出来.j是单调递增的，所以选$b_i$一定比不选$b_i$得到的数要大. 将要查询的数k变成转换成二进制数. 若k-1的第i位为0，那么第t-i个基底就要选. (每个基底有选or不选两种可能). 两个二进制数一一对应. 若$k > 2^t$,则无解.   有一个问题：题目不允许选一个数2次，而异或空间中是可能的. 从$a_i$中选出几个数来异或有可能得到0. 实际上在上述运算中，如果不可能异或出0，那么就直接对k - 1 + 1 = k求.(跳过掉基底异或的结果为0的情况).怎么判断$a_i$能否异或出0呢？看最后的矩阵中是否有一行为0即可. 因为做高斯消元实际上就是每次选不同的两行异或嘛. ###总结   1.xor问题一定要转化为二进制数再去思考.   2.高斯消元后的主元下标单调性，基与行之间的关系.   3.xor问题有0的情况.
