bzoj3992 [SDOI2015]序列统计

发表于 2018-05-09 | 分类于 计划 |

中文题面

Description
小C有一个集合S，里面的元素都是小于M的非负整数。他用程序编写了一个数列生成器，可以生成一个长度为N的数
列，数列中的每个数都属于集合S。小C用这个生成器生成了许多这样的数列。但是小C有一个问题需要你的帮助：
给定整数x，求所有可以生成出的，且满足数列中所有数的乘积mod M的值等于x的不同的数列的有多少个。小C认为
，两个数列{Ai}和{Bi}不同，当且仅当至少存在一个整数i，满足Ai≠Bi。另外，小C认为这个问题的答案可能很大
，因此他只需要你帮助他求出答案mod 1004535809的值就可以了。
Input
一行，四个整数，N、M、x、|S|，其中|S|为集合S中元素个数。
第二行，|S|个整数，表示集合S中的所有元素。
1<=N<=10^9，3<=M<=8000，M为质数
0<=x<=M-1，输入数据保证集合S中元素不重复x∈[1,m-1]
集合中的数∈[0,m-1]

Output
一行，一个整数，表示你求出的种类数mod 1004535809的值。

Sample Input
4 3 1 2

1 2
Sample Output
8

【样例说明】

可以生成的满足要求的不同的数列有(1,1,1,1)、(1,1,2,2)、(1,2,1,2)、(1,2,2,1)、

(2,1,1,2)、(2,1,2,1)、(2,2,1,1)、(2,2,2,2)

提示

  1.计数问题通用的方法是什么？
  2.n这么大一般用什么方法处理？

代码

分析

  吼题啊!
  计数题无非就是dp和数学. 这道题显然是dp. 令$f_{i,j}$表示序列的前i个数乘积对M取模的结果为j的方案数. 转移时枚举第i个数是哪一个. 时间复杂$O(nm^2)$.似乎一分都拿不到，在考场上实际能拿10分.
  观察第二档数据范围. n变成了$10^9$,而m变成了100,递推次数这么多，而递推方式不变，很容易想到矩阵快速幂优化递推，这样就有30分了.
  剩下的70分是给正解的.首先需要能想到：在取模意义下，利用原根，乘法可以等效转化为加法.模数是一个质数，有原根，而且这个模数不一般，可以用$2^k + 1$表示出来. 说明可以用NTT?
  将转移过程的乘法运算变成加法运算后确实可以用NTT. 将S中的元素看作一个多项式$f(x)$. 若大小为a的数出现了t次，则$f(x)$的a次项的系数为t. 对于dp数组也是一样,将数的大小看作x的幂次. 因为转化为了原根，所以a为$g^{a’} = a$中的a’.
  n很大，直接递推的话还是会T,根据矩阵快速幂的思想，每次乘的多项式都是同一个，我们可以对这个多项式快速幂.
  NTT和FFT很像，有三点不同：
  1.NTT中的乘法和加法操作要取模，FFT不用.
  2.NTT的旋转因子是原根的(mod - 1) / 步长次幂，或是其相反数.
  3.NTT算完一次后，要将原序列复制的那一部分的答案累加到原序列中.
  下面解释一下为什么有这些不同点：
  第一条是很显然的，这是用NTT算法的目的.
  第二条可以将原根的次数的取值范围类比复平面来看，在FFT中旋转因子也要除以步长，是因为我们把复平面分成了步长份. 而NTT中的原根是有取值范围的，即(0~mod - 1)，类似复平面一样除以步长。
  第三条是只有在模意义下才能这样. $g^t = g^{p + t} (mod  m)$,因为$g^p$与m互质，所以可以把$g^p$给拿走. 从而将序列后半段的答案移到前面去. 为了方便统计答案.

脑残错误

  1.把步长初始化成了1，上界写成了<.
  2.忘了取模.
  3.将序列后半段累加到前半段时，i的枚举顺序弄反了.

总结

  对不是1e9+7的模数要敏感！模意义下的乘法转化为加法操作！
