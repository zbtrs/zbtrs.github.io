Codeforces 967 D. Resource Distribution

发表于 2018-05-18 | 分类于 计划

###题目大意   给出n个服务器的资源数$c_1,c_2,...,c_n$和运行两个程序需要的资源$x_1,x_2$,求一个解决方案，选取k1个资源数不小于$x_1 / k_1$的服务器使资源总和不小于$x_1$,选取k2个资源数不小于$x_2/k_2$的服务器使资源总和不小于$x_2$,两次选取的服务器不能有交叉. ###代码

###分析   这些限制条件中有一个是显然满足的：资源总和不小于$x_1$,和资源总和不小于$x_2$.关键就是选出$k_1$个资源≥$x_1 / k_1$的服务器. 在此基础上，还要让$k_1$尽量小,因为k越大，可选的数就越多，那么留给第二个任务的选择就更多了. 那么将服务器按照资源数从大到小排序,枚举$k_1$，找到最小的满足要求的. 接着按照相同的方法枚举$k_2$. 最后将两个任务交换，重复这个过程,为了避免出现满足第一个任务却不能满足第二个任务的情况.
