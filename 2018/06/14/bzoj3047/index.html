bzoj3047 Freda的传呼机

发表于 2018-06-14 | 分类于 算法竞赛进阶指南 |

中文题面

Description
为了随时与rainbow快速交流，Freda制造了两部传呼机。Freda和rainbow所在的地方有N座房屋、M条双向光缆。每条光缆连接两座房屋，传呼机发出的信号只能沿着光缆传递，并且传呼机的信号从光缆的其中一端传递到另一端需要花费t单位时间。现在Freda要进行Q次试验，每次选取两座房屋，并想知道传呼机的信号在这两座房屋之间传递至少需要多长时间。Freda和rainbow简直弱爆了有木有T_T，请你帮帮他们吧……
N座房屋通过光缆一定是连通的，并且这M条光缆有以下三类连接情况：
A：光缆不形成环，也就是光缆仅有N-1条。
B：光缆只形成一个环，也就是光缆仅有N条。
C：每条光缆仅在一个环中。。

Input

第一行包含三个用空格隔开的整数，N、M和Q。
接下来M行每行三个整数x、y、t，表示房屋x和y之间有一条传递时间为t的光缆。
最后Q行每行两个整数x、y，表示Freda想知道在x和y之间传呼最少需要多长时间。

Output
输出Q行，每行一个整数，表示Freda每次试验的结果。

HINT

提示

将环缩成一个点，放到原图中变成一棵树来考虑.

分析

分析

  这道题……
  三类数据，一类一类分析. 对于第一类数据，就是求树上两个点之间的路径长度，求出根到每个点的距离，那么答案就是$d_u + d_v - 2*d_{LCA(u,v)}$.
  对于第二类数据，不是特别好搞. 整体上来说是一个基环树，基本处理方法，先把环求出来. 对于环上的每一个点，以它为根，求出它的子树内的每个点的倍增数组，还要求出以它为一端，另一端在其子树内的最长链. (这里说的子树不包括环上其它点). 如果要求的两个点在同一个子树内，那么可以直接利用倍增求得答案，否则路径就是从一个点走到环上，然后走到另一个点. 那么就要先预处理出环上的点，以及环上的每个点到环上第一个点的路径长度. 涉及到点到下标的映射，处理起来比较麻烦.
  对于第三类数据…… 想到了bzoj1023. 只会求直径，给定两个点要怎么求路径啊？似乎也不能预处理，离线不好做，那么对于每一次询问，都要在$O(logn)$的时间复杂度内求出解. 这似乎只有树能办到…… 把环缩成点，变成一棵树来考虑？圆方树确实也行，但这样很麻烦，因为最后路径可能会包括环上的一段，那么就需要知道环上的具体信息. 一个解决方法是将进入到环上的第一个点连边向环中其它所有的点. 这样同一个环内的点的深度就都是一样的了，并且这样处理出来的图是一棵树，不断往LCA上跳，直到最后一步. 如果两个点在同一个环上了，就计算出这两个点在该环上的距离，否则各自往上跳一步即可.
  需要进行两次dfs,一次是处理出所有的环，一次是在环内部进行连边.
  处理了第三种情况，前两种情况顺便也被处理了.

小结

  断环为链，缩环为点，环内部连边，环上的信息是否需要记录……
  要在$O(logn)$的时间复杂度内求任意两个点的最短路，只有树上能做到，往这方面转化!
