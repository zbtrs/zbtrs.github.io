bzoj1079 [SCOI2008]着色方案

发表于 2018-06-06 | 分类于 算法竞赛进阶指南

###中文题面 Description 　　有n个木块排成一行，从左到右依次编号为1~n。你有k种颜色的油漆，其中第i种颜色的油漆足够涂ci个木块。 所有油漆刚好足够涂满所有木块，即c1+c2+...+ck=n。相邻两个木块涂相同色显得很难看，所以你希望统计任意两 个相邻木块颜色不同的着色方案。

Input 　　第一行为一个正整数k，第二行包含k个整数c1, c2, ... , ck。

Output 　　输出一个整数，即方案总数模1,000,000,007的结果。

Sample Input 3

1 2 3 Sample Output 10 HINT 100%的数据满足：1 <= k <= 15, 1 <= ci <= 5 ###提示   减少状态数，数字之间有差异吗？ ###代码

###分析   非常巧妙的一道题.   一开始看题面，这么短肯定是数学方法吧？去看数据范围，woc怎么只有这么点. 感觉不对劲,利用容斥原理等各种数学方法都做不出来. 那么考虑dp. 因为是排列计数问题，有两种思考方式：要么考虑插入一个数到某个位置，要么按照顺序填数. 如果是插数，因为可以放数的位置数不确定，所以这种方法不行. 那么只有第二种方法了.   每个集合内的数都是有限的，那么在填第i个数之前，需要知道每个集合剩下的数的个数. 这样状态数就是$5^{15}$,显然是过不了的. 如果反过来，$15^5$就可以过了. 事实上所有数字都是没有差异的，那么只需要知道大小为i的集合有多少个以及上一次选的数所在的集合的大小是多少即可. ###总结   计数题小数据范围一般对应dp   排列计数的两种dp顺序.   若对于所有对象而言，决策都是一样(类似)的，那么可以认为这些对象没有区别. 状态中只需要记录公共特征值即可(是多少-->有多少个)
