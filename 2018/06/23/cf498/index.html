Codeforces Round

发表于 2018-06-23 | 分类于 计划 |

前言

  跪的很惨(要不然我也不会来发整场比赛的总结).5道题竟然只做出了3道题，还WA了几次.

B.Nastya Studies Informatics

题目大意

  给定$l,r,x,y$,求有多少对$(a,b)$,满足$gcd(a,b) = x,lcm(a,b) = y,l ≤ a,b ≤ r$.

代码

分析

  很容易想到用唯一分解定律去处理，知道每个质因子可以取多少个，通过乘法原理得到答案？遗憾的是无法保证拼出来的树在$[l,r]$内.
  挺容易想到唯一分解定律的一个推论：$x * y = a*b$. 得到这个式子后再来求个数就很简单了. 枚举$a$,$b = x * y / a$ ,$a$必须是$y$的因数，枚举因数可以在$O(sqrt(y))$内完成.
  第一次提交WA了一次，很蠢的代码：

  约数的对称性弄错了

C.Nastya and a Wardrobe

题目大意

  一开始有一个数x,k+1天. 前k天，数会翻倍，并且会有50%的几率-1,最后1天数只翻倍不-1,求最后的数的期望.

代码

分析

  手推样例,发现最后得到的所有数形成一个等差数列，公差为1. 最后的答案是$2^{k+1}x - 2^k + 1$.
  我发现x和k很大，把它们读入后都取模了. 对x取模倒没问题，我竟然把k取模了!

D.Nastya and a Game

题目大意

  给定一个序列和常数k. 问有多少个连续的子序列满足该子序列的乘积为和的k倍?

代码

分析

  这题绝了.
  没仔细地看数据范围，一直在想怎么存积. 对一个质数取模搞一搞？ 越想越觉得这道题毒瘤不可做. 存下积后要怎么做呢？ 一开始我推了一个前缀积和前缀和的表达式，想着能不能利用map求方案数，结果发现$i,j$两项是乘在一起的，不能处理……
  看来不能一遍扫描出解，那岂不是要枚举左端点再来快速跳右端点统计答案？注意到乘积的增长非常快，和的相对缓慢一些，如果乘上不是1的数，最多$63$次就爆long long了. 而题目中所有数的和的k倍是不会爆long long的! 因此如果跳右端点，每次往不是1的数上跳，最多只会跳63次. 如果中间隔着一段1呢？这就是并查集的经典应用了. 把它们合并成一个集合，集合的顶端是最右边的1的位置，每次跳到集合顶端即可.

E.Nastya and King-Shamans

题目大意

  长度为n的序列，q次操作，每次把第p个数换成x. 求每次操作完后，是否存在一个位置上的数，使得这个位置前的所有数的和等于该数? 输出其位置.

代码

分析

  一眼线段树题.
  这道题我一开始想着维护区间$[l,r]$内满足要求的数的最左下标，以及$a_i - sum_{i - 1}$,如果一个位置上的$a_i - sum{i - 1} = 0$,则$pos_i = i$,否则$pos_i = n + 1$. pos很好合并，前面这个东西不好搞，不能直接累加合并. 于是我的一个想法就是只操作叶子区间. 向上传递pos即可. 这样做有一个问题：标记无法及时下传. 查询时需要将所有区间都访问一遍，才能修改所有叶子区间的值，并更新pos的值.
  一个非常神的思路：维护区间$[l,r]$内的$a_i - sum{i - 1}$的最大值，如果最大值<0,则不进行查询，如果已经找到了解，也不进行查询，否则将所有位置都搜一遍. 复杂度有点问题，但可以通过全部数据(93组).

小结

  这套题出的非常的吼啊，难度大概是弱省省选，比较考察选手的细致程度和思维以及乱搞能力.(似乎我都不具备)
  BC两道题各WA了一次，而且都是非常低级的错误. B是因为把约数的对称性弄错了，C是因为把幂次取模了. 低级错误在考场上不能再犯，要引起警惕.
  D题差最后一步就能想出来了. 说明对这类子序列计数题还不是很熟.
  E题也是只差最后一步维护最大值. 线段树维护的信息如果不能通过加减法合并，则取最值来合并.
