最小环问题

发表于 2018-06-10 | 分类于 算法竞赛进阶指南 |

两种类别

  1.无向图,输出路径.
  2.有向图,输出路径.

环的本质

  两个点之间的最短路+次短路.

无向图

Description

There is a travel agency in Adelton town on Zanzibar island. It has decided to offer its clients, besides many other attractions, sightseeing the town. To earn as much as possible from this attraction, the agency has accepted a shrewd decision: it is necessary to find the shortest route which begins and ends at the same place. Your task is to write a program which finds such a route.

In the town there are N crossing points numbered from 1 to N and M two-way roads numbered from 1 to M. Two crossing points can be connected by multiple roads, but no road connects a crossing point with itself. Each sightseeing route is a sequence of road numbers y_1, …, y_k, k>2. The road y_i (1<=i<=k-1) connects crossing points x_i and x_{i+1}, the road y_k connects crossing points x_k and x_1. All the numbers x_1,…,x_k should be different.The length of the sightseeing route is the sum of the lengths of all roads on the sightseeing route, i.e. L(y_1)+L(y_2)+…+L(y_k) where L(y_i) is the length of the road y_i (1<=i<=k). Your program has to find such a sightseeing route, the length of which is minimal, or to specify that it is not possible,because there is no sightseeing route in the town.
Input

The first line of input contains two positive integers: the number of crossing points N<=100 and the number of roads M<=10000. Each of the next M lines describes one road. It contains 3 positive integers: the number of its first crossing point, the number of the second one, and the length of the road (a positive integer less than 500).
Output

There is only one line in output. It contains either a string ‘No solution.’ in case there isn’t any sightseeing route, or it contains the numbers of all crossing points on the shortest sightseeing route in the order how to pass them (i.e. the numbers x_1 to x_k from our definition of a sightseeing route), separated by single spaces. If there are multiple sightseeing routes of the minimal length, you can output any one of them.
Sample Input

5 7
1 4 1
1 3 300
3 1 10
1 2 16
2 3 100
2 5 15
5 3 20
Sample Output

1 3 5 2
Source

CEOI 1999

分析

  floyd算法的经典应用. 用floyd算法做这道题，需要明白floyd算法的原理.实质上是动态规划，将阶段放在最外层. $f_{k,i,j}$表示经过编号不大于k的点从i到j的最短路是多少.
  因为环就是两个点之间的最短路+次短路，利用迭代更新求最小环. 即用$d_{i,j} + d_{i,k} + d_{k,j}$更新. 环可以表示成由编号不超过k的节点构成的一条路径和经过节点k的一条路径构成. 因为对称性，这样做不会影响结果. 为什么这么考虑？如果不这样做可能会导致$d_{i,j}$和$d_{i,k} + d_{k,j}$是同一条路径.

代码

有向图

  有向图的环一定是从原点出发，然后再回到原点，有向图跑最短路用dijkstra算法，因为dijkstra算法是处理单源最短路用的，所以要枚举环的起点s. 一开始令d[s] = 0,加入优先队列中扩展，然后将d[s]设置为inf,当s第二次被取出时，d[s]即为所求.

小结

  环的本质是两个点之间两条不同的路径. 无向图上因为两条路径方向可能相同，故用floyd处理，对路径进行分类，避免路径重复. 有向图上因为两条路径方向相反，所以可以特殊处理枚举起点进行dijkstra算法.
